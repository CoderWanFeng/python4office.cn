第 6 讲：发布到微信/网页 - 多渠道发布

2026-03-02 17:25
AI 聊天机器人，微信发布，网页嵌入

大家好，我是正在实战各种 AI 项目的程序员晚枫。

机器人搭好了，接下来最关键的一步——发布，让用户能找到你。

很多人在这一步卡住了，因为微信发布有点复杂。没关系，这讲我手把手带你搞定所有渠道。

🎯 先搞清楚：扣子支持哪些发布渠道？

发布渠道总览

渠道 难度 适合场景 用户量级 推荐指数
抖音 ⭐ 最简单 粉丝互动、获客 大 ⭐⭐⭐⭐⭐
飞书 ⭐ 最简单 企业内部、客户群 中 ⭐⭐⭐⭐
微信 ⭐⭐⭐ 中等 私域运营、公众号 大 ⭐⭐⭐⭐⭐
企业微信 ⭐⭐ 中等 客户服务、销售 中 ⭐⭐⭐⭐
网页嵌入 ⭐⭐ 中等 官网、落地页 不限 ⭐⭐⭐⭐
钉钉 ⭐⭐ 较简单 企业内部 中 ⭐⭐⭐

⚠️ 重要说明：为什么微信发布复杂？

因为微信本身不开放这个接口。

抖音、飞书都是字节/腾讯自家的产品，可以直接接入。但微信（公众号/个人号）需要通过第三方工具中转，所以稍微麻烦一点。

不过别担心，我会把每种方式都讲清楚。

📱 发布到抖音（最简单，推荐！）

为什么先讲抖音？

因为抖音是最简单的，你只需要 3 步就能发布，10 分钟搞定。

而且抖音用户量大，一条爆款视频能带来几百个精准客户。

方式1：抖音私信机器人

适合场景：有人给你发私信，自动回复

步骤：

第1步：在扣子后台发布
→ 点击左侧「📢 发布」
→ 选择「抖音」
→ 关联你的抖音账号
→ 选择「私信机器人」模式
→ 点击「发布」

第2步：在抖音设置
→ 打开抖音 App
→ 进入「我」→「设置」
→ 找到「自动回复」设置
→ 开启「收到私信自动回复」
→ 关联你的扣子机器人

第3步：测试
→ 用另一个账号给你的抖音发私信
→ 看看机器人是否自动回复

效果示意：

用户：发私信 "课程多少钱？"
机器人：自动回复 "我们的课程定价如下：
        💚 基础课：299 元
        💙 实战课：599 元
        有疑问加微信：aiwf365"

方式2：抖音评论区机器人

适合场景：有人评论你的视频，自动回复引导

步骤：

第1步：在扣子后台发布
→ 选择「抖音」
→ 选择「评论区机器人」模式
→ 设置触发关键词（如：课程、咨询、价格）

第2步：在抖音设置
→ 进入视频评论区管理
→ 设置关键词自动回复
→ 关联扣子机器人

第3步：测试
→ 用另一个账号评论触发关键词
→ 看看机器人是否回复

效果示意：

用户：评论 "课程多少钱"
机器人：自动回复 "我们的课程定价如下：
        💚 基础课：299 元
        💙 实战课：599 元
        私信我了解更多~"

📬 发布到飞书（同样简单！）

适合人群

  • 企业内部使用
  • 有飞书群的团队
  • B端客户的服务群

步骤

第1步：在扣子发布
→ 点击「📢 发布」
→ 选择「飞书」
→ 关联飞书账号
→ 点击「发布」

第2步：创建飞书机器人
→ 打开飞书
→ 进入群聊
→ 点击「设置」→「群机器人」
→ 添加自定义机器人
→ 复制 Webhook 地址

第3步：在扣子配置
→ 回到扣子
→ 填写 Webhook 地址
→ 完成配置

第4步：测试
→ 在群里 @机器人 发消息
→ 看是否自动回复

效果示意：

用户：在群里 @小智 "明天上班吗？"
机器人：@用户 您好，我们的工作时间是：
        周一至周五 9:00-18:00
        周末休息~

💬 发布到微信公众号（需要中转，推荐方案）

为什么需要中转？

微信不像抖音/飞书，没有直接开放机器人接口。所以我们需要用一个"中间商"——腾讯云 HiFlow，把公众号消息转发给扣子。

整体原理

用户 → 公众号 → 腾讯云 HiFlow → 扣子机器人 → 回复
                                            ↓
                                        返回 HiFlow → 返回公众号 → 用户

步骤详解

准备工作：

✅ 一个微信公众号（订阅号/服务号都可以）
✅ 一个扣子机器人（已经搭好的）
✅ 腾讯云 HiFlow 账号（免费注册）

第一步：注册腾讯云 HiFlow

1. 打开 hiflow.tencent.com
2. 点击「免费注册」
3. 用微信扫码注册
4. 进入控制台

第二步：在扣子获取 API

1. 打开你的扣子机器人
2. 找到「Bot API」或「API 设置」
3. 复制 API 地址和 Token

第三步：在 HiFlow 创建流程

1. 点击「创建流程」
2. 选择触发器：
   → 搜索「微信公众号」
   → 选择「收到消息时」
   → 授权你的公众号

3. 添加动作：
   → 搜索「HTTP」
   → 选择「发送 HTTP 请求」
   → 填写扣子 API 地址
   → Method: POST
   → Headers: 
     {
       "Authorization": "Bearer 你的Token",
       "Content-Type": "application/json"
     }
   → Body:
     {
       "query": "{{trigger.content}}",
       "user_id": "{{trigger.openid}}"
     }

4. 添加回复动作：
   → 选择「微信公众号」
   → 选择「发送消息」
   → 填写回复内容（可以用 {{steps.1.response}} 返回的内容）

5. 保存并启用流程

第四步：配置微信公众号

1. 登录微信公众平台 mp.weixin.qq.com
2. 进入「设置与开发」→「自动回复」
3. 关闭「被关注自动回复」
4. 开启「接收消息推送」

5. 如果需要客服消息：
   → 进入「设置与开发」→「客服消息」
   → 配置「公众平台后台」
   → 授权给 HiFlow

第五步：测试

1. 用另一个微信给你的公众号发消息
2. 看是否能收到自动回复
3. 如果有问题，检查 HiFlow 日志排查

常见问题排查

❓ 问题：收不到回复
→ 检查 HiFlow 流程是否启用
→ 检查公众号是否授权
→ 查看 HiFlow 日志

❓ 问题：回复很慢
→ HiFlow 免费版有延迟，付费版更快
→ 或者减少 API 调用次数

❓ 问题：回复内容乱码
→ 检查 Content-Type 设置为 application/json
→ 检查 Body 格式是否正确

💼 发布到企业微信

为什么企业微信更简单？

因为企业微信和扣子是打通的，不需要第三方中转，直接配置就行。

步骤

第一步：注册企业微信
→ 打开 work.weixin.qq.com
→ 点击「立即注册」
→ 选择「企业」类型
→ 填写企业信息
→ 完成注册

第二步：在扣子获取配置
→ 扣子后台 → 「📢 发布」
→ 选择「企业微信」
→ 复制 AgentId、Secret、Token

第三步：在企业微信创建应用
→ 企业微信管理后台 → 「应用管理」
→ 创建自建应用
→ 填写应用名称和描述
→ 设置应用可见范围

第四步：配置扣子
→ 填写企业微信 AgentId 和 Secret
→ 配置回调地址
→ 完成验证

第五步：测试
→ 在企业微信里给应用发消息
→ 看是否自动回复

🌐 发布到网页（适合有网站的人）

场景说明

🏢 如果你有：
   → 公司官网（需要客服功能）
   → 产品落地页（需要咨询功能）
   → 个人网站（需要展示功能）

→ 用网页嵌入最合适

扣子官方网页插件

第一步：在扣子发布
→ 「📢 发布」
→ 选择「网页嵌入」
→ 复制嵌入代码

第二步：嵌入到网站
→ 把代码粘贴到网站 HTML 中
→ 放在 </body> 标签之前

代码示例：
<!-- 扣子机器人嵌入代码 -->
<script src="https://www.coze.cn/static/bot.js"></script>
<div id="coze-bot" 
     data-bot-id="你的BotID"
     data-mode="embedded">
</div>

自定义网页嵌入（需要一点前端知识）

如果你想让机器人更美观，可以用扣子 API 自己写前端：

// 核心代码示例
const query = document.getElementById('input').value;

// 调用扣子 API
fetch('https://api.coze.cn/v1/chat', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer 你的Token',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    bot_id: '你的BotID',
    user: 'user_id',
    query: query
  })
})
.then(response => response.json())
.then(data => {
  document.getElementById('output').innerText = data.messages[0].content;
});

📊 多渠道发布策略

根据业务选择

业务类型 主攻渠道 辅助渠道
电商卖家 抖音 微信、网页
企业服务 微信、网页 飞书
内容创作者 抖音、B站 微信
个人IP 微信、抖音 知乎、小红书

统一知识库策略

💡 核心原则：同一套知识库，多个渠道发布

✅ 做法：
1. 在扣子维护一个核心知识库
2. 所有渠道都发布同一个机器人
3. 或者用同一知识库同步到多个机器人

好处：
→ 维护成本低（改一处，全渠道生效）
→ 用户体验一致
→ 数据统一分析

多渠道管理技巧

📋 多渠道发布后的维护：

✅ 每天：
→ 查看各渠道对话数据
→ 关注用户反馈

✅ 每周：
→ 分析各渠道效果
→ 优化渠道配置

✅ 每月：
→ 评估渠道 ROI
→ 调整渠道策略

⚠️ 渠道发布避坑指南

❌ 坑1：只做一个渠道
   → 用户在哪里，就去哪里发布
   → 不要只守着一个渠道

❌ 坑2：多渠道知识库不一致
   → 各渠道回答不一样，用户会困惑
   → 建议统一知识库

❌ 坑3：发布后不测试
   → 每个渠道都要测试
   → 确保功能正常

❌ 坑4：忽视渠道限制
   → 微信限制多，注意合规
   → 抖音/飞书相对开放

🛠️ 本讲作业

✅ 作业1：发布到至少1个渠道
   → 优先推荐：抖音（最简单）
   → 如果有微信公众号码：按教程配置 HiFlow

✅ 作业2：测试你的发布效果
   → 用另一个账号测试
   → 确保能正常收到回复

✅ 作业3：规划其他渠道
   → 想想你的用户还在哪里
   → 列出可以扩展的渠道
