OpenClaw Tools 工具调用 Exec Browser 文件操作

大家好，我是正在实战各种AI项目的程序员晚枫。

今天学习工具调用（Tools），这是OpenClaw最强大的功能。通过工具，AI可以执行命令、操作浏览器、读写文件…从"聊天机器人"变成"智能助手"。

什么是工具调用？

**工具（Tools）**是AI可以调用的功能函数。当AI需要执行某些操作时，它会主动请求使用工具。

工作流程

用户提问 → AI分析 → 需要工具？
    ↓ 否
直接回答
    ↓ 是
选择工具 → 传入参数 → 执行 → 返回结果 → AI总结

OpenClaw内置工具

工具 功能 风险等级
exec 执行shell命令 🔴 高
browser 控制浏览器 🟡 中
read 读取文件 🟢 低
write 写入文件 🟡 中
edit 编辑文件 🟡 中
web_search 网络搜索 🟢 低
web_fetch 抓取网页 🟢 低

配置工具权限

编辑 ~/.openclaw/openclaw.json：

{
  "tools": {
    "exec": {
      "enabled": true,
      "sandbox": true,
      "allowedCommands": ["ls", "cat", "grep", "curl", "python3"],
      "blockedCommands": ["rm", "dd", "mkfs", "sudo"],
      "workingDirectory": "~/.openclaw/sandbox",
      "requireApproval": false
    },
    "browser": {
      "enabled": true,
      "headless": true,
      "timeout": 30000
    },
    "read": {
      "enabled": true,
      "allowedPaths": ["~/projects", "~/documents"],
      "blockedPaths": ["~/.ssh", "~/.aws", "~/.env"]
    },
    "write": {
      "enabled": false,
      "requireApproval": true
    }
  }
}

安全最佳实践

生产环境建议

  1. 启用沙盒模式：限制命令执行范围
  2. 白名单机制：只允许特定命令
  3. 人工确认：危险操作前要求确认
  4. 日志审计：记录所有工具调用

危险操作示例

{
  "tools": {
    "exec": {
      "blockedCommands": [
        "rm -rf /",
        "dd if=/dev/zero",
        "mkfs",
        "format",
        "> /etc/passwd"
      ]
    }
  }
}
