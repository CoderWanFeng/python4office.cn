第 11 讲：知识问答机器人 - 基于文档的智能问答

2026-03-02 17:50
AI 聊天机器人，知识问答，文档问答

大家好，我是正在实战各种 AI 项目的程序员晚枫。

你有没有遇到过这种情况：

  • 公司有一堆文档，员工想找某个信息，翻半天找不到
  • 想做培训，但每次都要重复讲同样的内容
  • 想做个 FAQ，但维护起来太麻烦了

这一讲，我教你做一个知识问答机器人，基于你的文档自动回答问题。

你把文档扔给它，它就是你们公司最懂业务的"员工"。

🎯 先说说什么是知识问答机器人

一句话定义

把一堆文档上传给机器人，用户问问题，机器人从文档里找答案回答。

📄 你上传的文档：
├── 员工手册.pdf
├── 产品手册.docx
├── 制度文件.docx
└── 培训资料.pptx

🤖 机器人能力：
用户：这个岗位怎么晋升？
    → 机器人从员工手册里找答案 → 回答

用户：XX产品怎么用？
    → 机器人从产品手册里找答案 → 回答

用户：请假制度是什么？
    → 机器人从制度文件里找答案 → 回答

和普通问答机器人的区别

❌ 普通问答机器人：
   → 需要你一条条手动写问题和答案
   → 知识库 = 人工整理的问答对
   → 适合：FAQ、客服话术

✅ 知识问答机器人：
   → 把你现有的文档扔进去
   → 机器人自动理解文档内容
   → 用户问任何相关问题，机器人从文档里找答案
   → 适合：手册、制度、培训资料、产品文档

📄 知识库文档准备

支持的文档格式

✅ 扣子支持直接上传：

├── 📄 PDF文档          → 最推荐，格式稳定
├── 📝 Word文档         → .docx 格式
├── 📊 Excel表格        → .xlsx 格式
├── 📊 CSV表格          → .csv 格式
├── 📽️ PPT演示文稿     → .pptx 格式
├── 📝 TXT纯文本        → 最简单
└── 🌐 网页内容         → URL链接

💡 建议：优先用PDF，格式最稳定

文档质量标准

好的文档特点：
✅ 结构清晰：有标题、章节、目录
✅ 内容准确：信息正确，没有错误
✅ 格式规范：排版整齐，阅读体验好
✅ 内容完整：覆盖全面的知识点

❌ 差的文档问题：
❌ 扫描版PDF（无法识别文字）
❌ 图片为主的文档（AI识别效果差）
❌ 格式混乱（排版不规范）
❌ 内容过时（信息已失效）

文档处理技巧

技巧1：把长文档拆分成多个短文档

❌ 不推荐：
上传一本500页的员工手册（整本书）

✅ 推荐：
├── 01-公司简介.md
├── 02-组织架构.md
├── 03-入职指南.md
├── 04-考勤制度.md
├── 05-请假制度.md
├── 06-报销制度.md
└── ...

好处：
→ 机器人更容易精准定位答案
→ 用户问具体问题，答案更准确

技巧2：用表格整理FAQ

📊 知识库FAQ表格格式：

┌────────────────┬──────────────────────────────┐
│ 问题            │ 答案                         │
├────────────────┼──────────────────────────────┤
│ 入职需要准备什么 │ 身份证、学历证书、体检报告...   │
│ 请假怎么申请    │ 在OA系统提交申请，审批通过后... │
│ 工资发放日期    │ 每月15日发放上个月工资...      │
└────────────────┴──────────────────────────────┘

→ 直接上传Excel，扣子会自动解析
→ 比逐条添加效率高很多

技巧3：给文档加摘要/说明

上传文档时，可以添加文档说明：

文档名称：员工手册2026版.pdf
文档说明：本手册包含公司制度、员工福利、行为规范等内容。
         发布日期：2026年1月
         适用范围：全体员工

→ 文档说明帮助AI更好地理解文档内容
→ 提高回答准确率

🛠️ 扣子配置教程

步骤1：创建知识库

扣子后台 → 「📚 知识库」→ 「创建知识库」
→ 输入知识库名称
→ 选择知识库类型（文档库 / FAQ）
→ 确认创建

步骤2：上传文档

创建知识库后 → 「上传文档」

支持方式：
├── 拖拽上传（直接拖文件）
├── 点击上传（从电脑选择）
└── URL导入（输入网页链接）

上传后：
→ 自动解析文档内容（需要等待几秒~几分钟）
→ 解析完成后可以看到文档摘要
→ 检查解析是否完整

步骤3：关联到机器人

上传完文档 → 「关联到Bot」

→ 选择要关联的机器人
→ 确认关联
→ 机器人就能回答文档里的内容了

步骤4：测试效果

关联后 → 「💬 调试与预览」

测试问题：
→ 从文档里找几个问题测试
→ 看机器人能否准确回答
→ 检查回答是否引用了正确的文档内容

⚠️ 知识问答机器人避坑

❌ 坑1：上传扫描版PDF
   → AI无法识别扫描件
   → 回答效果极差

   ✅ 避坑：用文字版PDF，或者重新识别

❌ 坑2：上传一个大而全的文档
   → 内容量太大，机器人难以精准定位
   → 回答不够准确

   ✅ 避坑：把大文档拆分成多个小文档

❌ 坑3：知识库长期不更新
   → 文档内容过时，机器人回答错误
   → 用户被错误信息误导

   ✅ 避坑：定期更新知识库，标注日期

❌ 坑4：机器人"幻觉"
   → 用户问的问题文档里没有，但机器人瞎编
   → 传播错误信息

   ✅ 避坑：人设设定中明确"不知道就说不知道"
           开启"仅在知识库中回答"模式
