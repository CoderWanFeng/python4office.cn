今天学习故障排查与优化，掌握诊断问题的方法和提升性能的技巧。

日志系统

日志文件位置

~/.openclaw/state/logs/
├── gateway.log       # Gateway主日志
├── channels.log    # 通道相关日志
├── sessions.log    # 会话日志
├── tools.log       # 工具调用日志
├── audit.log       # 审计日志
└── error.log      # 错误日志

查看实时日志

# 查看Gateway日志
tail -f ~/.openclaw/state/logs/gateway.log

# 查看特定通道日志
tail -f ~/.openclaw/state/logs/channels.log | grep telegram

常用诊断命令

系统状态检查

openclaw gateway status
openclaw doctor
openclaw config validate
openclaw system stats

通道诊断

openclaw channels list
openclaw channels test telegram
openclaw channels reconnect whatsapp

常见问题解决

问题1：Gateway无法启动

# 检查端口占用
lsof -i :18789

# 检查配置文件
openclaw config validate

# 查看详细错误
tail -50 ~/.openclaw/state/logs/gateway.log

问题2：AI不回复消息

# 检查API密钥
openclaw providers test anthropic

# 检查通道状态
openclaw channels status telegram

问题3：工具调用失败

# 检查工具权限
cat ~/.openclaw/openclaw.json | jq '.tools'

# 查看工具日志
tail -f ~/.openclaw/state/logs/tools.log

性能优化

内存优化

{
  "sessions": {
    "maxHistory": 50,
    "ttl": 3600
  }
}

缓存配置

{
  "cache": {
    "enabled": true,
    "ttl": 3600,
    "maxSize": "100MB"
  }
}

响应时间 <2s >5s
错误率 <1% >5%
