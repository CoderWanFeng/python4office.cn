想用AI写代码？先申请个"API Key"身份证

2026-04-16 16:55 · AI 基础概念
Python AI编程 API Key API

作者：程序员晚枫

有人把API Key发到GitHub上，一晚上被刷了2万块。有人把Key贴到微信群里"请教问题"，第二天账户就欠费了。

API Key不是一串乱码，它是你的钱包钥匙。 谁拿到了，谁就能用你的钱调AI。

今天咱们把API Key彻底讲清楚——它是什么、怎么用、怎么保命。

🎯 一句话先说清楚

::: tip 核心结论 API Key = 你的AI钱包钥匙 + 身份证

没有它，AI服务不认识你；有了它，你每次调AI都花钱。泄露了？别人拿你的钱白嫖AI。 :::

💡 API Key到底是什么？一个类比就懂

🏦 去银行取钱

你去银行取钱，柜员第一件事让你干嘛？出示身份证。

没有身份证 → 柜员拒绝办理 有身份证 → 知道你是谁 → 可以取你的钱

API Key就是AI服务的"身份证"：

场景 没有API Key 有API Key
调用ChatGPT ❌ 拒绝服务 ✅ 识别身份，开始工作
查用量 ❌ 不知道谁在查 ✅ 查你的账户数据
扣费 — ✅ 按Token从你账户扣钱

但比身份证更危险的是——身份证丢了别人没法直接取钱，API Key丢了，别人可以直接花你的钱。

🚨 API Key泄露的真实代价

这不是吓你，是真实发生的事：

案例1：GitHub泄露，一夜被刷2万

某开发者把代码推到GitHub
代码里硬编码了API Key：api_key = "sk-proj-AbCdEf1234..."
→ GitHub是公开的，全世界都能看到
→ 爬虫专门扫描GitHub找泄露的Key
→ 有人用他的Key疯狂调GPT-4
→ 一晚上刷了2万块

案例2：微信群截图，当天欠费

某用户在群里截图"请教API Key格式"
截图里完整Key清晰可见
→ 群里几百人看到了
→ 有人复制去用
→ 当天账户就欠费了

案例3：前端代码暴露，被白嫖

有人把API Key写在网页JS里
→ 用户F12就能看到
→ 被人拿去批量调用
→ 账单暴涨

记住：API Key = 密码 + 钱包。泄露了不是"别人能冒充你"，是"别人能花你的钱"。

🔍 API Key长什么样？

不同平台的Key格式不同：

平台 Key格式 免费额度
DeepSeek sk-xxxxx ✅ 有
阿里通义 sk-xxxxx ✅ 有
OpenAI sk-proj-xxxxx ❌ 新号需绑卡
Claude sk-ant-xxxxx ❌ 需付费

怎么申请？三步搞定：

  1. 去平台官网注册账号
  2. 找到"API Keys"或"密钥管理"页面
  3. 点击"创建新密钥"，复制保存（注意：大多数平台只显示一次！）

💰 API Key和钱的关系：每次调AI都在花钱

API Key不是"申请了就能免费用"，它绑定的是你的付费账户：

你用API Key调AI
  ↓
AI系统识别你的账号
  ↓
记录你用了多少Token（上一讲讲过）
  ↓
按Token数量从你账户扣钱
  ↓
给你发账单

平台 输入价格（/百万Token） 输出价格（/百万Token） 1万字约花多少

🛡️ API Key安全：4条铁律

铁律1：代码里永远别硬编码Key

# ❌ 作死写法——推到GitHub全世界都看到了
api_key = "sk-proj-xxxxx"

# ✅ 正确写法——从环境变量读取
import os
api_key = os.getenv("OPENAI_API_KEY")

铁律2：用.env文件统一管理

# 项目根目录创建 .env 文件
OPENAI_API_KEY=sk-proj-xxxxx
DEEPSEEK_API_KEY=sk-xxxxx

# .gitignore 里加上
.env

换电脑、换环境只改一个文件，不用翻代码找Key在哪。

铁律3：每个平台单独Key + 设月度预算

❌ 所有服务用一个Key → 泄露一个全完
✅ 每个平台单独Key → 泄露一个只影响一个

大多数平台支持设置"月度消费上限"，新手建议设50-100元/月。

铁律4：每3个月换一次Key

即使没泄露也定期换。开启平台账单告警，发现异常消费立即撤销Key。

🔥 API Key常见术语翻译

“需要先申请API Key”

= 去注册账号，在控制台申请一个"钥匙"

“API Key泄露了”

= 你的"钱包钥匙"被别人拿到了，立刻去平台撤销！

泄露了怎么办：

  1. 立刻去平台撤销该Key（优先级最高）
  2. 申请新Key
  3. 检查最近使用记录，看有没有异常消费

“统一API格式”

= 换模型不用改代码，只换Key和接口地址就行。现在各家都在向OpenAI格式靠拢

⚠️ 常见误区避坑

❌ “API Key可以分享给同事”

错！ 每个人申请自己的Key。共享Key = 共享钱包，出问题谁负责？

❌ “GitHub上很多代码都有Key，没事吧”

危险！ GitHub有专门的爬虫扫描泄露的Key，发现了会自动通知，但之前可能已经被用了。

❌ “用别人的Key也行”

违规。 大部分平台禁止共享Key，被发现会封号，别人的Key随时可能失效。
