# 可执行文件目录结构说明

## 新的统一输出目录

所有打包后的可执行文件现在统一保存在 `code/exe/` 目录下，按照工具类型分类：

```
code/
├── exe/                    # 统一的可执行文件目录
│   ├── Excel工具/          # Excel相关工具的可执行文件
│   ├── Word工具/           # Word相关工具的可执行文件
│   ├── PDF工具/            # PDF相关工具的可执行文件
│   └── 其他工具/          # 其他工具的可执行文件
├── excel_tools/           # Excel工具源代码
├── word_tools/            # Word工具源代码
├── pdf_tools/             # PDF工具源代码
├── other_tools/           # 其他工具源代码
└── run_all.bat           # 启动所有工具的脚本（更新为指向exe目录）
```

## 优化内容

1. **统一输出目录**：所有可执行文件集中到 `code/exe/` 目录
2. **分类存储**：按工具类型创建子目录，便于管理
3. **自动创建**：脚本会自动创建必要的目录结构
4. **启动脚本更新**：`run_all.bat` 已更新为从新的 `exe` 目录启动所有工具

## 使用方法

1. 打包所有工具：
   ```bash
   python build_all.py
   ```

2. 启动所有工具（Windows）：
   ```bash
   run_all.bat
   ```

3. 单独启动某个工具：
   ```bash
   # Excel工具
   start code/exe/Excel工具/01_excel_merge.exe
   
   # PDF工具
   start code/exe/PDF工具/11_pdf_merge.exe
   ```

## 优势

- **集中管理**：所有可执行文件集中在一个目录下，便于备份和分发
- **保持结构**：按工具类型分类，保持原有的组织结构
- **避免污染**：源代码目录不再包含可执行文件
- **便于部署**：只需复制 `exe` 目录即可部署所有工具
