Ark Coding Plan Says Unlimited Tools, But 36 Yuan Really Isn't Enough...

TL;DR

Models are unlimited, tools are unlimited, but 36 yuan won't last a week.

This isn't a product problem, it's my problem — I use it too heavily.

What is Ark Coding Plan?

Official says:

Supports Doubao, GLM, DeepSeek, Kimi, MiniMax and other models, unlimited tools

Sounds great, right?

Doubao, GLM, DeepSeek, Kimi, MiniMax...

Use freely, unlimited.

Then I saw the price:

36 yuan/month (after 10% discount)

Not bad, the longer the subscription, the better the deal.

I believed it.

Week 1: Really香 (Truly Awesome)

Switch Models Freely

I tried each model:

  • Coding questions → Use DeepSeek (cheap and effective)
  • Writing copy → Use Kimi (great Chinese understanding)
  • Complex tasks → Use GLM (strong reasoning)

Switch around, cross-verify.

Use Tools Freely

Write code, debug, review, explain...

Unlimited tools is real.

Way better than those tools that only let you use one model.

End of Week

"Your monthly quota has been exhausted."

Huh?

Actual Usage of 36 Yuan

Let me do the math:

My Usage Pattern

DayMain Tasks消耗
MondayWrite code + review~40 yuan
TuesdayFix bugs + ask questions~35 yuan
WednesdayWrite proposals + edit copy~30 yuan
ThursdayRefactor code + test~45 yuan
FridayWrite docs + summarize~30 yuan
Total180 yuan equivalent

Used 180 yuan worth in one week, but only spent 36 yuan.

No wonder it ran out.

Why Does It Run Out So Fast?

Reason 1: Multi-Model Switching Costs More

Before, using a single tool meant fixed quota consumption.

Now with 5 models to choose from, I switch back and forth:

Each model is calculated separately, quota depletes faster.

Reason 2: Unlimited Tools = More Usage

Before, I was afraid of running out, so I'd handle tasks myself.

Now since "tools are unlimited", I just ask directly when I encounter problems:

  • "What does this code mean?" → Ask
  • "How do I fix this bug?" → Ask
  • "Write me a test case" → Ask
  • "Optimize this logic" → Ask

Shifted from "afraid to use" to "use freely."

Reason 3: Complex Tasks Cost More

Simple tasks:

  • "How do I define a function in Python" → Low consumption

Complex tasks:

  • "Help me refactor this 300-line class" → High consumption
  • "Write me a complete user authentication module" → Very high consumption

Ark Coding Plan's model capability is strong, so I prefer letting it handle complex tasks.

The more you use it, the better it feels; the better it feels, the more you use it.

Use Doubao, consume 3 yuan equivalent

"Optimize this logic for me"

Use GLM, consume 4 yuan equivalent

Daily total: 21 yuan equivalent

36 yuan lasts how many days? Less than 2 days.

Official Says "Longer Subscriptions Are Better Value"

This is true.

Monthly 36 yuan → Yearly even cheaper → Team version even better value

But the problem is:

I use a lot, it's not a subscription method issue.

So What To Do?

Solution 1: Use Less

  • Solve simple problems yourself
  • Only use free models
  • Control usage frequency

Drawback: Reduces efficiency, putting the cart before the horse

Solution 2: Subscribe to Higher Tier

  • Higher quota
  • Or unlimited

Drawback: More expensive

Solution 3: Pair with Free Tools

  • Use free tools for simple tasks
  • Save Ark Coding Plan for complex tasks

Drawback: Need to manage multiple tools

Who It's For / Not For

For:

  • Light users
  • Occasionally switching models
  • Want to experience multi-model

Not for:

  • Programmers (usage too high)
  • High-frequency daily users
  • Heavy users relying on AI for efficiency

Summary

Ark Coding Plan is a great product.

Many models, unlimited tools, strong features.

But 36 yuan/month, for programmers, really isn't enough.

If you're a light user, 36 yuan is sufficient.

If you're a heavy user like me, either pay more or pair with free tools.
