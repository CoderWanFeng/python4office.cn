我用AI写了100篇技术文章后，总结出一套不焦虑的创作工作流

发表于2025-04-05|更新于2026-04-30
|阅读量:

最近收到很多读者私信："晚枫，AI这么强，我是不是要被取代了？"

我的回答始终如一：AI是杠杆，不是对手。会用AI的人，淘汰不会用AI的人。

从2024年开始all in AI编程实战，到现在用AI辅助创作了100+篇技术文章，我想分享一套真正实用的AI创作工作流——不是那种花里胡哨的 prompt 工程，而是能落地、能复现、能让你少加班的方法。

一、我的AI创作三板斧

1. 选题：让AI当你的情报员

很多人写不出文章，不是因为不会写，而是不知道写什么。

我的做法很简单：

关键心法：不要问AI"有什么热门话题"，而是告诉它你的受众是谁、你想解决什么问题。AI给你的答案会精准10倍。

2. 写作：人机协作，而不是人机对抗

我从不让AI直接生成整篇文章。为什么？

因为AI写的文章，有知识没温度，有逻辑没灵魂。

我的工作流是这样的：

步骤我做什么AI做什么
1确定核心观点和大纲结构-
2写开头（抓人眼球）-
3-根据大纲扩写每个论点
4修改AI生成的内容，加入个人案例-
5-生成3个备选标题
6选择并优化最终标题-

记住：你是主编，AI是实习生。 实习生的稿子能直接用吗？不能。但它能帮你省掉80%的重复劳动。

3. 优化：用数据说话

文章发出去只是开始。我会用AI分析：

  • 哪段话读者停留时间最长？
  • 哪个标题点击率最高？
  • 评论区最常问的问题是什么？

这些数据会反哺下一篇文章的选题和写作。

二、3个我亲测好用的AI工具组合

组合一：DeepSeek + Cursor（技术写作神器）

  • DeepSeek：负责选题、资料搜集、初稿生成
  • Cursor：代码示例自动生成、技术细节校验

适合写技术教程类文章，比如我最近写的《用Python自动处理Excel的10个场景》。

组合二：ChatGPT + python-office（自动化发布）

这是我自己的开源项目组合拳：

从写作到发布，全程自动化。现在我10分钟能发8篇文章，不是因为我写得快，是因为工具用得好。

组合三：Claude + Notion（知识管理）

  • Claude：长文理解、知识萃取
  • Notion：素材库管理、写作计划

所有读过的书、看过的文章、灵光一现的想法，都丢进Notion。写的时候让Claude帮我检索、整合。

三、关于AI焦虑，我想说几句

作为一个法律硕士转行的程序员，我见过太多"技术焦虑"。

35岁危机、AI取代程序员、内卷……这些词天天在耳边响。

但我的态度是：焦虑没用，行动有用。

AI不会取代程序员，但会用AI的程序员会取代不会用的。

这不是威胁，这是机会。

就像当年Excel没有取代会计，而是让会计从算盘里解放出来，去做更有价值的财务分析。

AI也是一样。它帮你写代码、写文章、做PPT，不是为了让你失业，是为了让你有时间去做更有创造力的事。

四、给你的行动清单

如果你也想用AI提升创作效率，建议按这个顺序来：

  1. 第一周：选一个你常用的AI工具，深度使用，熟悉它的脾气
  2. 第二周：用它辅助完成一篇完整的文章，记录卡点
  3. 第三周：优化你的工作流，形成SOP
  4. 第四周：尝试自动化，让AI帮你做重复性工作

不要追求完美，先完成再完美。

写在最后

有人问我：晚枫，你40岁想退休，是不是因为赚够了？

我说不是。是因为我想证明：程序员可以不被焦虑绑架，可以用技术创造自由。

AI是这个时代给程序员最好的礼物。用好了，它能帮你提前10年过上想要的生活。

用不好，它确实会成为你的竞争对手。

选择权在你手里。
