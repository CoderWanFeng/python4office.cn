# 系列化命名规范

> 统一命名，让内容成体系。
> 
> 核心原则：看到名字就知道是什么系列、哪一篇。

---

## 当前系列

### 口播稿系列

命名格式：`口播稿-NNN-标题.md`

示例：
```
口播稿/001-做了X年AI博主，我想聊一个很多人不会讲的问题.md
口播稿/002-明明国内算力更稀缺，为什么反而更便宜.md
口播稿/012-GPT-5.5突然被禁了？OpenAI内部到底发生了什么.md
```

### 公众号系列

命名格式：`公众号-NNN-标题.md`

示例：
```
公众号/001-那些让你焦虑的AI博主，没告诉你的3个真相.md
公众号/002-明明国内算力更稀缺，为什么反而更便宜.md
```

### 个人叙事系列

命名格式：`自由职业第NNN天-事件.md`

示例：
```
log/2025/自由职业第001天-开始.md
log/2025/自由职业第010天-第一次收入.md
log/2025/自由职业第018天-月入过万.md
```

### AI热点解读系列

命名格式：`AI热点-NNN-事件.md`

示例：
```
2026/AI热点-001-GPT5.5被禁.md
2026/AI热点-002-腾讯云涨价.md
```

---

## 命名建议

### 新系列模板

如果你要开新系列，建议：

```
系列名/NNN-具体标题.md
```

其中：
- `系列名` - 中文，简短
- `NNN` - 3位数字，从001开始
- `标题` - 20字以内，能表达核心

---

## 禁止的做法

❌ 随意命名
```
xxx-final-v2-最终版.md
xxx-备份.md
xxx-思考-突然想到.md
```

❌ 中英混杂
```
AI-分析-2026.md
GPT-5.5-thoughts.md
```
