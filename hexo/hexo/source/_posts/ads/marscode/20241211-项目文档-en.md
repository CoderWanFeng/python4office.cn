Hey everyone, this is 程序员晚枫 (Programmer Wanfeng), currently all in on [AI Programming Practice](https://mp.weixin.qq.com/s/8p2eviFUmYa1V0pswmDRmw).

Like everyone else, every time I hear about writing documentation, my head hurts.

So I thought — can I use a large language model to write docs?

Coincidentally, I've been iterating on my open-source project python-office, and for one sub-project, popdf, I wrote all the code comments and project documentation using large language models.

The large language model I used: MarsCode Programming Assistant.

## Requirements

I developed the python-office open-source project for handling documents in Python, and it's been 5 years. But because I'm lazy, the documentation has never been great.

However, this project is mainly for beginners, so documentation is extremely important.

So this time I decided to write a document for each feature, so everyone can use it on demand.

## Implementing with Large Language Models

### 1. Install the Plugin
First, install the MarsCode Programming Assistant plugin in VS Code.

### 2. Generate Document Templates

Then, enter this command in the plugin:

> Based on the python-office code repository: https://github.com/CoderWanFeng/python-office, write documentation for the popdf module of the python-office project.

### 3. Edit the Documentation

Next is copy-paste, then modify the details.

But during modification, MarsCode provides documentation completion features, which greatly improved my work efficiency.

### 4. Results

Finally, I finished generating all the project documentation, and the results were very good.

As you can see, using MarsCode to write documentation is much faster than doing it myself.

I also put this project's documentation on Gitee — you can download and use it directly.

> https://gitee.com/CoderWanFeng/python4office.cn/tree/main/hexo/hexo/source/_posts/python-office/popdf


## Closing Thoughts

Tech people focus more on the details of technical implementation, but writing documentation is also a very important task for any project.

For repetitive descriptions, you can completely delegate them to large language models.
