## 先说结论

| 类型 | TokenPlan | Coding Plan |
|------|----------|------------|
| 计费方式 | **按 Token** | **按月订阅** |
| 价格波动 | 会变 | **固定** |
| 适合 | 调用量大 | 调用量小 |

**简单说：TokenPlan 像"电费"，Coding Plan 像"包月宽带"**。

---

## 3 大核心区别

### 区别 1：计费方式

**TokenPlan（按量付费）**
- 用多少付多少
- 按 Token 数计费
- 1 个 Token ≈ 0.0001-0.001 元

**Coding Plan（订阅付费）**
- 按月订阅
- 不限次数
- ¥9.4-80/月

### 区别 2：价格波动

**TokenPlan**
- 高峰期会涨价
- 不同模型价格不同
- 写代码用得多 = 烧钱快

**Coding Plan**
- **价格固定**
- 不会因为用得多涨价
- 心理账户稳定

### 区别 3：使用门槛

**TokenPlan**
- 需要懂技术
- 需要看用量
- **新手不友好**

**Coding Plan**
- 不需要懂技术
- 不需要看用量
- **新手友好**

## 4 类人，对号入座

### 👶 完全没编程基础

**建议：Coding Plan**

原因：TokenPlan 需要看用量、算成本，新手搞不懂。Coding Plan 一口价，省心。

### 👨‍💻 自学编程的初学者

**建议：Coding Plan**

原因：每天都在写代码，TokenPlan 烧钱快，Coding Plan 包月划算。

### 💼 专业程序员

**建议：看情况**
- 公司项目多 → **Coding Plan**
- 自己的小项目 → **TokenPlan**（用多少付多少）
- 混合使用 → **Coding Plan + TokenPlan**

### 🏢 企业用户

**建议：Coding Plan**

原因：企业需要"可预测的成本"，Coding Plan 固定价格，方便财务预算。

## 该选哪个？3 步决策

### 步骤 1：估算月使用量

| 月使用量 | 推荐 |
|---------|------|
| < 5 次 | TokenPlan |
| 5-50 次 | Coding Plan（体验版） |
| 50-200 次 | Coding Plan（标准版） |
| > 200 次 | Coding Plan（专业版）|

### 步骤 2：考虑稳定性

- 喜欢"一口价" → **Coding Plan**
- 喜欢"用多少付多少" → **TokenPlan**

### 步骤 3：考虑技术能力

- 懂技术、会算账 → **TokenPlan**
- 不懂技术、图省心 → **Coding Plan**
