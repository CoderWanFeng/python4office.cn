## 🔍 2. 创作前准备

> ⚠️ **核心原则**：选题必须**基于数据、不靠拍脑袋**。本节所有搜索动作必须**调用以下两个 skill** 完成，**禁止用 AI 内置知识代替实时热搜**。

### 2.1 强制使用的两个 Skill（每次必拉最新版本）

每次创作前，**必须先拉取最新版本**这两个 skill，再按其配置执行：

| Skill | 仓库路径 | 职责 | 调用方式 |
|------|---------|------|---------|
| **hot-topic-radar**（热点雷达） | [CoderWanFeng/wanfeng-skills/tree/main/skills/hot-topic-radar](https://github.com/CoderWanFeng/wanfeng-skills/tree/main/skills/hot-topic-radar) | 7×24h 监控**全网**热搜（微博/抖音/知乎/小红书/B站/今日头条/百度），按关键词过滤 + 领域相关度打分 + 记忆去重，输出「今天值得跟」的 TOP 选题清单 | `python skills/hot-topic-radar/scripts/fetch_hot.py --stdout` |
| **aihot-topic-selector**（AI 圈选题器） | [CoderWanFeng/wanfeng-skills/tree/main/skills/aihot-topic-selector](https://github.com/CoderWanFeng/wanfeng-skills/tree/main/skills/aihot-topic-selector) | 专注 **AI 圈垂直**热点（AIHOT 数据源），按「核心 2 问」从候选清单中筛出 1 条最值得跟的 | 读取 `SKILL.md` 按其流程执行 |

**拉取最新版本的命令**（每次创作前必跑）：

```bash
# 1. 拉取 / 更新 skill 仓库
cd <wanfeng-skills 本地路径>
git pull origin main

# 2. 确认两个 skill 的 SKILL.md 是最新的
ls -la skills/hot-topic-radar/SKILL.md skills/aihot-topic-selector/SKILL.md
```

> ❌ **禁止做法**：
> - 用 AI 训练数据里的"近期热点"代替实时热搜（信息可能滞后数月）
> - 跳过 skill 直接凭印象写选题
> - 不拉取最新版本就调用（旧版本关键词/排除词可能已失效）

### 2.2 选题三步流程（强制顺序）

**Step 1 · 全网扫描（hot-topic-radar）**

```bash
python skills/hot-topic-radar/scripts/fetch_hot.py --stdout
```

输出 TOP 10 候选清单（带热度 × 相关度综合分）。重点关注分数 ≥ 60 的条目。

**Step 2 · AI 圈垂直筛选（aihot-topic-selector）**

读取 `skills/aihot-topic-selector/SKILL.md`，把 hot-topic-radar 输出的清单交给它，按「核心 2 问」筛出 1-3 条 AI 圈强相关的选题。

**Step 3 · 选题确认 4 步**

1. 看 hot-topic-radar + aihot-topic-selector 输出的候选清单
2. 判断是否有可借势热点
   - **有**：从清单里挑 1-3 条做切入角度
   - **无**：从第 8 章「三大内容合集」里选
3. 结合产品/活动推广需求（参考 7.6 返佣链接规则）
4. 确认最终选题和标题（参考第 3 章标题规范）

### 2.3 关键词与排除词配置（参考）

`hot-topic-radar` 的 `radar-config.yaml` 默认关键词配置（首次使用需核对）：

```yaml
keywords:
  - AI
  - 大模型
  - ChatGPT
  - Claude
  - Sora
  - 编程
  - 副业
exclude_keywords:
  - 娱乐
  - 明星
  - 电视剧
```

> 💡 **配置更新原则**：如果发现候选清单质量差（全是噪音 / 漏掉重要热点），按 `hot-topic-radar` §九「故障排查」调整 `keywords` / `exclude_keywords` / `weights.relevance`。

## 📌 3. 标题写作规范 · 必须执行

> ⚠️ **标题是爆款的第一要素**，决定了打开率。本节规则来自 `viral-title` 标题 skill（[kangarooking/kangarooking-skills](https://github.com/kangarooking/kangarooking-skills)），并结合「公众号内容的创作指南」实战沉淀。
>
> **适用范围（重要）**：本节规则**同时覆盖主标题和正文小标题**（`##` 二级 / `###` 三级）。每个小标题也必须按本节规则打磨，不允许"主标题打磨、小标题随便写"。

### 3.1 核心公式（3 元素，缺一不可）

```
具体对象 + 情绪/反差 + 悬念/判断
```

**更强版本**（建议优先使用）：

```
具体数字/时间 + 大厂/产品名 + 反常识判断 + 情绪词
```

**3 元素含义**：
| 元素 | 作用 | 示例 |
|------|------|------|
| **具体对象** | 防止空泛，让读者一眼知道在说谁 | Copilot、Netflix、国产 AI、这本开源书 |
| **情绪/反差** | 制造冲突感，触发点击冲动 | 涨价 26 倍、不懂中文、太贵 |
| **悬念/判断** | 留下钩子 / 亮出立场，让读者想知道答案或想反驳 | 时代结束了、问题、这件事 |

### 3.2 高点击标题 5 类模板（必须改 5 版时各选 1 类）

| 类型 | 模板 | 实战示例 |
|------|------|---------|
| **数字冲击** | X 变成 Y，某件事结束了 | `Copilot涨价26倍，AI免费时代结束了` |
| **反常识** | 明明 A，为什么反而 B | `明明是国产AI，为什么反而不懂中文` |
| **个人体验** | 我用了 X 天，发现一个问题 | `我用Codex一周，发现它最让我意外的不是写代码` |
| **避坑提醒** | 别急着 X，先看 Y | `别急着买AI电脑，先看这3个问题` |
| **结果导向** | 用 X 做 Y，结果 Z | `AI编码太贵？Netflix开源神器，token直降95%` |

### 3.3 标题硬指标（4 条，与 3 元素一一对应）

- ✅ **必须出现具体对象**：产品名、大厂名、价格、数字
- ✅ **必须有情绪或悬念**：震惊、讽刺、警惕、意外
- ✅ **必须表达判断或悬念**：核心观点、用户会问的问题
- ✅ **长度 15-32 字，硬上限 36 字**（手机一屏显示）

### 3.4 标题 4 大禁忌

| 禁忌 | 错误示例 | 原因 |
|------|---------|------|
| ❌ 空泛标题 | `AI 趋势分析`、`AI 工具推荐` | 没有具体对象，等于没说 |
| ❌ 虚假夸张 | `全球第一`、`最强`、`彻底颠覆` | 平台判为标题党，限流 |
| ❌ 太长（>36 字） | `关于2026年人工智能发展趋势的深度分析报告` | 手机显示不完整，打开率暴跌 |
| ❌ 太专业 | `大语言模型的Transformer架构优化` | 普通读者看不懂，定位不符 |

### 3.5 标题改 5 版（强制流程）

**每篇文章的主标题 + 关键小标题，必须按下面流程改 5 版**：

1. 把选题关键词 + 选题材料（来自 hot-topic-radar / aihot-topic-selector）交给 `viral-title` skill
2. skill 输出 N 个候选标题
3. 按 3.3 硬指标筛掉不合格的
4. 剩下的候选按 3.2 的 5 类模板归类，**每类至少挑 1 版作为主强化点**（数字冲击 / 反常识 / 个人体验 / 避坑提醒 / 结果导向）
5. 手动在 5 版基础上再做一次微调（具体词替换、节奏调整）
6. 最后用 3.6 自检清单逐条勾选

> 💡 **5 类必须都覆盖**的原因：单类模板会审美疲劳，混搭出 5 版才能对比出最戳人的那一条。

### 3.6 标题自检清单（发布前必查）

- [ ] 长度 15-32 字（硬上限 36 字）？
- [ ] 有具体对象（产品/大厂/数字/价格）？
- [ ] 有情绪词或反差感？
- [ ] 有判断或悬念（不是单纯描述）？
- [ ] 普通人一眼能看懂？
- [ ] 没出现"全球第一/最强/彻底"等夸张词？
- [ ] 你自己看到会想点进去？

## 📝 4. 文章结构模板 · 必须执行

### 4.1 爆款字数标准（2025-2026新榜数据）
> 📊 **推广类文章字数：2000-4000字**（推广文章需要充分铺垫，字数偏长）
>
> 微信算法权重优先级：完读率 > 字数长度，**推广文章需要充分建立信任，字数可以适当放宽**

| 文章类型 | 推荐字数 | 说明 |
|---------|---------|------|
| **热点观点类** | **800-1200字** | 快速产出，抢占流量，3-4分钟读完 |
| **AI工具评测/推广类** | **2000-4000字** | 充分铺垫痛点、建立信任、自然植入推广 |
| **深度分析类** | **1500-2500字** | 建立专业度，适合行业观察类 |
| **故事类/短评** | 600-1500字 | 叙事性内容，言简意赅 |

**字数分配参考**（以1000字爆款为例）：
| 章节 | 占比 | 参考字数 |
|------|------|---------|
| 开头引入（钩子） | 10% | ~100字 |
| 每个正文大节 | 25-30% | ~250-300字/节 × 3节 |
| 结尾总结+行动 | 10% | ~100字 |

### 4.2 Front-Matter 规范（必须严格格式）
```yaml
---
title: "本篇文章的标题（可与对外发布标题略有差异，确保内部可检索）"
date: YYYY-MM-DD HH:MM:SS
tags: ["公众号文章"]
categories: ["公众号文章"]
cover: 封面图链接（参考 6.3）
---
```

> ⚠️ **发布时间强制要求**：`date` 字段的时间必须与**当前系统时间完全一致**，精确到分钟。禁止使用过去的时间或虚构的时间，这会影响公众号的推送算法和读者体验。
>
> 📌 **YAML引号规则**：能不加就不加，特殊字符必须加
>
> **必须加引号的特殊字符**：`:` `,` `#` `{}` `[]` `'` `!`

### 4.3 标准开头模板（必须使用）

```markdown
大家好，我是程序员晚枫。

[直接抛出热点事件或读者最痛的疑问，1-2 句话]
```

**正确示例**：

### 4.4 结尾结构总览（与第 7 章结尾模板强绑定）

> ⚠️ **本节只做结构说明，**实际可套用的结尾模板以 7.4 为唯一标准**。任何与 7.4 冲突的写法以 7.4 为准。

一篇文章的结尾由 5 块组成（按顺序）：

1. **升华式总结**（7.1）：一句话戳透核心观点 + 留白反问 + 温度收尾
2. **相关阅读**（4.4.1）：从候选库挑 3 条最相关的，固定位置，**所有文章必加**
3. **互动提问**（7.2）：3 种爆款提问公式选其一
4. **返佣链接 + CTA**（7.6.3 / 7.6.2）：第 3 处完整 CTA
5. **固定结尾 Slogan**（7.3）：一字不差使用

> ❌ **绝对禁止**：「## 总结 + 要点列表」式的 AI 味总结。

## 💬 5. 语言风格与排版 · 必须执行

### 5.1 语言风格9大要求
| 要求 | 具体说明 | 正确示例 | 错误示例 |
|------|---------|---------|---------|
| **生活化比喻** | 用大众熟悉的事物做类比 | "这就像手机流量一样稀缺" | "这是稀缺资源" |
| **口语化表达** | 使用口语化连接词拉近距离 | "说实话"、"你知道吗"、"我跟你说" | "综上所述"、"由此可见" |
| **分段严格** | 每段不超过3行 | （手机上≤3行） | 大段文字墙 |
| **小标题体系** | 每300-500字一个二级小标题 | 用 `## 标题` 做章节分隔 | 连续多段无小标题 |
| **重点加粗** | 核心数据、金句必须加粗 | `**最高每月500元**` | 全文无加粗 |
| **善用类比** | 用生活化例子解释复杂概念 | "Token就像AI的粮票" | "Token是大语言模型的输入单位" |
| **适当情绪** | 合理使用感叹号表达真实情感 | "这个价格真的太香了！" | 全程无情绪波动 |
| **情绪价值** | 有共情、有温度、有共鸣 | "我知道你也被这个问题困扰很久了" | "本文将分析..." |
| **大众友好** | 避免过于新潮的网感词汇，也不用过于专业的术语 | 用大白话讲明白 | "家人们谁懂啊"、"绝绝子"、专业术语 |

### 5.2 排版6大铁律（缺一不可）
| 要求 | 具体标准 | 检查方法 |
|------|---------|---------|
| **短段落** | 每段不超过3行 | 手机预览：一段超过4行就不合格 |
| **多留白** | 章节之间用 `---` 分隔，关键转折处空一行 | 视觉上不拥挤 |
| **小标题层级** | 用 `##` 做章节标题，`###` 做子标题，不要跳级 | 没有孤立的 `###` 或 `####` |
| **重点加粗** | 核心观点、数据、金句用 `**加粗**` 标出 | 扫读时能快速抓住关键信息 |
| **列表替代长段** | 3个以上并列项用 `-` 列表 | 不用大段文字罗列 |
| **表格对比** | 有对比关系的内容用表格呈现 | 对比信息一目了然 |

### 5.3 Markdown加粗与标点位置规则（必须执行）

> ⚠️ **CommonMark侧翼判定规则**：`**` 粗体标记紧邻中文引号 `""` 时，加粗会失效。

| 情况 | 写法 | 结果 | 说明 |
|------|------|------|------|
| ❌ 错误 | `**"重点内容"**` | 加粗失效 | `**` 紧邻 `""`（Unicode标点），不构成粗体 |
| ✅ 正确 | `"**重点内容**"` | 正常加粗 | 引号包在外面，`**` 紧邻汉字 |
| ✅ 正确 | `**重点内容。**` | 正常加粗 | 直接不加引号 |

**记忆口诀：引号包外头，加粗包里头。**

## 📁 12. 文件命名与存放规范

### 12.1 文件名硬性规则

❌ 禁止文件名中包含以下字符——所有标点必须用 **英文中横线 `-`** 替代：

- URL 特殊字符：`%`、`?`、`#`、`&`、空格等
- 中文/英文标点：`，`、`。`、`、`、`：`、`；`、`（`、`）`、`——` 等

**错误示例**：`030-500万人周活、知识工作者暴增3倍，Codex不再只是程序员工具了.md`
**正确示例**：`030-500万人周活-知识工作者暴增3倍-Codex不再只是程序员工具了.md`

### 12.2 原因（重要）

Hexo 的 permalink 配置为 `:title/`，文件名直接作为 URL 路径：

- `%` 会被当作 URL 编码片段二次解析导致乱码 404
- 空格会被编码为 `%20` 导致链接断裂
- 中文/英文标点在跨平台传输、URL 编码、搜索引擎索引时容易出现兼容性问题，可能导致 Hexo 构建后输出**0 字节空文件**
- 中横线 `-` 是 URL 安全的「万能分隔符」，是**唯一推荐的分隔符**

### 12.3 文件存放规范

> 📌 **本指南只服务公众号文章**。口播稿创作规范在另一份独立文档中管理，不在本文档范围内。

| 内容类型 | 存放路径 | 命名格式 |
|---------|---------|---------|
| 公众号文章 | `/source/_posts/公众号文章/` | `序号-文章标题.md`（如 `001-xxx.md`） |
| 三大合集 | `/source/_posts/ai-hotpoints/`、`/source/_posts/ai-tools/`、`/source/_posts/ai-avoid-pitfalls/` | 按合集主题命名 |

**命名细节**：
- 序号统一 3 位补零（`001`、`012`、`123`），便于按时间排序
- 标题部分所有标点替换为英文中横线 `-`（见 12.1）
