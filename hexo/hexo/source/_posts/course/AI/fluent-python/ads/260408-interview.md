## 背题 vs 理解，是两种不同的人生

很多人备考 Python 面试的方式是这样的：

- 打开"Python 高频面试题 500 道"
- 每道题读一遍答案
- 背住关键词，比如"GIL 是全局解释器锁"、"装饰器本质是语法糖"……

然后面试官稍微追问一层——"GIL 在什么情况下会释放？多线程真的完全没用吗？"——就开始语塞。

这不是你的错，是路子走窄了。

Python 面试考的高频考点，几乎全部来自同一个地方：**《流畅的Python（第2版）》**。

不是我说的，是那些年年出题的面试官，都是从这本书里找灵感的。

## 那本书里到底藏着哪些面试题？

随便数数——

**数据模型 / 特殊方法**（第1讲）：
- `__repr__` 和 `__str__` 有什么区别？
- 如何让自定义类支持 `len()`、`[]`、`for` 循环？

**装饰器 / 闭包**（第6讲）：
- 装饰器的底层原理是什么？
- `functools.wraps` 有什么用，不写会怎样？

**生成器 / yield**（第7讲）：
- `yield` 和 `return` 的区别？
- 生成器和迭代器的关系？

**描述符**（第13讲）：
- `__get__`、`__set__`、`__delete__` 分别什么时候触发？
- `property` 的底层是怎么实现的？

**GIL / 多线程 / 多进程**（第17讲）：
- Python 的 GIL 是什么？有什么影响？
- 什么场景用多线程，什么场景用多进程？

**asyncio / 协程**（第18讲）：
- `async/await` 和多线程有什么本质区别？
- `asyncio.gather` 的作用？

这些，全在书里。全是高频。

## 读过书和没读过，差距有多大？

举个最典型的例子——**装饰器**。

没读过这本书的人，面试时大概率会这么回答：

> "装饰器就是在函数外面套一层函数，用 `@` 语法糖实现。"

读过这本书的人，答的是：

> "装饰器本质是一个可调用对象，接收函数作为参数，返回新函数。它是闭包的应用。需要注意的是，如果不用 `functools.wraps`，被装饰函数的 `__name__` 和 `__doc__` 会丢失，这在框架开发中可能导致问题。另外，带参数的装饰器需要三层嵌套……"

面试官的反应，你懂的。
