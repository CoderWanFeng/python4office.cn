## 工作流程梳理

### 第一步：列出日常工作
**提示词**：
```
你是一名工作效率专家。
我是一名 [你的职位]，日常工作包括：
请帮我列出这个职位的典型日常工作：
1. 按频率分类（每天/每周/每月）
2. 标注哪些是重复性工作
3. 标注哪些适合自动化
输出表格格式。
```

**示例输出（销售经理）**：
| 工作 | 频率 | 重复性 | 可自动化 |
|------|------|--------|----------|
| 整理销售数据 | 每天 | 高 | ✅ |
| 发送日报 | 每天 | 高 | ✅ |
| 客户跟进 | 每天 | 中 | 部分 |
| 周报撰写 | 每周 | 高 | ✅ |
| 销售会议 | 每周 | 中 | 部分 |
| 月度报表 | 每月 | 高 | ✅ |
| 客户合同 | 不定期 | 高 | ✅ |

### 第二步：绘制流程图
**提示词**：
```
你是一名流程设计师。
我有以下工作流程：[描述工作流程]
请帮我绘制流程图：
1. 用文字描述每个步骤
2. 标注决策点（是/否分支）
3. 标注输入和输出
4. 标注哪些步骤可以自动化
用 Mermaid 格式输出，方便我可视化。
```

### 第三步：识别自动化机会
**评估标准**：
| 标准 | 说明 | 权重 |
|------|------|------|
| 重复性 | 是否重复执行 | ⭐⭐⭐⭐⭐ |
| 规则明确 | 是否有明确规则 | ⭐⭐⭐⭐ |
| 数据量 | 处理数据量大小 | ⭐⭐⭐ |
| 错误率 | 人工操作错误率 | ⭐⭐⭐ |
| 时间消耗 | 单次耗时 | ⭐⭐⭐ |

**提示词**：
```
你是一名自动化专家。
我有以下工作流程：[描述工作流程]
请帮我分析：
1. 哪些步骤适合自动化（按优先级排序）
2. 每个步骤的自动化难度（低/中/高）
3. 预估自动化后的效率提升
4. 推荐使用的工具
输出表格格式。
```

## 工具选择指南

### 根据场景选工具
| 场景 | 推荐工具 | 理由 |
|------|----------|------|
| Excel 处理 | ChatExcel、Excel AI | 专业、简单 |
| 文档写作 | WPS AI、Notion AI | 中文好、集成 |
| PPT 生成 | Gamma、WPS AI | 设计感好 |
| 跨应用 | 集简云、HiFlow | 应用多、国内快 |
| 批量处理 | Python+AI | 灵活、强大 |

### 选择原则
1. **先简单后复杂**：能用现成工具，不自己开发
2. **先核心后边缘**：优先自动化最耗时的部分
3. **先试点后推广**：先在一个场景试点，成功后再推广

**提示词**：
```
你是一名自动化工具专家。
我有以下自动化需求：[描述需求]
预算：[免费/付费]
技术能力：[零基础/有点基础/熟练]
时间要求：[紧急/不急]
请推荐最适合的工具：
1. 推荐 2-3 个工具
2. 说明每个工具的优缺点
3. 给出选择建议
```

## 落地实施步骤

### 第一步：小范围试点
**选择试点场景的标准**：
- 高频重复
- 规则明确
- 影响范围小
- 容易衡量效果

### 第二步：配置和测试
**配置清单**：
- [ ] 工具账号注册
- [ ] 权限配置
- [ ] 流程配置
- [ ] 数据对接
- [ ] 异常处理

**测试清单**：
- [ ] 正常流程测试
- [ ] 异常情况测试
- [ ] 边界条件测试
- [ ] 性能测试

**提示词**（生成测试用例）：
```
你是一名测试工程师。
我有以下自动化流程：[描述流程]
请帮我生成测试用例：
1. 正常流程测试用例（3-5 个）
2. 异常情况测试用例（3-5 个）
3. 边界条件测试用例（2-3 个）
输出表格格式，包含测试步骤和预期结果。
```

### 第三步：上线和监控
**上线检查清单**：
- [ ] 所有测试通过
- [ ] 相关人员培训
- [ ] 异常处理流程
- [ ] 监控和日志
- [ ] 回滚方案

**监控指标**：
- 执行成功率
- 平均执行时间
- 异常次数
- 节省时间

### 第四步：优化和迭代
**优化方向**：
- 提升执行速度
- 降低错误率
- 扩展功能
- 用户体验

**提示词**：
```
你是一名流程优化专家。
我有以下自动化流程，已运行 1 个月：
执行情况：
- 总执行次数：500 次
- 成功次数：480 次
- 失败次数：20 次
- 平均执行时间：2 分钟
问题反馈：
1. 偶尔会超时
2. 某些特殊情况处理不了
请帮我分析优化点，给出改进建议。
```

## 完整案例：销售自动化

### 背景
- 公司：某 SaaS 公司
- 团队：10 人销售团队
- 痛点：每天花 1 小时整理销售数据、周报撰写耗时、客户跟进记录混乱

### 解决方案
**自动化流程 1：销售数据自动汇总**
```
定时触发（每天 18 点）→ 获取 CRM 销售数据 → 汇总到 Excel → 生成日报 → 发送钉钉通知
```

**自动化流程 2：周报自动生成**
```
定时触发（每周五 17 点）→ 获取本周销售数据 → AI 生成周报内容 → 发送到邮箱 → 人工审核后发送
```

**自动化流程 3：客户跟进提醒**
```
CRM 客户状态变更 → 判断是否需要跟进 → 创建飞书待办 → 到期发送提醒
```

### 实施效果
- 日报时间：1 小时 → 5 分钟
- 周报时间：2 小时 → 15 分钟
- 客户跟进遗漏：每周 5 次 → 0 次
