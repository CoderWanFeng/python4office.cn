## 什么是外部服务？

### 常见外部服务
**1. 查询类**：天气查询、快递查询、订单查询、库存查询
**2. 操作类**：创建订单、预约登记、发送通知、更新数据
**3. 第三方服务**：支付接口、短信发送、邮件发送、CRM 系统

## 扣子的插件系统

### 什么是插件？
**插件**：预先配置好的外部服务连接

**扣子官方插件**：
- 天气查询
- 新闻获取
- 图片生成
- 网页搜索
- 计算器

### 使用官方插件
**步骤**：
1. 进入"插件"页面
2. 浏览可用插件
3. 点击"添加"
4. 配置参数
5. 在工作流中调用

**示例：天气查询插件**
```
用户：北京天气怎么样？
    ↓
调用天气插件
    ↓
获取天气数据
    ↓
回复用户
```

## 自定义 API 调用

### 什么是 API？
**API**：应用程序接口，让不同系统可以通信

**示例**：
- 天气 API：获取天气数据
- 快递 API：查询快递状态
- 支付 API：处理支付

### 在扣子中调用 API
**步骤**：
1. **准备工作**：获取 API 地址，获取 API 密钥（如果需要），了解 API 参数和返回格式
2. **创建 HTTP 请求**：在工作流中添加"HTTP 请求"节点，填写 API 地址，配置请求方法（GET/POST），配置请求头和参数
3. **处理返回数据**：解析 API 返回的 JSON，提取需要的字段，格式化回复
4. **测试和调试**：测试 API 调用，检查返回数据，优化回复格式

**示例：查询订单状态**

**API 信息**：
```
地址：https://api.example.com/order/status
方法：GET
参数：order_id（订单号）
返回：{status: "已发货", time: "2026-03-01"}
```

**工作流配置**：
```
用户：帮我查一下订单
    ↓
询问订单号
    ↓
用户：123456
    ↓
HTTP 请求：GET /order/status?order_id=123456
    ↓
解析返回：{status: "已发货", time: "2026-03-01"}
    ↓
回复：您的订单已发货，发货时间：2026-03-01
```

**提示词**：
```
你是一名 API 集成专家。
我要调用以下 API：
- 地址：[API 地址]
- 方法：[GET/POST]
- 参数：[参数列表]
- 返回：[返回格式]
请帮我配置扣子工作流：
1. HTTP 请求配置
2. 参数映射
3. 返回数据解析
4. 回复格式化
输出详细步骤。
```

## 常见应用场景

### 场景 1：订单查询
**需求**：用户查询订单状态

**流程**：
```
用户：查订单
    ↓
询问订单号/手机号
    ↓
调用订单 API
    ↓
返回订单状态
```

**提示词**：
```
你是一名系统集成专家。
我要实现订单查询功能。
现有系统：[描述系统]
API 信息：[API 文档]
请帮我设计对接方案：
1. 数据流程
2. API 调用配置
3. 错误处理
4. 回复模板
输出详细方案。
```

### 场景 2：预约登记
**需求**：用户预约服务

**流程**：
```
用户：我想预约
    ↓
收集信息（姓名、时间、联系方式）
    ↓
调用预约 API 创建预约
    ↓
确认预约成功
    ↓
发送提醒
```

### 场景 3：库存查询
**需求**：用户查询产品库存

**流程**：
```
用户：XX 产品有货吗？
    ↓
调用库存 API
    ↓
返回库存数量
    ↓
如果有货，引导购买
    ↓
如果没货，推荐其他产品
```

### 场景 4：数据录入
**需求**：用户提交信息到系统

**流程**：
```
用户：我要提交 XX
    ↓
收集必要信息
    ↓
调用录入 API
    ↓
确认提交成功
    ↓
发送确认通知
```

## 错误处理

### 常见错误
**1. API 调用失败**：网络问题、API 服务异常、参数错误
**2. 数据格式错误**：返回数据解析失败、缺少必要字段
**3. 权限问题**：API 密钥无效、权限不足

### 错误处理策略
**策略 1：友好提示**
```
抱歉，系统暂时繁忙，请稍后再试。
```

**策略 2：转人工**
```
这个问题我需要人工协助，请稍等，我帮您转接客服。
```

**策略 3：重试机制**
```
自动重试 1-2 次
如果还是失败，提示用户
```

**提示词**：
```
你是一名系统架构师。
我要调用外部 API，需要处理可能的错误。
可能的错误：
1. 网络超时
2. API 返回错误
3. 数据格式错误
请帮我设计错误处理方案：
1. 错误检测
2. 错误分类
3. 应对策略
4. 用户提示
输出详细方案。
```
