## 多轮对话设计

### 什么是多轮对话？
**单轮对话**：
```
用户：你们的产品多少钱？
机器人：[回答价格]
```

**多轮对话**：
```
用户：我想了解一下你们的产品
    ↓
机器人：好的，请问您想了解哪方面？1.功能 2.价格 3.服务
    ↓
用户：价格
    ↓
机器人：[回答价格] 请问您是个人使用还是企业使用？
    ↓
用户：企业
    ↓
机器人：[推荐企业版方案]
```

### 多轮对话的应用场景
**场景 1：需求挖掘**
```
用户：我想买个产品
    ↓
机器人：请问您的预算是多少？
    ↓
用户：1000 左右
    ↓
机器人：这个价位有几款可选，您主要用来做什么？
    ↓
用户：主要是办公用
    ↓
机器人：[推荐办公款]
```

**场景 2：预约登记**
```
用户：我想预约演示
    ↓
机器人：好的，请问您贵姓？
    ↓
用户：我姓王
    ↓
机器人：王先生您好，请问什么时间方便？
    ↓
用户：明天下午
    ↓
机器人：好的，明天下午几点？
    ↓
用户：2 点
    ↓
机器人：好的，已为您预约明天下午 2 点。请问您的电话是？
```

**场景 3：问题诊断**
```
用户：我的设备出问题了
    ↓
机器人：请问是什么问题？1.无法开机 2.功能异常 3.其他
    ↓
用户：无法开机
    ↓
机器人：请问指示灯亮吗？
    ↓
用户：不亮
    ↓
机器人：请问电源插好了吗？
    ↓
用户：插好了
    ↓
机器人：可能是电源问题，建议您...
```

### 在扣子中配置多轮对话
**步骤**：
1. **开启上下文记忆**：进入"设置"，开启"多轮对话"，设置上下文轮数（建议 3-5 轮）
2. **配置工作流**：点击"工作流"，创建工作流，添加多个节点，配置每个节点的回复和条件
3. **设置变量**：定义变量（如用户姓名、需求等），在对话中收集变量，在后续回复中使用

**提示词**：
```
你是一名专业的对话设计师。
我要做一个 [机器人类型]，需要多轮对话。
场景：[描述场景]
需要收集的信息：[列出信息]
请帮我设计多轮对话流程：
1. 对话步骤
2. 每步的回复
3. 条件分支
4. 变量定义
输出详细的流程图和配置说明。
```

## 上下文理解

### 什么是上下文理解？
**没有上下文**：
```
用户：北京天气怎么样？
机器人：[回答北京天气]

用户：上海呢？
机器人：[不知道"上海呢"是什么意思]
```

**有上下文**：
```
用户：北京天气怎么样？
机器人：[回答北京天气]

用户：上海呢？
机器人：[理解是问上海天气，回答上海天气]
```

### 配置上下文
**在扣子中**：
1. 进入"设置"
2. 开启"上下文记忆"
3. 设置记忆轮数（3-5 轮）

**注意**：
- 轮数越多，消耗越大
- 建议 3-5 轮够用
- 敏感信息不要记忆

### 上下文应用技巧
**技巧 1：代词理解**
```
用户：你们的产品有什么功能？
机器人：[回答功能]

用户：价格呢？
机器人：[理解"价格呢"是问产品价格]
```

**技巧 2：省略理解**
```
用户：我想买基础版
机器人：好的，基础版 99 元/月。请问怎么称呼您？
用户：小王
机器人：[理解是用户姓名]
```

**技巧 3：对比理解**
```
用户：基础版和专业版有什么区别？
机器人：[回答区别]

用户：那企业版呢？
机器人：[理解是对比企业版]
```

## 个性化回复

### 为什么要个性化？
**好处**：
- 让用户感觉被重视
- 提高转化率
- 增强用户粘性

### 个性化方法
**方法 1：称呼个性化**
```
收集用户姓名后：
- "王先生，您好"
- "小美，很高兴为您服务"
```

**方法 2：历史行为个性化**
```
如果是老客户：
- "欢迎回来，上次您购买了..."
- "根据您的使用记录，推荐..."
```

**方法 3：偏好个性化**
```
根据用户偏好：
- "知道您喜欢简洁，我长话短说"
- "知道您是专业人士，我详细说"
```

### 在扣子中实现个性化
**步骤**：
1. **收集用户信息**：在工作流中设置变量，收集姓名、需求等
2. **存储用户信息**：用数据库或表格存储，或用扣子的用户记忆功能
3. **在回复中使用**：调用变量，个性化回复

**提示词**：
```
你是一名专业的对话设计师。
我要实现个性化回复。
需要收集的信息：姓名、公司、需求、偏好
存储方式：[数据库/表格/记忆]
请帮我设计：
1. 信息收集流程
2. 存储结构
3. 个性化回复模板
4. 变量调用方法
输出详细方案。
```

## 高级技巧

### 技巧 1：情绪识别
**场景**：识别用户情绪，调整回复

**配置**：
```
如果用户情绪激动：
- 语气更温和
- 快速转人工

如果用户情绪平静：
- 正常回复
```

**提示词**：
```
你是一名客服专家。
请帮我设计情绪识别和应对策略：
1. 如何识别用户情绪（关键词）
2. 不同情绪的应对策略
3. 回复模板
输出详细方案。
```

### 技巧 2：意图识别
**场景**：识别用户真实意图

**示例**：
```
用户：太贵了

可能意图：
1. 真的觉得贵 → 介绍价值
2. 想砍价 → 给优惠
3. 随便说说 → 继续沟通
```

**配置**：
- 在工作流中添加意图判断
- 根据不同意图走不同分支

### 技巧 3：主动营销
**场景**：在对话中引导购买

**示例**：
```
用户：这个功能不错
机器人：是的，这个功能很受欢迎。现在购买还有优惠，要了解一下吗？
```

**注意**：
- 不要过度营销
- 在合适时机引导
- 给用户选择权
