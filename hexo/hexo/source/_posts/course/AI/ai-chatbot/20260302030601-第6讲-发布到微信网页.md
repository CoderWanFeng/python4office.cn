## 发布渠道总览

### 扣子支持的渠道
| 渠道 | 难度 | 适合场景 | 推荐指数 |
|------|------|----------|----------|
| 抖音 | ⭐ | 抖音私信/评论 | ⭐⭐⭐⭐⭐ |
| 飞书 | ⭐ | 企业内部/客户 | ⭐⭐⭐⭐ |
| 网页嵌入 | ⭐⭐ | 官网/落地页 | ⭐⭐⭐⭐ |
| 微信 | ⭐⭐⭐ | 公众号/个人号 | ⭐⭐⭐⭐⭐ |

### 微信发布的难点
**问题**：扣子不直接支持微信

**解决方案**：
1. 用腾讯云 HiFlow 中转
2. 用企业微信
3. 用第三方工具

## 发布到抖音

### 方法 1：抖音私信机器人
**步骤**：
1. 在扣子点击"发布"
2. 选择"抖音"
3. 关联抖音账号
4. 设置为私信机器人
5. 配置自动回复

**适合场景**：抖音获客、粉丝互动、私信自动回复

### 方法 2：抖音评论区机器人
**步骤**：
1. 在扣子点击"发布"
2. 选择"抖音"
3. 设置为评论区机器人
4. 配置触发关键词

**适合场景**：自动回复评论、引导私信、提高互动率

---

## 发布到飞书

### 步骤
1. 在扣子点击"发布"
2. 选择"飞书"
3. 关联飞书账号
4. 创建飞书机器人
5. 邀请到群聊

### 适合场景
- 企业内部问答
- 客户服务群
- 培训/考试机器人

---

## 发布到微信

### 方法 1：用腾讯云 HiFlow 中转（推荐）
**原理**：微信公众号 → 腾讯云 HiFlow → 扣子机器人

**步骤**：
1. **注册腾讯云 HiFlow**：打开 hiflow.tencent.com，注册账号
2. **创建应用**：创建新应用，选择"微信公众号"
3. **配置触发器**：选择"微信公众号 - 收到消息"，授权公众号
4. **配置动作**：选择"HTTP 请求"，填写扣子 API 地址，配置参数映射
5. **配置回复**：选择"微信公众号 - 发送消息"，配置回复内容
6. **测试和启用**

**提示词**：
```
你是一名自动化专家。
我要用腾讯云 HiFlow 连接微信公众号和扣子机器人。
请帮我设计配置流程：
1. 触发器配置
2. HTTP 请求配置（扣子 API）
3. 回复配置
4. 参数映射
输出详细步骤。
```

### 方法 2：用企业微信
**步骤**：
1. **注册企业微信**：打开 work.weixin.qq.com，注册企业
2. **创建应用**：创建自建应用，获取 AgentId 和 Secret
3. **配置扣子**：在扣子选择"企业微信"，填写 AgentId 和 Secret
4. **配置回调**：填写扣子提供的回调地址，验证配置
5. **测试和启用**

**适合场景**：企业内部使用、客户服务、销售跟进

### 方法 3：用第三方工具
**工具推荐**：微伴助手、尘锋 SCRM、探马 SCRM

**步骤**：
1. 注册第三方工具
2. 配置微信接入
3. 配置扣子 API
4. 测试和启用

**优点**：配置简单、功能丰富
**缺点**：需要付费、依赖第三方

## 嵌入到网站

### 方法 1：扣子官方嵌入
**步骤**：
1. 在扣子点击"发布"
2. 选择"网页嵌入"
3. 复制嵌入代码
4. 粘贴到网站 HTML 中

**代码示例**：
```html
<!-- 扣子机器人嵌入代码 -->
<script src="https://www.coze.cn/static/bot.js"></script>
<div id="coze-bot" data-bot-id="你的 Bot ID"></div>
```

**适合场景**：官网客服、落地页咨询、产品页问答

### 方法 2：自定义嵌入
**步骤**：
1. 获取扣子 API
2. 自己写前端界面
3. 调用扣子 API
4. 展示回复

**优点**：完全自定义、可以定制样式
**缺点**：需要开发能力

## 多渠道发布策略

### 根据业务选择
**电商卖家**：抖音（主要）、微信（私域）、网站（官网）
**企业服务商**：飞书（内部）、微信（客户）、网站（官网）
**内容创作者**：抖音（粉丝互动）、微信（私域运营）、网站（内容展示）

### 统一知识库
**原则**：
- 所有渠道用同一知识库
- 保证回答一致性
- 集中管理，分散发布

**方法**：
1. 在扣子维护知识库
2. 多渠道发布同一个机器人
3. 或同步多个机器人知识库

## 本讲作业

1. **选择一个发布渠道**：根据你的业务，选择主要发布渠道（抖音/飞书/微信/网站）
2. **配置发布**：按照教程配置，测试是否正常，记录遇到的问题
3. **多渠道发布（可选）**：选择 2-3 个渠道，配置发布，测试一致性
