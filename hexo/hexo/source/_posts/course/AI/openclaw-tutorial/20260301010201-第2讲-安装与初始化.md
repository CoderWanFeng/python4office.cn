整个过程大约需要**5分钟**，跟着做就能成功。

## 环境要求

### 硬件要求
| 配置项 | 最低要求 | 推荐配置 |
|--------|---------|---------|
| CPU | 1核 | 2核+ |
| 内存 | 2GB | 4GB+ |
| 硬盘 | 1GB可用空间 | 5GB+ |
| 网络 | 能访问国际互联网 | 稳定连接 |

### 系统支持
- ✅ macOS 11+ (Intel/Apple Silicon)
- ✅ Linux (Ubuntu 20.04+, CentOS 8+, Debian 11+)
- ✅ Windows 10/11 (通过WSL2)
- ✅ Docker容器环境

### 软件依赖
- **Node.js 22+** （必须）
- **npm** 或 **pnpm** （包管理器）

---

## 第一步：安装Node.js

### macOS用户
```bash
# 使用Homebrew安装
brew install node@22

# 验证安装
node --version  # 应显示 v22.x.x
npm --version   # 应显示 10.x.x
```

### Linux用户（Ubuntu/Debian）
```bash
# 安装NodeSource仓库
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -

# 安装Node.js
sudo apt-get install -y nodejs

# 验证
node --version
```

### Linux用户（CentOS/RHEL）
```bash
# 安装NodeSource仓库
curl -fsSL https://rpm.nodesource.com/setup_22.x | sudo bash -

# 安装Node.js
sudo yum install -y nodejs

# 验证
node --version
```

### Windows用户（WSL2）
在WSL2 Ubuntu中执行Linux安装命令即可。

**常见问题**：如果已安装旧版本Node.js，建议先卸载再安装新版本。

---

## 第二步：安装OpenClaw

官方提供了一键安装脚本，支持macOS和Linux：

### macOS/Linux一键安装
```bash
# 官方推荐安装方式
curl -fsSL https://openclaw.ai/install.sh | bash
```

安装过程会自动：
- 下载最新版OpenClaw
- 安装到系统PATH
- 创建必要的目录结构

安装完成后，验证：
```bash
openclaw --version
# 输出类似：openclaw version 0.25.0
```

### Windows PowerShell安装
```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

### npm手动安装（备选）
如果一键脚本遇到问题：
```bash
npm install -g openclaw@latest
```

---

## 第三步：运行初始化向导

OpenClaw提供了交互式初始化向导，帮你完成基础配置：

```bash
openclaw onboard --install-daemon
```

向导会引导你完成以下步骤：

### 1. 选择安装模式
```
? Select installation mode:
  ▸ Standard (recommended)  - 标准模式，适合大多数用户
    Minimal                 - 最小化安装，仅核心功能
    Custom                  - 自定义安装
```

**建议选择Standard**，包含完整功能。

### 2. 配置API密钥
向导会询问你是否要配置AI模型提供商：

```
? Configure AI provider now? (Y/n)
```

选择`Y`，然后输入你的API密钥：

```
? Select provider:
  ▸ Anthropic (Claude)
    OpenAI (GPT)
    Google (Gemini)
    Other

? Enter your API key: sk-ant-api03-xxxxxxxx
```

**没有API密钥？** 可以先跳过，后面再配置。

### 3. 安装系统服务（daemon）
```
? Install as system service? (Y/n)
```

选择`Y`，这样OpenClaw会在开机时自动启动。

### 4. 完成初始化
看到以下提示说明初始化成功：
```
✓ OpenClaw initialized successfully!
✓ Gateway service installed
✓ Config saved to ~/.openclaw/openclaw.json

Next steps:
  1. Run 'openclaw gateway status' to check status
  2. Run 'openclaw dashboard' to open Control UI
```

---

## 第四步：检查Gateway状态

初始化完成后，检查Gateway是否正常运行：

```bash
# 查看服务状态
openclaw gateway status
```

正常输出示例：
```
Gateway Status: running
PID: 12345
Port: 18789
Uptime: 0h 2m 15s
Channels: 0 connected
Sessions: 0 active
```

如果显示`stopped`，手动启动：
```bash
# 前台启动（调试用）
openclaw gateway --port 18789

# 或使用服务管理
openclaw daemon start
```

---

## 第五步：打开Control UI

OpenClaw自带Web管理界面，无需额外配置通道即可使用：

```bash
# 自动打开浏览器
openclaw dashboard

# 或手动访问
# http://127.0.0.1:18789/
```

首次打开会看到：
- **Chat界面**：直接与AI对话
- **Sessions**：会话管理
- **Config**：配置查看
- **Status**：运行状态

### 测试对话
在Chat界面输入：
```
你好，请介绍一下你自己
```

如果能收到AI回复，说明**基础部署成功**！🎉

## 目录结构说明

安装完成后，OpenClaw会在用户目录创建以下结构：

```
~/.openclaw/
├── openclaw.json          # 主配置文件
├── state/                 # 运行时状态
│   ├── sessions/          # 会话数据
│   ├── memory/            # 记忆文件
│   └── logs/              # 日志文件
├── skills/                # 技能插件目录
└── cache/                 # 缓存数据
```

**重要文件**：
- `~/.openclaw/openclaw.json` - 所有配置都在这里
- `~/.openclaw/state/logs/` - 排查问题时先看日志

## 常见问题解决

### Q1: 安装时报权限错误
```bash
# macOS/Linux
sudo chown -R $(whoami) /usr/local/lib/node_modules

# 或使用npx免安装
npx openclaw@latest
```

### Q2: Gateway启动失败，端口被占用
```bash
# 查找占用18789端口的进程
lsof -i :18789

# 杀死进程或更换端口
openclaw gateway --port 18888
```

### Q3: 提示Node版本过低
```bash
# 升级Node.js到22+
nvm install 22
nvm use 22

# 或重新安装
brew upgrade node
```

### Q4: Control UI打不开
```bash
# 检查Gateway是否真的在运行
openclaw gateway status

# 查看日志找错误
tail -f ~/.openclaw/state/logs/gateway.log

# 尝试重启
openclaw daemon restart
```
