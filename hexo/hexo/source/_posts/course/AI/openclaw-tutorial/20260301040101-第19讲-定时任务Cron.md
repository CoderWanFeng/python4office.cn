## 什么是Cron？

**Cron**是类Unix系统中用于执行定时任务的守护进程。OpenClaw内置了Cron功能，可以：
- 定时发送消息
- 定期执行检查
- 自动生成报告
- 触发自动化工作流

### Cron表达式格式
```
* * * * *
│ │ │ │ │
│ │ │ │ └─── 星期 (0-7, 0和7都是周日)
│ │ │ └───── 月份 (1-12)
│ │ └─────── 日期 (1-31)
│ └───────── 小时 (0-23)
└─────────── 分钟 (0-59)
```

### 常用示例
| 表达式 | 含义 |
|--------|------|
| `0 9 * * *` | 每天上午9点 |
| `0 */6 * * *` | 每6小时 |
| `0 0 * * 1` | 每周一凌晨 |
| `0 0 1 * *` | 每月1号 |
| `*/5 * * * *` | 每5分钟 |

## 配置定时任务

### 方式1：配置文件
```json
{
  "cron": {
    "enabled": true,
    "jobs": [
      {
        "name": "daily_report",
        "schedule": "0 9 * * *",
        "task": "generate_daily_report",
        "channel": "telegram",
        "target": "123456789"
      },
      {
        "name": "health_check",
        "schedule": "*/30 * * * *",
        "task": "check_system_health",
        "action": "log"
      }
    ]
  }
}
```

### 方式2：命令行
```bash
# 添加定时任务
openclaw cron add \
  --name backup_reminder \
  --schedule "0 18 * * 5" \
  --message "记得备份本周数据！"

# 列出所有任务
openclaw cron list

# 删除任务
openclaw cron remove backup_reminder
```

## 实战案例

### 案例1：每日晨报
每天早上9点自动发送天气和新闻：
```json
{
  "cron": {
    "jobs": [
      {
        "name": "morning_briefing",
        "schedule": "0 9 * * *",
        "timezone": "Asia/Shanghai",
        "steps": [
          {
            "tool": "web_search",
            "query": "今日新闻头条"
          },
          {
            "tool": "web_search", 
            "query": "北京今日天气"
          },
          {
            "action": "send_message",
            "template": "📰 今日简报\n\n{news}\n\n🌤️ 天气：{weather}"
          }
        ]
      }
    ]
  }
}
```

### 案例2：服务器健康检查
每30分钟检查一次系统状态：
```json
{
  "cron": {
    "jobs": [
      {
        "name": "health_check",
        "schedule": "*/30 * * * *",
        "script": "health_check.sh",
        "onError": {
          "action": "send_alert",
          "channel": "telegram",
          "target": "admin_user_id"
        }
      }
    ]
  }
}
```

### 案例3：周报生成
每周五下午6点生成本周工作总结：
```bash
openclaw cron add \
  --name weekly_report \
  --schedule "0 18 * * 5" \
  --command "openclaw task generate_weekly_report --output ~/reports/weekly_$(date +%Y%m%d).md"
```

## Heartbeat机制

除了Cron，OpenClaw还支持**Heartbeat（心跳）**，适合需要响应式触发的场景。

### Cron vs Heartbeat
| 特性 | Cron | Heartbeat |
|------|------|-----------|
| 触发方式 | 时间驱动 | 事件驱动 |
| 适用场景 | 定期检查 | 等待用户输入 |
| 资源占用 | 固定间隔运行 | 按需唤醒 |
| 灵活性 | 低 | 高 |

### Heartbeat配置
```json
{
  "heartbeat": {
    "enabled": true,
    "prompt": "Read HEARTBEAT.md and check for pending tasks",
    "interval": 300
  }
}
```

创建 `~/.openclaw/HEARTBEAT.md`：
```markdown
# Heartbeat Tasks

## Daily Checks
- [ ] Check email for urgent messages
- [ ] Review calendar for upcoming events  
- [ ] Monitor server metrics

## On-Demand
- Process any pending approvals
- Respond to queued messages
```

## 监控与告警

### 任务执行日志
```bash
# 查看Cron日志
tail -f ~/.openclaw/state/logs/cron.log

# 输出示例：
[2024-01-15 09:00:00] INFO: Job 'morning_briefing' started
[2024-01-15 09:00:05] INFO: Task completed successfully
[2024-01-15 09:00:05] INFO: Message sent to user_001
```

### 失败告警
```json
{
  "cron": {
    "alerting": {
      "onFailure": true,
      "webhook": "https://your-webhook.com/alerts",
      "channels": ["telegram"],
      "targets": ["admin_user_id"]
    }
  }
}
```
