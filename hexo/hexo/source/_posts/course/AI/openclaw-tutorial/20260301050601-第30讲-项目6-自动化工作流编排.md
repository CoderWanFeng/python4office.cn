## 什么是Lobster？

**Lobster**是OpenClaw的工作流引擎，可以：
- 编排多步骤业务流程
- 支持审批门控
- 并行执行子任务
- 可视化流程状态

### 适用场景
| 场景 | 示例 |
|------|------|
| 内容发布 | 撰写→审核→发布→推广 |
| 数据处理 | 采集→清洗→分析→报表 |
| 客户服务 | 受理→分派→处理→回访 |
| 开发部署 | 代码审查→测试→部署→监控 |

## 工作流定义

### YAML格式
```yaml
# deploy_workflow.yaml
name: 应用部署流程
version: 1.0

triggers:
  - type: webhook
    endpoint: /deploy
    
steps:
  - id: code_review
    name: 代码审查
    agent: code_reviewer
    task: 审查提交的代码变更
    approval:
      required: true
      approvers: ["tech_lead"]
      timeout: 3600
    
  - id: run_tests
    name: 运行测试
    depends_on: [code_review]
    agent: test_runner
    task: 执行单元测试和集成测试
    
  - id: build_image
    name: 构建镜像
    depends_on: [run_tests]
    action: exec
    command: docker build -t myapp:${version} .
    
  - id: deploy_staging
    name: 部署到预发布环境
    depends_on: [build_image]
    action: exec
    command: kubectl apply -f k8s/staging/
    
  - id: smoke_test
    name: 冒烟测试
    depends_on: [deploy_staging]
    agent: tester
    task: 验证核心功能正常
    
  - id: deploy_production
    name: 部署到生产环境
    depends_on: [smoke_test]
    approval:
      required: true
      approvers: ["ops_manager", "product_owner"]
      timeout: 7200
    action: exec
    command: kubectl apply -f k8s/production/
    
notifications:
  on_success:
    channel: slack
    message: "✅ 部署成功: ${workflow.name}"
  on_failure:
    channel: slack
    message: "❌ 部署失败: ${workflow.name}\n错误: ${error.message}"
```

## 实战案例：内容发布流程

### 场景
新媒体团队的内容从创作到发布的完整流程。

### 工作流程
```
编辑撰稿 → AI初审 → 主编审核 → 排版设计 → 法务合规检查 → 定时发布 → 数据追踪
    ↑           ↓           ↓           ↓              ↓            ↓         ↓
   退回修改   通过/驳回   通过/驳回   通过/驳回      通过/驳回     发布完成   生成报告
```

### OpenClaw配置
```json
{
  "lobster": {
    "enabled": true,
    "workflows_dir": "./workflows",
    "default_timeout": 86400
  }
}
```

### 启动工作流
```bash
# 手动触发
openclaw lobster run content_publish \
  --input '{"title": "新品发布", "author": "张三"}'

# 查看状态
openclaw lobster status content_publish_001

# 批准等待中的步骤
openclaw lobster approve content_publish_001 step_editor_review
```

## 多Agent协作

### 复杂任务分解
```yaml
steps:
  - id: research
    name: 资料收集
    parallel:
      - agent: web_searcher
        task: 搜索行业资讯
      - agent: data_analyst
        task: 获取市场数据
      - agent: competitor_analyst
        task: 分析竞品动态
    merge: concatenate
    
  - id: analysis
    name: 综合分析
    depends_on: [research]
    agent: strategist
    task: 基于收集的资料撰写分析报告
    
  - id: review
    name: 质量审核
    depends_on: [analysis]
    agent: reviewer
    task: 审核报告质量
    approval:
      required: true
```
