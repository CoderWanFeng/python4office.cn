## 什么是Webhook？

**Webhook**是一种HTTP回调机制。当某个事件发生时，服务器主动向指定的URL发送HTTP请求。

### 工作流程
```
外部系统 ──事件触发──▶ 发送HTTP POST ──▶ OpenClaw Gateway
                                      ↓
                                  处理请求
                                      ↓
                                  执行相应操作
```

### 典型应用场景
| 场景 | 触发源 | 动作 |
|------|--------|------|
| Git代码推送 | GitHub/GitLab | 自动部署、通知审查 |
| 订单支付完成 | 支付平台 | 发送确认消息 |
| 服务器告警 | 监控系统 | 紧急通知管理员 |
| 表单提交 | 网站 | 自动回复、数据录入 |

## 配置Webhook接收

### 基础配置
```json
{
  "webhooks": {
    "enabled": true,
    "port": 18790,
    "path": "/webhook",
    "secret": "your-webhook-secret",
    
    "handlers": [
      {
        "name": "github_push",
        "source": "github",
        "event": "push",
        "action": "notify_deployment"
      },
      {
        "name": "alertmanager",
        "source": "prometheus",
        "action": "send_alert"
      }
    ]
  }
}
```

### 创建Webhook处理器
在 `~/.openclaw/hooks/` 目录创建处理脚本：

**github_push.sh**：
```bash
#!/bin/bash
# 解析GitHub webhook payload
REPO=$(echo $1 | jq -r '.repository.name')
BRANCH=$(echo $1 | jq -r '.ref' | sed 's/refs\/heads\///')
AUTHOR=$(echo $1 | jq -r '.pusher.name')

# 发送通知
openclaw message send \
  --channel telegram \
  --target "$ADMIN_ID" \
  --message "📦 $REPO 有新推送\n分支: $BRANCH\n作者: $AUTHOR"

# 如果是main分支，触发部署
if [ "$BRANCH" = "main" ]; then
  ./deploy.sh
fi
```

## 实战案例

### 案例1：GitHub自动通知
当代码推送到GitHub时，自动发送Telegram通知：

**GitHub配置**：
1. 进入仓库 Settings → Webhooks
2. Add webhook
3. Payload URL: `http://your-server:18790/webhook/github`
4. Content type: `application/json`
5. Secret: 你的密钥
6. 选择事件：Just the push event

**OpenClaw配置**：
```json
{
  "webhooks": {
    "handlers": [
      {
        "name": "github_webhook",
        "endpoint": "/webhook/github",
        "verifySignature": true,
        "script": "handle_github.sh"
      }
    ]
  }
}
```

### 案例2：服务器告警转发
将Prometheus Alertmanager的告警转发到微信：

```json
{
  "webhooks": {
    "handlers": [
      {
        "name": "prometheus_alerts",
        "endpoint": "/webhook/alert",
        "transform": {
          "template": "🚨 **{{status}}**\n\n{{alerts}}",
          "channel": "wechat",
          "target": "alert_group"
        }
      }
    ]
  }
}
```

### 案例3：表单提交自动回复
网站联系表单提交后，自动发送确认邮件和内部通知：

```javascript
// webhook_handler.js
exports.handler = async (payload) => {
  const { name, email, message } = payload;
  
  // 发送确认给用户
  await openclaw.message.send({
    channel: 'email',
    to: email,
    subject: '感谢您的留言',
    body: `Hi ${name}, 我们已收到您的消息，会尽快回复。`
  });
  
  // 通知内部团队
  await openclaw.message.send({
    channel: 'slack',
    target: '#leads',
    text: `新线索: ${name} (${email})\n内容: ${message}`
  });
};
```

## 安全验证

### Signature验证
防止伪造请求：
```json
{
  "webhooks": {
    "handlers": [
      {
        "name": "secure_endpoint",
        "verifySignature": true,
        "signatureHeader": "X-Hub-Signature-256",
        "signatureSecret": "${WEBHOOK_SECRET}"
      }
    ]
  }
}
```

### IP白名单
限制请求来源：
```json
{
  "webhooks": {
    "allowedIPs": [
      "192.30.252.0/22",
      "140.82.112.0/20"
    ]
  }
}
```

## 调试Webhook

### 本地测试工具
```bash
# 使用curl模拟webhook
curl -X POST http://localhost:18790/webhook/github \
  -H "Content-Type: application/json" \
  -H "X-GitHub-Event: push" \
  -d @test_payload.json

# 或使用ngrok暴露本地服务
ngrok http 18790
```

### 查看日志
```bash
tail -f ~/.openclaw/state/logs/webhook.log
```
