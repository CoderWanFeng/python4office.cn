## 什么是通道（Channel）？

在OpenClaw中，**通道**是指消息进出的管道。通过通道，你可以：
- 从微信/Telegram/WhatsApp发送消息给AI
- 让AI回复到这些平台

### 支持的通道类型

| 类型 | 代表平台 | 接入方式 |
|------|---------|---------|
| **即时通讯** | Telegram/WhatsApp/Signal | Bot API或官方库 |
| **社交软件** | Discord/Slack | Bot Token |
| **国内平台** | 微信/钉钉/飞书 | 插件或Webhook |
| **传统协议** | IRC/Matrix | 标准协议 |
| **其他** | iMessage/SMS | 特殊方案 |

---

## Pairing机制详解

**Pairing（配对）**是OpenClaw建立通道连接的核心机制。

### 配对流程

```
┌─────────┐     1.发起配对      ┌──────────┐
│  用户    │ ─────────────────▶ │ Gateway  │
│ (手机)   │                    │ (服务器)  │
└─────────┘                    └──────────┘
     │                              │
     │  2.扫描二维码/输入验证码       │
     │ ◀────────────────────────────│
     │                              │
     │  3.确认授权                   │
     │ ────────────────────────────▶│
     │                              │
     │◀──────── 4.配对成功 ─────────│
```

### 配对方式对比

| 方式 | 适用平台 | 安全性 | 便捷性 |
|------|---------|--------|--------|
| **QR码扫描** | WhatsApp/微信 | ⭐⭐⭐ | ⭐⭐⭐ |
| **Token输入** | Telegram/Discord | ⭐⭐⭐ | ⭐⭐⭐ |
| **OAuth授权** | Slack/飞书 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |
| **手机号验证** | Signal/iMessage | ⭐⭐⭐⭐ | ⭐⭐ |

## 安全认证体系

### 三层防护

```
┌─────────────────────────────────────┐
│  Layer 3: Application Level         │
│  - allowFrom白名单                  │
│  - 群组@触发规则                     │
├─────────────────────────────────────┤
│  Layer 2: Channel Level             │
│  - Bot Token验证                    │
│  - Session加密                      │
├─────────────────────────────────────┤
│  Layer 1: Gateway Level             │
│  - Auth Token                       │
│  - TLS/HTTPS                        │
└─────────────────────────────────────┘
```

### 1. Gateway层认证

所有请求必须携带有效的Auth Token：

```bash
# 请求头中包含Authorization
curl -H "Authorization: Bearer your-token" \
     http://localhost:18789/api/status
```

配置：
```json
{
  "gateway": {
    "auth": {
      "type": "token",
      "token": "your-secret-token-min-32-chars"
    }
  }
}
```

### 2. Channel层认证

每个通道有自己的认证方式：

**Telegram**：Bot Token
```json
{
  "channels": {
    "telegram": {
      "botToken": "123456789:ABCdefGHIjklMNOpqrsTUVwxyz"
    }
  }
}
```

**WhatsApp**：Session密钥
```json
{
  "channels": {
    "whatsapp": {
      "sessionName": "default",
      "pairingCode": null
    }
  }
}
```

### 3. Application层控制

最细粒度的权限控制：

```json
{
  "channels": {
    "telegram": {
      "allowFrom": [123456789, 987654321],
      "blockFrom": [111111111],
      "groups": {
        "allowedGroups": [-1001234567890],
        "requireMention": true,
        "mentionPatterns": ["@mybot", "/ai"]
      }
    }
  }
}
```

---

## 权限配置最佳实践

### 场景1：个人使用
```json
{
  "channels": {
    "telegram": {
      "allowFrom": ["你的用户ID"],
      "groups": {"enabled": false}
    }
  }
}
```

### 场景2：小团队
```json
{
  "channels": {
    "telegram": {
      "allowFrom": ["成员1ID", "成员2ID", "成员3ID"],
      "groups": {
        "allowedGroups": ["团队群ID"],
        "requireMention": true
      }
    }
  }
}
```

### 场景3：公开服务
```json
{
  "channels": {
    "telegram": {
      "allowFrom": [],
      "rateLimit": {
        "requestsPerMinute": 10,
        "requestsPerHour": 100
      },
      "usageQuota": {
        "dailyTokens": 100000
      }
    }
  }
}
```

## 获取用户ID的方法

### Telegram
1. 发送消息给 `@userinfobot`
2. 机器人会返回你的用户ID

或在Control UI中查看日志，发送一条消息后能看到：
```
[telegram] Message from user_id: 123456789
```

### WhatsApp
就是你的手机号，格式：`+86138xxxxxxxx`

### Discord
1. 开启开发者模式（设置→高级）
2. 右键点击用户头像→复制ID

## 通道状态管理

### 查看通道状态
```bash
# 列出所有通道
openclaw channels list

# 查看特定通道详情
openclaw channels status telegram
```

### 连接/断开通道
```bash
# 连接通道
openclaw channels connect telegram

# 断开通道
openclaw channels disconnect telegram

# 重新配对
openclaw channels repair whatsapp
```

## 故障排查

### 问题1：配对失败
**可能原因**：
- Token无效或过期
- 网络连接问题
- 平台限制（如国内访问Telegram需要代理）

**解决**：
```bash
# 检查日志
tail -f ~/.openclaw/state/logs/channels.log

# 测试网络连通性
curl https://api.telegram.org/bot<token>/getMe
```

### 问题2：收不到消息
**检查清单**：
- [ ] Gateway是否运行
- [ ] 通道是否已连接
- [ ] 用户是否在allowFrom列表
- [ ] 群组中是否正确@了Bot

### 问题3：消息发不出去
**检查**：
- API密钥是否有效
- 是否触发了频率限制
- 目标用户是否屏蔽了Bot
