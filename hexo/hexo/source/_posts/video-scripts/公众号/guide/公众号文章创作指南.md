# 公众号文章创作指南

> 本文件是「程序员晚枫」公众号长文专属创作规范，整合了写作风格、图片规范、排版模板等全部规则。
> 每次创作新文章前，必须先阅读本文件。
> 每次发布后根据阅读数据反馈，更新优化本文件。
> **本文件不涉及抖音口播稿创作，任何视频脚本相关规范请查看「口播稿创作指南.md」。**

**账号名称**：程序员晚枫

- 粉丝数量：**全网40万+粉丝**

## 二、不可动摇的核心创作原则

| 原则 | 说明 |
|------|------|
| **内容价值优先** | 所有内容必须提供真实可落地的价值 |
| **绝对零门槛** | 全程不用代码、编程术语、行业黑话，面向普通人而非程序员 |
| **信任优先** | 绝不制造焦虑，不割韭菜，帮用户避坑 |
| **聚焦不发散** | 严格围绕AI/科技主线 |
| **阅读友好** | 适配长文阅读习惯，段落清晰，重点突出 |

## 四、标题公式

```
【具体数字/时间锚点】+【事件/产品】+【悬念/预测】= 点击欲望
```

### 4.1 标题类型

| 模式 | 示例 | 适用场景 |
|------|------|---------|
| **数字预测型**（最佳） | 30天后，机器人就要进你家了 | 预测性结论，点击欲望最强 |
| **数字+痛点型** | 花499元，我带你把全球畅销300万册的Python神书从头敲一遍 | 课程推广、实战教程 |
| **对比反差型** | 700页太厚看不下去？跟着我3个项目全部实战，零基础也能学完 | 解决痛点型内容 |
| **事件切入型** | 芯片被卡脖子 + AI反而更便宜 | 从具体事件深度解读 |
| **认知冲突型** | AI这么火 + 为什么一堆公司倒闭 | 已知热点 + 反常识现象 |
| **承诺型** | Python入门第一书：3个项目从入门到实战，不学完不罢休 | 自媒体文章、课程推荐 |
| **疑问型** | 为什么芯片被卡脖子，中国的AI反而越来越便宜？ | 深度分析、热点解读 |
| **故事型** | 30岁被裁员，多亏我提前学了AI | 成长故事、经验分享 |

### 4.2 标题写作规范

1. 长度控制在15-30字之间（比口播稿标题可略长，公众号标题展示空间更大）
2. 必须包含具体数字/时间锚点
3. 要有预测性或悬念感
4. 从具体产品/事件切入，避免空泛

### 4.3 标题禁忌

- ❌ 太长（超过30字）
- ❌ 太官方（缺乏情感）
- ❌ 标题党但不切题
- ❌ 过于平淡（无法引起点击欲望）

## 五、文章开头公式

**核心原则：直击痛点、引发共鸣、快速切入主题**

**标准开头模式**：

```
大家好，我是程序员晚枫。

[直接提出问题或现象]

今天[做某事/分享某经验]...
```

**示例**：

> 大家好，我是程序员晚枫。
>
> 最近在AI编程交流群里，有个问题被反复问到："晚枫，Claude Code我想用，但是因为网络用不了怎么办？"
>
> 我每次都要解释一遍，干脆写篇文章，一次说清楚。

## 六、语言风格

**必须做到**：

1. **生活化比喻**：用70后熟悉的事物（粮票、供销社、大锅饭、下海、万元户）
2. **口语化表达**：使用"说实话"、"你知道吗"等口语化表达，拉近与读者距离
3. **分段清晰**：每个章节不超过300字，列表项控制在5-7个以内
4. **小标题体系**：每300-500字一个大章节，加二级小标题
5. **加粗强调**：核心数据、金句加粗，方便快速浏览
6. **善用类比**：用生活化例子解释复杂概念
7. **情绪调动**：适当使用感叹号，表达真实情感，引起读者共鸣
8. **情绪价值**：有共情、有温度、有共鸣
9. **70后友好**：避免过于新潮的网感词汇，用大白话讲明白就行

## 七、Markdown排版规范

> **核心原则**：写完文章不是终点，必须做一遍排版检查，确保符合公众号阅读习惯。

### 7.1 排版要求

| 要求 | 说明 |
|------|------|
| **短段落** | 每段不超过3行，手机屏幕上超过4行就没人看了 |
| **多留白** | 章节之间用 `---` 分隔，关键转折处空一行 |
| **小标题层级** | 用 `##` 做章节标题，`###` 做子标题，不要跳级 |
| **重点加粗** | 核心观点、数据、金句用 `**加粗**` 标出，方便快速浏览 |
| **列表替代长段** | 3个以上并列项用 `-` 列表，比大段文字更易读 |
| **表格对比** | 有对比关系的内容用表格呈现，一目了然 |

### 7.2 禁止的排版习惯

- ❌ 一段超过5行（手机屏幕上就是一面墙）
- ❌ 连续多段没有小标题（读者找不到路标）
- ❌ 全文没有加粗重点（扫读时抓不住关键信息）
- ❌ 用 `#` 一级标题做正文标题（一级标题留给文章主标题）
- ❌ 代码块裸露无说明（非程序员读者会直接跳过）

### 7.3 排版自检清单

写完文章后，逐项检查：

- [ ] 每段 ≤ 3行
- [ ] 每300-500字有 `##` 小标题
- [ ] 核心观点/数据/金句已加粗
- [ ] 章节间有 `---` 分隔
- [ ] 并列内容用了列表或表格
- [ ] 手机预览阅读体验流畅（没有大段文字墙）

## 九、黑色幽默与嘲讽公式

> 公众号文章节奏比口播稿缓，黑色幽默可适当展开，但同样要遵循点到为止原则。

**核心公式**：
```
具体嘲讽对象 + 反差感 + 一句话点到为止
```

**三种用法**：

① **嘲讽不行动的旁观者**
> Excel用了二十年，每天加班到晚上十点，数据处理还是手动复制粘贴。
> 嗯，那祝你习惯愉快。

② **嘲讽假装努力的假象**
> AI工具买了，钱花了，账号注册了，然后呢？放在那里落灰。
> 这就像你办了健身卡，然后天天看着它过期。

③ **嘲讽自以为占便宜的心态**
> 你以为你在白嫖，其实你才是那个被嫖的。

**禁用**：
- ❌ 不要嘲讽具体的人，只嘲讽行为和现象
- ❌ 不要人身攻击，保持「朋友善意调侃」的尺度

## 十、口播稿同步规则

> **核心原则**：每次创作新的口播稿时，必须同步生成1篇对应的公众号文章。两者为内容同源的不同平台版本，核心观点、案例、数据保持100%一致，仅做平台适配优化。

### 9.1 同步逻辑

| 对比项 | 口播稿 | 公众号文章 |
|--------|--------|-----------|
| 核心论点 | 必须完全一致 | 完全一致 |
| 案例/数据 | 必须完全一致 | 完全一致 |
| 语言风格 | 口语化短句，朗读友好 | 段落式长文，阅读友好 |
| 结构 | 悬念驱动，节奏紧凑 | 逻辑递进，层次分明 |
| 结尾引流 | 标准口播引流格式 | 底部课程板块 |

### 9.2 改编规则

1. **保留100%核心信息**：论点、案例、数据不能因改编而走样
2. **排版适配**：加入二级/三级小标题，每段不超过3行，提升阅读体验
3. **重点突出**：核心观点、数据、金句加粗显示
4. **保留引流元素**：进群banner、课程链接、固定slogan、底部引流板块全部保留

#### YAML引号规则：能不加就不加，特殊字符必须加

**可以去掉引号的情况**：纯中文/英文描述、数字、含普通标点的句子：

```yaml
# ✅ 无需引号
title: AI Agent是什么？2026年最火赛道，5分钟给你讲明白
description: 一文读懂AI Agent，从概念到实战，附5个真实案例。
```

**必须保留引号的情况**：

| 特殊字符 | 原因 | 正确写法 |
|---------|------|---------|
| `:` | YAML键值分隔符 | `"标题: 副标题"` |
| `,` | YAML列表分隔符 | `"华为, Coding Plan"` |
| `#` | YAML注释，会截断值 | `"A#1标签"` |
| `{}` `[]` | YAML对象/数组语法 | `"{name}"` `"第[1]节"` |
| 单引号 `'` | 与YAML单引号冲突 | `"I've used this"` |
| `!` | YAML类型转换标记 | `"价格!查询"` |

```yaml
# ✅ 冒号引号包裹
description: "标题: 副标题"

# ✅ 逗号引号包裹
description: "华为, Coding Plan, 腾讯云"
```

#### 日期格式规范

- 必须使用**当前真实日期**，不得使用未来日期
- 格式必须精确到**时:分:秒**，例如：`2026-04-30 14:30:00`

### 10.2 字数要求

**核心规则：每篇公众号文章不低于 1500 中文字符**

| 文章类型 | 最低字数 | 说明 |
|---------|---------|------|
| 公众号文章 | **1500字** | 含标题、正文，不含代码块/图片alt |
| 故事类文章 | 1200字 | 有叙事性，字数可略少 |
| 干货教程 | 1800字 | 信息量大，可适当增加 |

**为什么1500字？**
- 微信订阅号推送超过1400字后，算法权重更高
- 太短的文章信息密度低，读者获得感不足
- 1500字是「读完有收获」的最低门槛

**章节字数分配参考**：

| 章节 | 占比 | 参考字数 |
|------|------|---------|
| 开头引入 | 10% | ~150字 |
| 每个正文大节 | 20-25% | ~300-350字/节 |
| 结尾行动号召 | 10% | ~150字 |
| 相关阅读+作者信息 | 5% | ~50字 |

## 十二、图片使用规范

### 11.1 图片来源

| 来源 | 网址 | 适用场景 |
|------|------|---------|
| **Unsplash** | https://unsplash.com/ | 通用免费图片（推荐） |

### 11.2 图片类型与尺寸

| 图片类型 | 位置 | 尺寸 | 说明 |
|---------|------|------|------|
| 封面图片 | YAML配置中 | 1200px宽 | 与文章核心主题相关 |
| 章节插图 | 关键段落后 | 800px宽 | 内容与段落主题紧密相关 |
| 产品/课程图 | 产品介绍处 | 原图 | 官方提供或自行制作 |

**封面图片示例**：
```yaml
cover: https://images.unsplash.com/photo-1518770660439-4636190af475?q=80&w=1200&auto=format&fit=crop
```

**章节插图示例**：
```markdown
![](https://images.unsplash.com/photo-1507676270988-22c95378f4b3?q=80&w=800&auto=format&fit=crop)
```

### 11.3 图片主题对照表

| 文章主题 | 封面关键词 | 插图关键词 |
|---------|-----------|-----------|
| AI/大模型 | chip, circuit, AI, technology | data center, server, code |
| Python编程 | python, code, programming | computer, keyboard, code screen |
| 职场/成长 | office, meeting, teamwork | career, business, professional |
| 数据分析 | chart, graph, analytics | data visualization, dashboard |
| 创业/商业 | startup, business, money | business meeting, money, growth |

### 11.4 Unsplash图片URL获取

1. 访问 https://unsplash.com/
2. 搜索关键词
3. 点击图片进入详情页
4. 复制URL，格式：`https://images.unsplash.com/photo-[ID]?q=80&w=1200&auto=format&fit=crop`

**URL参数说明**：

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| `q` | 图片质量 | 80 |
| `w` | 宽度 | 800（插图）/ 1200（封面） |
| `auto=format` | 自动格式优化 | — |
| `fit=crop` | 裁剪适应尺寸 | — |
