# 公众号文章创作指南

> 本文件是「程序员晚枫」公众号长文创作的执行标准。
>
> 目标不是写一篇“正确的文章”，而是写一篇读者愿意点开、愿意读完、愿意留言、愿意转发的 AI 文章。

---

## 1. 账号定位

### 1.1 基本信息

| 项目 | 标准 |
|------|------|
| 账号名称 | 程序员晚枫 |
| 内容定位 | AI 大白话 + AI Coding / AI Agent 落地 |
| 核心 Slogan | 科技不高冷，AI很好用 |
| 粉丝量级 | 全网40万+粉丝 |
| 身份标签 | 帮企业研发和数据团队落地 AI Coding / AI Agent |
| 核心读者 | 企业研发负责人、数据团队 leader、技术管理者、想把 AI 真正用起来的职场人 |

### 1.2 内容边界

文章必须围绕以下方向之一：

- AI 热点大白话：把 AI 新闻翻译成普通人和管理者能听懂的话
- AI Coding / AI Agent 落地：讲工具、方法、团队推广、流程改造
- AI 工具评测：真实体验、真实场景、真实优缺点
- AI 避坑指南：帮读者少花冤枉钱、少踩坑、少被概念忽悠
- AI 时代的职场变化：岗位、效率、组织、成本、技能迁移

禁止把文章写成：

- 纯技术文档
- 纯产品广告
- 纯情绪吐槽
- 纯新闻搬运
- 纯概念科普

### 1.3 核心写作原则

| 原则 | 执行标准 |
|------|----------|
| 有情绪 | 开头必须有明确情绪：震惊、疑惑、讽刺、担忧、兴奋、无奈、警惕 |
| 有事实 | 每篇至少有一个事实锚点：产品发布、价格变化、用户数据、亲身体验、公开报道 |
| 有判断 | 不只复述新闻，必须给出晚枫自己的判断 |
| 有场景 | 必须落到读者的工作、团队、预算、效率、职业选择 |
| 有行动 | 读者读完知道下一步该试什么、避什么、观察什么 |
| 有克制 | 不夸大、不承诺收益、不踩竞品、不制造焦虑 |

## 2. 爆款文章底层公式

### 2.1 AI 公众号爆款四件套

一篇能传播的 AI 文章，通常同时具备这四个元素：

```text
热点事件 + 强烈情绪 + 反常识判断 + 普通人影响
```

示例：

| 热点事件 | 强烈情绪 | 反常识判断 | 普通人影响 |
|----------|----------|------------|------------|
| Copilot 涨价 | 震惊、愤怒 | 免费 AI 时代结束了 | 以后用 AI 会越来越贵 |
| Codex 周活暴涨 | 意外、兴奋 | 它不只是程序员工具 | 知识工作者也会被改变 |
| 国外 AI 懂中文 | 讽刺、疑惑 | 会中文不等于懂中文 | 选工具不能只看国产/海外标签 |

### 2.2 最适合本账号的 6 类选题

| 类型 | 适合标题方向 | 文章重点 |
|------|--------------|----------|
| 价格变化 | 终于开始收费了 / 免费时代结束了 | 拆商业逻辑和普通人怎么应对 |
| 大厂动作 | OpenAI / 微软 / 腾讯 / 阿里突然做了什么 | 拆战略意图和行业影响 |
| 工具体验 | 我用了 X 天，发现一个问题 | 讲真实体验和使用建议 |
| 反常识观察 | 明明应该 A，结果却 B | 把疑惑讲透，制造讨论 |
| 职场变化 | 某岗位减少 / 某能力变重要 | 讲个人和团队该怎么调整 |
| 避坑指南 | 别急着买 / 别急着上 / 先看这几点 | 给判断标准和行动清单 |

### 2.3 选题合格线

写之前先问 5 个问题：

- 这件事最近发生了吗，或者最近被重新讨论了吗？
- 它和 AI 工具、AI Coding、AI Agent、职场效率有关吗？
- 读者会不会有情绪反应？
- 我能不能给出一句明确判断？
- 读者读完能不能知道自己该怎么做？

如果 5 个问题有 2 个以上答不上来，不写。

## 3. 标题规范

### 3.1 标题核心公式

```text
具体对象 + 情绪/反差 + 悬念/判断
```

更强的版本：

```text
具体数字/时间 + 大厂/产品名 + 反常识判断 + 情绪词
```

### 3.2 标题硬指标

- 15-32 个中文字符优先，最长不超过 36 个中文字符
- 必须出现具体对象：产品名、大厂名、岗位、工具、价格、时间
- 必须有情绪或悬念：讽刺、震惊、警惕、奇怪、没想到、撑不住、变天
- 不写空泛标题：AI 趋势分析、AI 工具推荐、人工智能发展观察
- 不写虚假夸张：全球第一、最强、彻底颠覆、稳赚、必火

### 3.3 高点击标题模板

| 类型 | 模板 | 示例 |
|------|------|------|
| 数字冲击 | X 变成 Y，某件事结束了 | Copilot涨价26倍，AI免费时代结束了 |
| 反常识 | 明明 A，为什么反而 B | 明明是国产AI，为什么反而不懂中文 |
| 个人体验 | 我用了 X 天，发现一个问题 | 我用Codex一周，发现它最强的不是写代码 |
| 大厂信号 | 大厂做了 X，真正目的不是 Y | OpenAI进入第三阶段，ChatGPT不想只当聊天工具了 |
| 避坑提醒 | 别急着 X，先看 Y | 别急着买AI电脑，先看这3个问题 |
| 站队争议 | A 和 B，你到底该选谁 | Cursor、Trae、Cline，哪个更适合国内团队 |
| 结果导向 | 用 X 做 Y，结果 Z | 我用AI整理需求，老板以为我招了个助理 |

### 3.4 标题修改规则

初稿标题通常太平。必须至少改 5 版。

每版分别强化一个点：

- 强化数字
- 强化情绪
- 强化反差
- 强化产品名
- 强化读者利益

最终选标题时，优先选择“读者看到会产生问题”的标题。

## 4. 开头规范

### 4.1 开头必须完成 3 件事

前 120 字必须完成：

1. 交代发生了什么
2. 表达强烈情绪
3. 抛出核心问题

### 4.2 标准开头结构

```markdown
大家好，我是程序员晚枫。

最近我看到/用到/发现了一件特别[情绪]的事。

[具体事件或体验。]

我第一反应是：

**[一句强判断。]**

问题来了：

**[全文要回答的核心问题。]**
```

### 4.3 好开头示例

```markdown
大家好，我是程序员晚枫。

最近我用上了 Codex。

用了一段时间之后，我有一个特别强烈的感受：

**很讽刺，国外的 AI，竟然才懂中文。**

这句话不是为了吵架。

而是我真的被震到了。

问题来了：

为什么这种“被理解”的感觉，反而来自一个国外的大模型？
```

### 4.4 禁止的开头

不要这样写：

- “随着人工智能技术的发展……”
- “近年来，AI 逐渐进入大众视野……”
- “本文将从三个方面分析……”
- “今天给大家介绍一款工具……”
- “人工智能是未来发展的重要方向……”

这些开头没有情绪、没有现场感、没有点击后的奖励。

## 5. 正文结构

### 5.1 标准结构

热点观点类：

```text
开头钩子
↓
发生了什么
↓
为什么值得关注
↓
反常识判断
↓
对普通人/团队意味着什么
↓
怎么应对
↓
互动结尾
```

工具评测类：

```text
开头钩子
↓
工具是什么
↓
我实际拿它做了什么
↓
好用在哪里
↓
不好用在哪里
↓
适合谁，不适合谁
↓
怎么上手
↓
互动结尾
```

避坑指南类：

```text
开头痛点
↓
常见误区
↓
为什么很多人会踩坑
↓
正确判断标准
↓
具体避坑清单
↓
互动结尾
```

### 5.2 每篇必须有一个“晚枫判断”

文章不能只说“这件事发生了”。

必须写出一句可传播的判断：

- **免费的 AI 永远存在，但好用的 AI 会越来越贵。**
- **AI 不是替你负责的人，它只是替你干活的工具。**
- **会中文不等于懂中文，懂中文的本质是懂人话。**
- **企业上 AI，最难的不是工具，而是流程和责任边界。**

判断标准：

- 能单独摘出来发朋友圈
- 有一点锋利
- 不需要专业背景也能懂
- 不夸大到无法证明

### 5.3 每篇必须有真实感

以下至少出现 2 个：

- “我最近实际用了……”
- “有读者/群友问我……”
- “我看到一个案例……”
- “我试着让它做了三件事……”
- “最让我意外的是……”
- “这里我不建议大家盲目……”

不要写成公关稿。

读者不是来听产品发布会的。

### 5.4 每 300-500 字一个小标题

小标题要像判断，不要像论文目录。

好的小标题：

- 发生了什么？
- 真正贵的不是价格
- 它不是不会中文，是不懂语气
- 企业最怕的不是不好用，是说不清责任
- 普通人现在别急着买

差的小标题：

- 背景介绍
- 原因分析
- 功能说明
- 总结展望

## 6. 语言风格

### 6.1 核心风格

```text
专业判断 + 大白话表达 + 真实情绪 + 适度反讽
```

读者应该感觉：

- 这个人懂 AI
- 这个人说人话
- 这个人不是在卖焦虑
- 这个人有自己的判断

### 6.2 句子规则

- 每段不超过 3 行
- 单句尽量短
- 多用“但问题来了”“说白了”“真正麻烦的是”“这才是关键”
- 少用“赋能、闭环、抓手、生态、范式、矩阵、底座”等空话
- 专业词可以用，但必须解释成普通话

示例：

| 不推荐 | 推荐 |
|--------|------|
| Agent 具备多步骤任务规划能力 | 你给它一个目标，它能自己拆步骤往前做 |
| Token 消耗会造成边际成本上升 | 你问得越多，它背后的算力账单越贵 |
| 企业需要构建 AI 原生工作流 | 别只买工具，要把谁用、怎么用、结果谁负责说清楚 |

### 6.3 情绪表达规则

允许有情绪，但必须基于事实。

可以写：

- 我真的被震到了
- 这事越想越讽刺
- 这个价格普通人很难不骂
- 这不是小更新，是商业模式变了

不要写：

- 全网炸了
- 彻底完了
- 所有人都慌了
- 不买就落后
- 再不用就被淘汰

### 6.4 黑色幽默规则

每篇可以有 1-2 处轻微反讽。

公式：

```text
具体行为 + 反差 + 一句话收住
```

示例：

```markdown
AI工具买了，账号注册了，教程收藏了。

然后呢？

放在那里落灰。

这就像办了健身卡，然后每天路过健身房门口给自己点赞。
```

注意：

- 只讽刺行为，不攻击具体人
- 不用脏话
- 不做人身攻击
- 不把读者骂走

## 7. 工具评测专项规范

### 7.1 工具文章必须有实测

不能只写官网功能。

至少写清楚：

- 我用它做了什么任务
- 它哪里确实省事
- 它哪里还不稳定
- 它适合谁
- 它不适合谁
- 免费版/付费版/企业版有什么注意点

### 7.2 对比竞品要克制

不要写：

- “吊打”
- “碾压”
- “国产最强”
- “比 OpenAI 还强”
- “Codex 完全做不到”

推荐写：

- “更适合国内使用场景”
- “上手成本更低”
- “在中文业务里更顺”
- “企业合规更容易沟通”
- “不同工具适合不同场景”

### 7.3 工具评测推荐结构

```markdown
## 先说结论

[一句克制判断。]

---

## 我拿它做了什么？

[3 个真实任务。]

---

## 好用在哪里？

[具体体验，不写空话。]

---

## 坑在哪里？

[限制、门槛、不稳定点。]

---

## 适合谁？

[明确人群。]

---

## 怎么上手？

[链接、步骤、建议。]
```

### 7.4 商业合作/邀请链接写法

可以放链接，但不能让文章变成硬广。

推荐顺序：

```text
先讲真实体验
再讲适合人群
再讲注意事项
最后放链接
```

链接前推荐写：

```markdown
如果你本来就在找国内能用的 AI 编程工具，可以先注册试一下。

不要一上来就迁移正式项目。

先拿一个小项目测三件事：读代码、改页面、修报错。
```

## 8. 结尾与互动

### 8.1 结尾不要复读全文

不要写：

```markdown
今天我们讲了三点：
第一……
第二……
第三……
```

这种结尾像 AI 生成的会议纪要。

### 8.2 标准结尾结构

```text
一句扎心总结
↓
一句反问
↓
评论区问题
↓
固定 Slogan
```

### 8.3 互动问题必须具体

差：

- 大家怎么看？
- 欢迎留言讨论。
- 你觉得呢？

好：

- 你用国产 AI 和国外 AI 的时候，有没有明显感觉到差别？
- 如果 AI 编程工具开始按量收费，你每个月能接受多少钱？
- 你们公司现在最卡 AI 落地的点，是安全、预算、流程，还是员工不会用？
- 你试过让 AI 改真实项目吗？成功了还是翻车了？

### 8.4 固定结尾

每篇文章底部必须包含：

```markdown
**科技不高冷，AI很好用。**
我是晚枫，关注我，带你一起玩AI！
```

## 9. Front Matter 与文件规范

### 9.1 Front Matter 标准

```yaml
---
title: 文章标题
date: YYYY-MM-DD HH:MM:SS
tags: [公众号文章, AI热点, 标签1, 标签2]
categories: [公众号文章, AI热点大白话]
cover: https://images.unsplash.com/...
---
```

规则：

- 标题含冒号、逗号、感叹号、引号时，用英文双引号包住
- tags 控制在 4-6 个
- categories 按目录方向填写
- cover 必须是可访问图片

### 9.2 文件存放

| 内容方向 | 目录 |
|----------|------|
| AI热点大白话 | `video-scripts/公众号/ai-hotspots/` |
| AI工具评测/落地 | `video-scripts/公众号/ai-tools/` |
| AI避坑指南 | `video-scripts/公众号/ai-avoid-pitfalls/` |
| AI Coding 方案/计划 | `video-scripts/公众号/coding-plan/` |
| 创作指南 | `video-scripts/公众号/guide/` |

命名规则：

```text
序号-文章标题.md
```

示例：

```text
037-很讽刺，国外的AI竟然才懂中文.md
```

## 11. 合规规则

### 11.1 绝对禁止

- 不写医疗效果声明：AI 能治病、诊断、替代医生
- 不写金融收益承诺：稳赚、保证收益、荐股、内幕
- 不写违法违规用途：绕过监管、破解、盗号、诈骗、爬隐私数据
- 不写政治敏感内容
- 不写色情、暴力、低俗内容
- 不做人身攻击

### 11.2 谨慎使用

| 高风险表达 | 推荐替代表达 |
|------------|--------------|
| 最强、第一、全球领先 | 很强、很有代表性、值得关注 |
| 吊打、碾压 | 更适合某个场景 |
| 免费、永久免费 | 当前免费、基础功能免费、限时免费 |
| 保证、省钱赚钱 | 降低成本、提高效率、减少重复劳动 |
| 被淘汰、完蛋了 | 压力会变大、竞争门槛会提高 |

### 11.3 事实来源

涉及以下内容必须加来源或降级表达：

- 用户数
- 收入/亏损
- 融资金额
- 涨价幅度
- 公司战略
- 官方发布时间
- 竞品能力对比

没有来源时，写成：

- “有用户反馈”
- “从公开信息看”
- “我个人体验是”
- “更准确地说”
- “可能更适合”

不要写成确定事实。

## 12. 发布前检查清单

### 12.1 标题检查

- [ ] 标题有具体对象
- [ ] 标题有情绪、悬念或反差
- [ ] 标题不是空泛概念
- [ ] 标题没有夸大宣传
- [ ] 标题读完能让人产生问题

### 12.2 开头检查

- [ ] 前 120 字说明发生了什么
- [ ] 有明确情绪
- [ ] 有核心问题
- [ ] 没有“随着 AI 发展”这种套话

### 12.3 正文检查

- [ ] 有事实锚点
- [ ] 有晚枫自己的判断
- [ ] 有真实体验或具体场景
- [ ] 有普通人/团队影响
- [ ] 有行动建议
- [ ] 每 300-500 字有小标题
- [ ] 每段不超过 3 行

### 12.4 工具评测检查

- [ ] 写了真实任务，不只是官网功能
- [ ] 写了优点
- [ ] 写了限制或注意事项
- [ ] 写了适合谁和不适合谁
- [ ] 邀请链接没有压过正文价值

### 12.5 合规检查

- [ ] 没有“最强、第一、吊打、碾压”等绝对化表达
- [ ] 数据和报道有来源
- [ ] 竞品对比克制
- [ ] 没有收益承诺
- [ ] 没有医疗、金融、政治等高风险内容

### 12.6 结尾检查

- [ ] 不是列表式复读全文
- [ ] 有一句扎心总结
- [ ] 有具体互动问题
- [ ] 有固定 Slogan
