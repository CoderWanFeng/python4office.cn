# 口播稿创作指南

> 本文件是「程序员晚枫」抖音口播稿专属创作规范。
> 每次创作新视频脚本前，必须先阅读本文件。
> 每次发布后根据数据反馈，更新优化本文件。
> **本文件不涉及公众号文章创作，任何文章相关规范请查看「公众号文章创作指南.md」。**

---

## 一、账号定位与内容主线

**账号名称**：程序员晚枫  
**内容定位**：AI编程搞钱大白话  
**核心slogan**：科技不高冷，AI很好用

**内容比例**：
- 70% AI编程搞钱实战（主推）
- 20% 效率工具测评
- 10% 副业认知

**目标受众**：普通大众，尤其是有付费能力的职场人、宝妈、想做副业的人
**绝不针对程序员群体**——内容面向的是「想用AI搞钱的普通人」，不是「想学编程的技术人」

**绝对禁止**：纯工具安装/操作步骤类教程（只有搜索流量，无法涨粉）

---

## 二、不可动摇的核心创作原则

| 原则 | 说明 |
|------|------|
| **零门槛** | 全程不用代码、编程术语、行业黑话，面向普通人而非程序员 |
| **信任优先** | 绝不制造焦虑，不割韭菜，帮用户避坑 |
| **聚焦不发散** | 严格围绕AI编程搞钱主线 |
| **情绪价值** | 要有共情、有温度、有共鸣 |
| **黑色幽默** | 嘲讽现象不嘲讽人，一句话点到为止 |

## 三、数据验证成功的创作规则

基于账号历史数据，以下规则已经过验证，必须严格执行：

### 3.1 视频时长控制

| 推荐时长 | 说明 |
|--------|------|
| 90-120s | 最佳，既保证完整性，又兼顾完播率 |
| 60-90s | 可接受范围 |
| ＞120s | 若非必要，尽量避免 |

**完播率底线**：35%以上为优秀线

### 3.2 开头必须用「提问+悬念」公式

| ❌ 禁止使用的开头 | ✅ 必须使用的开头 |
|------------------|------------------|
| "今天教大家…" | "国内算力更便宜，90%的人不知道原因" |
| "保姆级教程…" | "花了3千块买来的教训，看完省一半" |
| "教你怎么用…" | "别再手动做表了，AI3秒帮你搞定" |

**标准结构**（15秒内完成）：
```
第一句：抛出"反常识"/"特别"/"你可能没想到"
中间：堆叠2-3个具体事实/数据
转折：用"但你看看"/"但实际上"
收尾：用短问句收住（凭什么？/ 为什么？/ 怎么回事？）
```

**成功案例**：
> 你有没有发现一件特别反常识的事——
> 芯片被卡脖子，GPU到处缺货，
> 训练一个大模型烧的钱够盖一栋楼，
> 按理说，AI应该贵到离谱才对。
> 但你看看现在，DeepSeek几分钱一次，豆包免费用，价格还在一直降。
> 凭什么？

### 3.3 内容方向死磕「AI编程搞钱」

- ✅ 优先做：AI搞钱、副业、职业发展、行业解读类
- ❌ 禁止做：纯工具安装、操作步骤类教程

## 四、标题公式

```
【具体数字/时间锚点】+【事件/产品】+【悬念/预测】= 点击欲望
```

**三种标题类型**（按效果排序）：

1. **数字预测型**（最佳）：具体时间 + 预测性结论
   - 30天后，机器人就要进你家了
   - 整整一个亿，单纯为了去找更有创意的作品和人？

2. **事件切入型**：具体事件 + 深度解读
   - 芯片被卡脖子 + AI反而更便宜 = 002数据最佳

3. **认知冲突型**：已知热点 + 反常识现象
   - AI这么火 + 为什么一堆公司倒闭

**写作规范**：
1. 长度控制在15-25字之间
2. 必须包含具体数字/时间锚点
3. 要有预测性或悬念感（"就要"、"居然"、"终于"）
4. 从具体产品/事件切入，避免空泛

## 五、封面文案公式

**标准格式**：
```
第一行：大众已知/认同的观点（1-5字）
第二行：反常识/出乎意料的现象（1-5字）
```

**核心原则（最重要）**：
- ✅ 严格2行，每行**不超过5个字**
- ✅ **越扎心、越有争议越好**
- ❌ 禁忌：平淡陈述、官话套话、没有情绪张力

**好封面的判断标准**：用户刷到这条视频，2行封面看完，不点进去难受。

**好封面 vs 差封面对比**：

| 类型 | 第一行 | 第二行 | 效果 |
|------|--------|--------|------|
| ❌ 差 | AI开始普及 | 你要学会 | 废话，没情绪 |
| ❌ 差 | AI时代来了 | 我们怎么办 | 太泛，没争议 |
| ✅ 好 | 行业很火 | 都在倒闭 | 认知冲突，反常识 |
| ✅ 好 | AI涨价了 | 你还在等 | 扎心，紧迫感 |
| ✅ 好 | 免费AI | 你被嫖了 | 争议性，拉满好奇心 |
| ✅ 好 | 你会AI吗 | 淘汰你更快 | 直击焦虑 |

**创作步骤**：
1. 先写第一行：陈述一个用户知道的现状（1-5字）
2. 再写第二行：对这个现状补一刀，要求比第一行更扎心（1-5字）
3. 检查：不点进去难受吗？→ 如果犹豫，改第二行
4. **口播稿里只保留最终选定的一个方案，不要留备选**

## 六、语言风格公式

**必须做到**：
1. **生活化比喻**：用熟悉的事物做比喻
2. **短句节奏**：每句不超过20字，多用短句
3. **情绪价值**：有共情、有温度、有共鸣
4. **聊天语气**：像朋友聊天，不像老师讲课

---

## 七、黑色幽默与嘲讽公式

**为什么用**：黑色幽默让人看完说「对对对，我就是这样的」，然后忍不住转发。

**核心公式**：
```
具体嘲讽对象 + 反差感 + 一句话点到为止
```

**三种用法**：

① **嘲讽不行动的旁观者**
> Excel用了二十年，每天加班到晚上十点，数据处理还是手动复制粘贴。
> 嗯，那祝你习惯愉快。

② **嘲讽假装努力的假象**
> AI工具买了，钱花了，账号注册了，然后呢？放在那里落灰。
> 这就像你办了健身卡，然后天天看着它过期。

③ **嘲讽自以为占便宜的心态**
> 你以为你在白嫖，其实你才是那个被嫖的。

**禁用**：
- ❌ 不要嘲讽具体的人，只嘲讽行为和现象
- ❌ 不要人身攻击，保持「朋友善意调侃」的尺度
- ❌ 不要太狠，用户听完不舒服会划走

**成功案例库**：
- ✅ "Excel用了二十年，还觉得自己挺勤快"
- ✅ "光喊口号不带头冲，那叫耍流氓"
- ✅ "免费才是最贵的奢侈品"
- ✅ "你以为你在白嫖，其实你才是那个被嫖的"
- ❌ "程序员都是笨蛋"（人身攻击，太狠）
- ❌ "你不用AI就是傻子"（攻击用户，会划走）

## 八、四内容合集专属标准

### 合集1：AI热点大白话
- **定位**：把AI/科技热点翻译成普通人听得懂的人话
- **结构**：热点事件→3句话讲清核心→对普通人的3个影响→行动指引→结尾引导

### 合集2：普通人的AI工具库
- **定位**：实测普通人能直接用的AI工具，讲清用法和避坑点
- **结构**：痛点引入→工具价值→分步实操→避坑提醒→结尾引导

### 合集3：AI入门避坑指南
- **定位**：帮零基础用户避开AI相关的坑、破解焦虑
- **结构**：坑/焦虑点引入→拆解坑人逻辑→正确认知→避坑清单→结尾引导

### 合集4：AI黑话翻译器 🆕
- **定位**：用大白话拆解老板/媒体高频提到的AI专业术语，帮普通人扫清认知障碍
- **核心公式**：术语抛出 + 一句话翻译 + 普通人用得上的场景 + 行动指引
- **禁忌**：❌ 只讲概念不讲用法 / ❌ 用其他术语解释术语
- **成功案例结构**：
  1. 先抛一个「老板/新闻最近老提的词」
  2. 大白话翻译成生活化比喻
  3. 给一个「普通人也能用上的具体场景」
  4. 结尾引导：学AI编程才能真正用上这些工具

| 术语 | 大白话翻译 | 普通人场景 |
|------|-----------|-----------|
| AI Agent | AI不再只聊天，开始自己干活了，相当于一个不用休息的员工 | 帮你查资料、做表格、盯股票 |
| LLM/大模型 | 一个读过全世界书的超级学生，什么都能聊 | DeepSeek/豆包/ChatGPT都是不同学生 |
| Token | AI的计费单位，相当于按字数收钱 | 问长问题比短问题贵 |
| Prompt | 跟AI说话的技巧，说得好AI就给好答案 | 同样工具，你问和高手问结果差十倍 |
| RAG | AI回答不准？给它配个专业资料库 | 问公司制度/产品手册，AI也能答对 |

**选题优先级**：AI Agent > LLM > Token > Prompt > RAG
