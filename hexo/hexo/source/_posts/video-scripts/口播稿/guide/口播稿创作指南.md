# 口播稿创作指南

> 每次创作前必须读一遍。每条稿子发布后根据数据更新。
> **本文不涉及公众号文章，相关规范见「公众号文章创作指南.md」**。
>
> ⚠️ **2026-06-11 全量重写**：账号定位从"普通职场人的AI实战派"全面切到 B2B 口径（企业研发/数据团队的 AI Coding / AI Agent 落地顾问）。所有栏目、平台策略、选题、写法全部按 B2B 排。

---

## 一、账号定位

- **名称**：程序员晚枫
- **定位**：帮企业研发/数据团队把 AI Coding 和 AI Agent 落地为可控、可培训、可评估的团队生产力的实战型博主
- **slogan**：科技不高冷，AI很好用
- **人设**：程序员底色（真做过 Cursor / Claude Code / Codex / WorkBuddy 等工具的 B 端落地陪跑）+ 真相挖掘（先讲企业不该做的，再讲该做的）+ 不忽悠（ROI 说不上来就直说）+ 团队落地（5-50 人研发团队怎么推、怎么算账、怎么考核）
- **受众**：企业研发负责人、数据团队 leader、CTO、想给团队上 AI 的中层管理者
- **不写**：宝妈 / 学生 / 打工人 / 副业 / 学 AI 写代码等大众钩子

**一句话定位**：

> 我帮企业研发/数据团队，把 AI Coding 和 AI Agent 变成可控、可培训、可评估的团队生产力。

**内容比例（2026-06-11 重新排定）**：

| 类型 | 占比 | 作用 |
|------|------|------|
| **AI Coding 落地实操**（B 站/知乎为主） | 40% | 涨粉、建立专业感、转化 B 端客户 |
| **AI 行业判断 / 趋势预判**（视频号/知乎） | 35% | 涨粉、B 端人设、老板转发 |
| **AI 热点大白话**（抖音/视频号） | 25% | 拉播放、引流到主战场 |

> ⚠️ **硬性约束**（不遵守 = 播放量再高也不涨粉）：
> - AI 热点大白话**每周不超过 2 条**（之前是 4 条）
> - AI Coding 落地实操**每周不少于 3 条**（主战场）
> - AI 行业判断**每周不少于 2 条**（B 端人设）
> - 任何一条都必须有"对企业研发/数据团队有什么用"的判断
> - 违反此规则，流量再高也不涨粉

**注意**：工具实操不是普通教程号写法。每条工具内容都必须回答一个问题：

> 这个工具对一个 5-50 人的研发团队来说，意味着什么改变？管理者应该先做哪 3 件事？

## 二、三类内容栏目写法

所有口播稿都必须先归入以下三类之一。不要混成"什么都讲一点"。

### 1. AI Coding 落地实操类：负责涨粉 + B 端转化

**适合选题**：

- Cursor / Claude Code / Codex / WorkBuddy 在企业研发团队的落地
- AI Agent 工程化（MCP、Function Calling、效果评估）
- 团队 AI 工具链选型横评
- 研发流程改造（代码 review、单元测试、CI/CD 加 AI）
- 团队 AI 培训 + 考核体系
- AI Coding ROI / 成本审计

**写法公式**：

```text
一个真实企业场景（5-50 人研发团队）
+ 这个场景下 AI Coding 工具怎么用
+ 团队踩了什么坑
+ 效果怎么量化（ROI / KPI / 工时）
+ 管理者可以怎么复刻
```

**开头示例**：

- "我让 5 人研发团队用 Cursor 重构老项目，30 天后学到了 3 件事。"
- "Claude Code 在企业落地的 4 个死结，90% 团队都会踩。"
- "Codex 进了企业才知道：3 个演示不会告诉你的真实问题。"

**结尾必须导向下一步**：

```text
下一条我直接拆解，一个 30 人研发团队推 Cursor 的 90 天复盘。
```

### 2. AI 行业判断 / 趋势预判类：负责人设 + 老板转发

**适合选题**：

- AI 涨价 / 免费 / 商业模式变化对企业研发预算的影响
- 大厂 AI Coding 采购信号（Copilot、Codex、WorkBuddy）
- 研发岗位被 AI 重排的 3 个信号
- AI Agent 工程化元年（MCP、效果评估、合规）
- Token 成本与 Coding Plan 选型

**写法公式**：

```text
一个 AI 新闻
+ 大多数团队以为 A
+ 真正的问题是 B
+ 对企业研发/数据团队意味着什么
+ 管理者该先做哪 3 件事
```

**开头示例**：

- "AI Coding 涨价不是坏事，真正危险的是另一件事。"
- "大厂都在 AI Coding 试点，真正影响的是你团队的 KPI。"
- "AI Agent 集体发布，研发岗位要被重组的 3 个信号。"

**结尾必须导向实操**：

```text
下一条我把这件事拆成 3 步，CTO 可以照做。
```

### 3. AI 热点大白话类：负责拉播放（每周不超过 2 条）

**适合选题**（必须和 B 端挂钩）：

- AI 公司涨价 / 免费 / 亏损 / 收费 → 对研发预算的影响
- 大厂发布新功能 → 对企业研发选型的影响
- 招聘 / 裁员 / 行业变化 → 对研发团队结构的影响
- AI 安全 / 隐私 / 合规新闻 → 对企业合规的影响

**写法公式**：

```text
一个 AI 新闻
+ 表面现象
+ 真正关键是什么
+ 对企业研发/数据团队意味着什么
+ 管理者该怎么做
```

**开头示例**：

- "OpenAI 涨价 26 倍，研发预算被砍 60%，你团队该怎么办。"
- "免费 AI 工具用得越爽，研发团队越要小心。"
- "这波 AI 裁员，真正受影响的不是程序员。"

**结尾必须导向下一条实操**：

```text
下一条我直接拆解，这件事对一个 30 人研发团队意味着什么。
```

## 三、三平台适配规则

同一条内容可以三平台同步，但标题、开头和结尾不能完全一样。

### B 站：深度长视频

**目标**：拿深度 + 拿收藏 + B 端专业人设。

**标题写法**：

- 我让 5 人研发团队用 Cursor 重构老项目，30 天后学到了 3 件事
- Claude Code 在企业落地的 4 个死结
- 一个 CTO 给团队的 AI Coding 工具选型方法论
- Cursor / Trae / Claude Code 4 款工具的 30 人研发团队横评
- AI Agent 工程化落地的 5 个真实难点

**结尾导向**：

```text
完整版工具链横评在我的 B 站专栏，关注后第一时间看下一条。
```

### 知乎：深度图文

**目标**：让 B 端决策者收藏、转发、引用。

**标题写法**：

- 企业落地 AI Coding 之前，先想清楚这 4 件事
- 30 人研发团队推 Cursor 的 90 天复盘：3 个教训
- AI Agent 工程化落地的 5 个真实难点（不是演示那种）
- 研发效能：AI Coding 怎么衡量 ROI
- 一个 CTO 给团队的 AI Coding 工具选型方法论

**结尾导向**：

```text
落地实操步骤我整理成了 AI Coding 落地实战课 · 企业版，关注后私信"企业版"获取。
```

### 视频号：老板 / HR 转发向

**目标**：适合转发给老板、同行、CTO 群。

**标题写法**：

- 研发总监注意了，AI Coding 落地前必须做的 3 件事
- 未来 2 年，研发团队会被 AI Coding 重排
- 老板不会告诉你的 AI Coding 真实成本
- 一个 30 人研发团队的 AI 落地 90 天复盘
- 这类研发岗位，会最先被 AI Coding 工作流改变

**结尾导向**：

```text
转给还在观望的研发总监。
```

### 抖音：热点拉流（每周不超过 2 条）

**目标**：拿停留、拿推荐、引流到主战场（B 站/知乎）。

**标题写法**：

- 我求你别再给团队乱发 Cursor 账号了
- AI Coding 越免费，研发团队越要小心
- Codex 不是程序员玩具，企业研发也该重新评估
- 这个 AI Coding 工具我不建议小公司乱碰
- AI 都在裁员，研发负责人该怎么和老板谈判

**结尾导向**：

```text
完整版拆解见知乎/B 站同名号"程序员晚枫"。
```

## 四、最高优先级规则：第一句话必须把人留住

> ⚠️ **这是整个指南里最重要的一条，没有之一。**
> 前 3 秒留不住人，后面写得再好也没用——用户已经划走了。

### 核心要求

**第一句话（前 3 秒）必须完成以下任意一项**：

| 钩子类型 | 效果 | 示例 |
|---------|------|------|
| **反常识冲突** | 用户"诶？不对吧"→ 想继续看 | "AI Coding 越免费，研发团队越要小心" |
| **信息差暗示** | 用户"他有我不知道的东西"→ 想验证 | "Codex 进了企业才知道，3 个演示不会告诉你的事" |
| **数字锚点** | 用户被具体数字吸引 → 想了解 | "我让 5 人研发团队用 Cursor 重构老项目，30 天后" |
| **直接打脸（B 端）** | 用户"他在说我们做错了？"→ 想反驳 | "AI Coding 用了 3 个月，团队 KPI 没变好的根本原因" |
| **管理者点名** | 用户"说的是我"→ 想看下去 | "研发总监注意了，AI Coding 落地前必须做的 3 件事" |

**禁止使用的开头**：
- ❌ "今天教大家…"
- ❌ "你有没有发现…"
- ❌ "大家好，我是…"
- ❌ 任何铺垫词（"其实啊""话说回来""大家都知道"）
- ❌ "宝妈/学生/打工人…"（B 端账号禁止用大众钩子）

### 检查标准

写完开头第一句，问自己三个问题：
1. 研发负责人/数据团队 leader 刷到这条，会在 3 秒内划走吗？→ 会 = 重写
2. 第一句能让目标用户想"然后呢？"吗？→ 不能 = 重写
3. 如果把第一句删掉，后面的内容还能成立吗？→ 能 = 第一句没用，重写

## 五、人设锚点规则：每篇必须让用户记住你是谁

> ⚠️ **流量高但不涨粉，核心原因就是缺这条。**
> 用户看完觉得"讲得不错"→ 但不记得是谁讲的 → 不关注。

### 核心要求

**每篇稿子必须至少有一个"人设锚点"**——让目标用户看完知道"这是那个帮企业落地 AI Coding 的程序员晚枫讲的"。

| 插入位置 | 写法示例 |
|---------|---------|
| 开头钩子之后（推荐） | "我是晚枫，专门帮企业研发/数据团队把 AI Coding 和 AI Agent 真正落地。" |
| 反转之后 | "我是晚枫，给 5-50 人研发团队做 AI Coding 落地陪跑。" |
| 结尾互动之前 | "关注晚枫，B 端 AI Coding 落地，只讲真话。" |
| 工具实操中 | "我是晚枫，Cursor / Claude Code / WorkBuddy 都在企业跑过真项目。" |

**注意**：不是每篇都写一样的话，而是每篇至少有一处——人设锚点可以换着说，但不能没有。

### 反面案例

| ❌ 没有锚点的结尾 | 问题 |
|---------------------|------|
| "收费了，你还用吗？" | 用户评论完就走了，不记得是谁 |
| "你怎么看？" | 没有任何人设记忆点 |
| "学会了吗？记得点赞哦" | 假流量话术，B 端用户反感 |

### 正面案例

| 账号 | 锚点手法 | 效果 |
|------|---------|------|
| 老爸评测 | "较真、死磕"每期都出现 | 强人设记忆 |
| 直男财经 | "我是直男"每期开场 | 强关注转化 |
| 程序员晚枫（B 端版） | "我是晚枫，帮企业研发落地 AI Coding" | 强 B 端人设 |

## 六、时长与节奏：观点完整优先，每句话必须推进

### 时长原则

| 内容类型 | 推荐时长 | 说明 |
|---------|---------|------|
| **AI 热点大白话** | 45-60s | 适合快评、单点热点 |
| **AI 行业判断 / 趋势预判** | 60-90s | 适合解读资讯、讲清一个判断 |
| **AI Coding 落地实操** | 90-180s | 适合展示过程、结果和复刻方式 |
| ＞180s | 谨慎 | 必须有强故事线和连续信息增量 |

**判断标准**：不是越短越好，而是不能散。

录完后问自己四个问题：

1. 这条的核心观点能不能用一句话说清？
2. 事实、判断、结论之间有没有因果关系？
3. 研发负责人/管理者看完会不会觉得"他帮我看懂了一件事"？
4. 如果删掉某一句，观点会不会更清楚？会 = 删。

### 解读类内容必须有主线

热点解读不能写成"新闻碎片合集"。必须按这个顺序组织：

```text
表面发生了什么
+ 大多数人会怎么看
+ 晚枫认为真正关键是什么
+ 这件事对企业研发/数据团队意味着什么
+ 管理者应该先做哪 3 件事
```

**底线要求**：

- 开头 10 秒必须抛出核心判断
- 中间每 15-20 秒必须推进一层，不重复同一个意思
- 结尾必须给管理者一个清晰结论，不要只留问题
- 可以超过 45 秒，但不能超过一个清晰故事所需的长度

### 删掉所有冗余

| ❌ 必须删除 | 原因 |
|-------------|------|
| "你有没有发现/有没有这种感觉" | 铺垫词，浪费前 3 秒 |
| "其实啊/话说回来/更重要的是" | 口头禅，没信息量 |
| 连续超过 3 句没有推进 | 用户走神，必须加入反转/例子/结论 |
| 单句超过 20 字 | 太长，观众记不住 |
| "宝妈/学生/打工人"等大众钩子 | 与 B 端定位冲突，必须删除 |

**节奏模板（资讯解读版）**：
```
0-5s   ：第一句话钩子 + 核心判断
5-20s  ：交代新闻事实，只讲必要信息
20-45s ：指出大众误读，给出你的反转判断
45-70s ：解释为什么，用具体产品/事件/数据支撑
70-90s ：落到企业研发/数据团队视角，给出 3 步行动建议
```

## 七、内容质量：绑定具体工具 + 真实企业场景，禁止空讲理论

### 必须绑定具体工具 / 真实企业场景

| ❌ 禁止（空讲理论） | ✅ 必须（绑定具体） |
|---------------------|---------------------|
| "AI Coding 很好用，团队要学" | "Cursor 进了 30 人研发团队，30 天后 ROI 怎么算" |
| "AI 可以帮研发提效" | "Claude Code 改了一个真实项目，节省多少工时" |
| "免费 AI 工具不行" | "DeepSeek 免费，但它的代码合规问题怎么解" |

**数据支撑**：绑定具体工具/真实企业的视频，平均播放量是纯理论内容的 3.6 倍。

### 工具内容必须绑定真实企业场景

| ❌ 禁止（像说明书） | ✅ 必须（像实战复盘） |
|---------------------|---------------------|
| "Cursor 有很多功能" | "我让 5 人研发团队用 Cursor 重构老项目" |
| "Claude Code 很好用" | "我用 Claude Code 处理了一个 30 人团队的真实项目" |
| "AI Coding 可以改流程" | "我们用 AI Coding 改造了 1 个研发流程，省了 40% 时间" |

工具实操必须说清楚三件事：

1. **真实场景**：5-50 人研发团队，具体业务是什么
2. **量化结果**：工时、KPI、ROI、合规风险中至少给出 1 个量化指标
3. **可复刻性**：管理者应该怎么开始，第一步做什么

### 观点闭环：必须讲清楚"所以企业该怎么做"

每条稿子都要形成闭环：

```text
新闻是什么
为什么重要
你的判断是什么
企业研发/数据团队该看懂什么
管理者该先做哪 3 件事
```

禁止只说"这很重要""这不是工具""这是入口"，但不解释为什么，也不告诉企业怎么做。

### 情绪反转：至少翻转一次

观众需要一个"诶？不对吧"的瞬间。每条稿子至少出现一次反转：

- 大多数人以为 → 真相是
- 表面现象 → 真正关键
- 大家都说 → 实际上
- 看似正确 → 其实错误

## 八、结尾标准模板

每条稿子结尾**必须**给到管理者一个明确动作，不能只留问题。

**3 类收尾**：

**1. 实操导向**（最推荐）
```text
下一条我直接拆解，一个 30 人研发团队推 Cursor 的 90 天复盘。
下一条我把这件事拆成 3 步，CTO 可以照做。
下一条给一个 5 步 ROI 评估法，研发负责人可以直接套。
```

**2. 转发导向**（视频号常用）
```text
转给还在观望的研发总监。
转给那个不敢给团队上 AI Coding 的 CTO。
```

**3. 思考导向**（B 站/知乎用）
```text
你怎么看？你团队推 AI Coding 的时候，最先卡在哪一步？评论区说说。
关于这件事，你站哪一边？评论区聊聊。
```

**禁止的结尾**：
- ❌ "关注我，下条继续"
- ❌ "你学会了吗？"
- ❌ "记得点赞收藏"
- ❌ 任何只有一句问题、不给管理者行动的结尾
