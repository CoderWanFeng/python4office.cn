# 口播稿创作指南

> 每次创作前必须读一遍。每条稿子发布后根据数据更新。
> **本文不涉及公众号文章，相关规范见「公众号文章创作指南.md」。**

---

## 一、账号定位

- **名称**：程序员晚枫
- **定位**：AI行业的真相挖掘者 + 普通职场人的AI实战派
- **slogan**：科技不高冷，AI很好用
- **人设**：程序员底色（查数据/看代码/实测工具）+ 真相挖掘（给你不知道的角度）+ 不忽悠（不好用直接说）+ 工作流实战（教你怎么用）
- **受众**：普通大众，有付费能力的职场人/宝妈/想做副业的人
- **绝不针对程序员群体**

**一句话定位**：

> 我帮普通职场人，把AI变成真正能干活的工具。

**内容比例（2026年6月三平台数据验证）**：
| 类型 | 占比 | 作用 |
|------|------|------|
| AI热点大白话（信息差） | 35% | 拉播放量，制造讨论 |
| AI深度判断/趋势预判 | 35% | 涨粉和人设核心 |
| AI工具实操/工作流 | 30% | 收藏、转粉、建立信任 |

> ⚠️ **硬性约束**（不遵守 = 播放量再高也不涨粉）：
> - 连续发3条热点大白话之后，第4条**必须**是深度思考/趋势预判或工具实操
> - 单周热点大白话不得超过4条
> - 单周工具实操/工作流不得少于2条
> - 违反此规则，流量再高也不涨粉——数据已验证

**注意**：工具实操不是普通教程号写法。每条工具内容都必须回答一个问题：

> 这个工具背后，说明普通人的工作方式正在发生什么变化？

## 二、三类内容栏目写法

所有口播稿都必须先归入以下三类之一。不要混成“什么都讲一点”。

### 1. AI热点大白话：负责拉播放

**适合选题**：

- AI公司涨价、免费、亏损、收费
- 大厂发布新功能
- 招聘、裁员、行业变化
- 普通人听过但没看懂的AI新闻

**写法公式**：

```text
一个AI新闻
+ 普通人误解在哪
+ 晚枫查到的真相
+ 你该怎么应对
```

**开头示例**：

- “AI免费时代，可能真要结束了。”
- “豆包敢收500，不是因为它飘了。”
- “这次裁员，很多人看反了。”

**结尾必须导向关注**：

```text
关注晚枫，下一条讲普通人怎么应对。
```

### 2. AI深度判断/趋势预判：负责涨粉和人设

**适合选题**：

- AI对职场人的影响
- AI工具背后的行业变化
- 普通人学习AI的误区
- 未来工作方式的变化

**写法公式**：

```text
大多数人以为A
+ 真正的问题是B
+ 晚枫的判断
+ 普通人先做C
```

**开头示例**：

- “你用了AI反而更忙，不是错觉。”
- “普通人学AI，第一步就错了。”
- “会用AI的人，不一定更轻松。”

**人设锚点必须更强**：

```text
我是晚枫，专门把AI行业的话，翻译成普通人听得懂的真话。
```

### 3. AI工具实操/工作流：负责收藏、转粉、信任

**适合选题**：

- Codex、Claude Code、Agent、Skill
- AI办公自动化
- Excel、文案、资料整理、脚本生成
- 真实工作任务的前后对比

**写法公式**：

```text
我用AI做了一个真实工作任务
+ 以前怎么做
+ 现在怎么做
+ 结果变成什么
+ 普通人怎么复刻
```

**开头示例**：

- “我用Codex改了一个真实网站，结果有点离谱。”
- “这个Excel，我让AI自动生成了。”
- “Claude Code不是玩具，但新手别乱碰。”

**硬性要求**：

- 必须有具体工具名
- 必须有具体任务
- 必须有结果展示
- 不能写成软件说明书
- 结尾必须预告下一条实操

## 三、三平台适配规则

同一条内容可以三平台同步，但标题、开头和结尾不能完全一样。

### 抖音：冲突感优先

**目标**：拿停留、拿推荐、拿讨论。

**标题写法**：

- 我求你别再乱学AI了
- AI工具越免费，普通人越要小心
- Codex不是程序员玩具，打工人也该懂
- 这个AI工具我不建议新手乱碰

**结尾导向**：

```text
你怎么看？关注晚枫，下一条继续拆真相。
```

### 小红书：收藏感优先

**目标**：让用户觉得“以后用得上”。

**标题写法**：

- 普通人可复刻的5个AI办公流程
- 我用AI自动整理Excel的完整步骤
- Claude Code使用体验：适合谁，不适合谁
- 适合打工人的AI工具清单

**结尾导向**：

```text
这条先收藏，下一条我把步骤拆开讲。
```

### 视频号：职场传播优先

**目标**：适合转发给同事、老板、朋友。

**标题写法**：

- 打工人必须掌握的AI自动化能力
- 未来3年，办公能力会被AI重排
- 老板不会告诉你的AI提效方法
- 会用AI的人，正在重新定义工作方式

**结尾导向**：

```text
转给还在手动加班的朋友，关注晚枫，继续讲AI怎么改变工作。
```

## 四、最高优先级规则：第一句话必须把人留住

> ⚠️ **这是整个指南里最重要的一条，没有之一。**
> 前3秒留不住人，后面写得再好也没用——用户已经划走了。

### 核心要求

**第一句话（前3秒）必须完成以下任意一项**：

| 钩子类型 | 效果 | 示例 |
|---------|------|------|
| 反常识冲突 | 用户"诶？不对吧"→ 想继续看 | "芯片被卡脖子，但AI反而更便宜了" |
| 信息差暗示 | 用户"他有我不知道的东西"→ 想验证 | "腾讯文档大裁员，我查到了真相" |
| 数字锚点 | 用户被具体数字吸引 → 想了解 | "OpenAI亏了100亿，免费GPT要没了" |
| 直接打脸 | 用户"他在说我不对？"→ 想反驳 | "淘汰你的不是AI，是你身边那个会用AI的同事" |

**禁止使用的开头（一律删除）**：
- ❌ "今天教大家…"
- ❌ "你有没有发现…"
- ❌ "大家好，我是…"
- ❌ 任何铺垫词（"其实啊""话说回来""大家都知道"）

### 检查标准

写完开头第一句，问自己三个问题：
1. 用户刷到这条，会在3秒内划走吗？→ 会 = 重写
2. 第一句能让用户想"然后呢？"吗？→ 不能 = 重写
3. 如果把第一句删掉，后面的内容还能成立吗？→ 能 = 第一句没用，重写

### 成功案例（完播率最高）

**案例1（023视频，完播率12.6%）**：
> 第一句："腾讯文档大裁员，我查到了真相。"
> → 用户想：什么真相？你说出来看看对不对。

**案例2（002视频，完播率5.8%）**：
> 第一句："芯片被卡脖子，GPU到处缺货，按理说AI应该贵到离谱才对。"
> → 用户想：对啊，然后呢？怎么不对了？

## 五、人设锚点规则：每篇必须让用户记住你是谁

> ⚠️ **流量高但不涨粉，核心原因就是缺这条。**
> 用户看完觉得"讲得不错"→ 但不记得是谁讲的 → 不关注。

### 核心要求

**每篇稿子必须至少有一个"人设锚点"**——让用户看完知道"这是那个程序员晚枫讲的"。

| 插入位置 | 写法示例 |
|---------|---------|
| 开头钩子之后（推荐） | "我是晚枫，专门挖AI行业不愿意让你知道的事" |
| 反转之后 | "我是晚枫，只讲AI行业里别人不敢讲的真话" |
| 结尾互动之前 | "关注晚枫，AI行业的真话只在这里讲" |
| 工具实操中 | "我是晚枫，不只看发布会，我会直接拿真实工作试一遍" |

**注意**：不是每篇都写一样的话，而是每篇至少有一处——人设锚点可以换着说，但不能没有。

### 反面案例

| ❌ 没有锚点的结尾 | 问题 |
|---------------------|------|
| "收费了，你还用吗？" | 用户评论完就走了，不记得是谁 |
| "你怎么看？" | 没有任何人设记忆点 |

### 正面案例（竞品参考）

| 账号 | 锚点手法 | 效果 |
|------|---------|------|
| 老爸评测 | "较真、死磕"每期都出现 | 强人设记忆 |
| 直男财经 | "我是直男"每期开场 | 强关注转化 |

## 六、时长与节奏：观点完整优先，每句话必须推进

### 时长原则

| 内容类型 | 推荐时长 | 说明 |
|---------|---------|------|
| 单一信息差/单反转 | 45-60s | 适合快评、单点热点 |
| 2-3个递进观点 | 60-90s | 适合解读资讯、讲清一个判断 |
| 工具实操/工作流 | 60-120s | 适合展示过程、结果和复刻方式 |
| ＞120s | 谨慎 | 必须有强故事线和连续信息增量 |

**判断标准**：不是越短越好，而是不能散。

录完后问自己四个问题：

1. 这条的核心观点能不能用一句话说清？
2. 事实、判断、结论之间有没有因果关系？
3. 用户听完后，会不会觉得“他帮我看懂了一件事”？
4. 如果删掉某一句，观点会不会更清楚？会 = 删。

### 解读类内容必须有主线

热点解读不能写成“新闻碎片合集”。必须按这个顺序组织：

```text
表面发生了什么
+ 大多数人会怎么看
+ 晚枫认为真正关键是什么
+ 这件事和普通人有什么关系
+ 下一步应该关注什么
```

**底线要求**：

- 开头10秒必须抛出核心判断
- 中间每15-20秒必须推进一层，不重复同一个意思
- 结尾必须给用户一个清晰结论，不要只留问题
- 可以超过45秒，但不能超过一个清晰故事所需的长度

### 删掉所有冗余

| ❌ 必须删除 | 原因 |
|-------------|------|
| "你有没有发现/有没有这种感觉" | 铺垫词，浪费前3秒 |
| "其实啊/话说回来/更重要的是" | 口头禅，没信息量 |
| 连续超过3句没有推进 | 用户走神，必须加入反转/例子/结论 |
| 单句超过20字 | 太长，观众记不住 |

**节奏模板（资讯解读版）**：
```
0-5s   ：第一句话钩子 + 核心判断
5-20s  ：交代新闻事实，只讲必要信息
20-45s ：指出大众误读，给出你的反转判断
45-70s ：解释为什么，用具体产品/事件/数据支撑
70-90s ：落到普通人视角，给出结论和关注理由
```

## 七、内容质量：绑定具体产品，禁止空讲理论

### 必须绑定具体产品/事件

| ❌ 禁止（空讲理论） | ✅ 必须（绑定具体） |
|---------------------|---------------------|
| "AI很厉害，大家要学" | "豆包宣布收费500/月，普通人怎么办" |
| "AI可以帮你做副业" | "用ChatGPT做小红书文案，我试了2周" |
| "免费AI有风险" | "DeepSeek免费，但它在用你的数据训练" |

**数据支撑**：绑定具体产品的视频，平均播放量是纯理论内容的**3.6倍**。

### 工具内容必须绑定真实工作任务

| ❌ 禁止（像说明书） | ✅ 必须（像实战复盘） |
|---------------------|---------------------|
| "Codex有很多功能" | "我用Codex改了一个真实网站" |
| "Claude Code很好用" | "我用Claude Code处理了一个真实项目" |
| "AI可以做Excel" | "我用AI把发票整理成Excel" |

工具实操必须说清楚三件事：

1. 以前手动做，麻烦在哪里
2. 用AI之后，流程怎么变短
3. 普通人能不能复刻

### 观点闭环：必须讲清楚“所以呢”

每条稿子都要形成闭环：

```text
新闻是什么
为什么重要
你的判断是什么
普通人该看懂什么
```

禁止只说“这很重要”“这不是工具”“这是入口”，但不解释为什么。

### 情绪反转：至少翻转一次

```
负面/争议开头 → 反转成正面/另类角度 → 情绪曲线丰富
```

**反转手法**：
| 手法 | 示例 |
|------|------|
| 对比反转 | 豆包敢定500→ 国产AI有底气了 |
| 视角反转 | 收费不是坏事 → 逼你算账，值不值 |
| 时间反转 | 短期是坏消息 → 长期倒逼你学会用AI |

**禁止**：一路负面到底（用户划走）/ 一路正能量到底（用户觉得鸡汤）。

### 互动问题：必须带关注动机

> ⚠️ **评论多 ≠ 关注多。互动问题必须让用户有"关注理由"。**

**核心公式**：
```
互动问题 + "关注我，[具体下期预告/持续追踪这个话题]"
```

| ✅ 好问题（带关注动机） | ❌ 坏问题（只有评论动机） |
|------------------------|------------------------|
| "收费了，你还用吗？**关注我，下期告诉你哪款还能免费用**" | "收费了，你还用吗？"（评论完就走） |
| "你会怎么选？**关注晚枫，AI省钱方案持续更新**" | "你会怎么选？"（无关注理由） |

**底线要求**：
- 互动问题 ≤10字（主体部分）
- 关注动机 ≤15字
- 表演：直视镜头 + 停顿2秒

## 八、语言风格

1. **口语化**：像跟朋友聊天，不像老师讲课
2. **短句**：每句不超过20字
3. **零术语**：不出现任何代码/编程/技术黑话
4. **生活化比喻**：用观众脑子里已有的东西做类比（点外卖/开保险箱/放大镜）

**禁止医疗/药品类比**（合规红线）：
| ❌ 禁止 | ✅ 替换 |
|--------|--------|
| "处方药不需要处方" | "保险箱不需要密码，谁都能开" |
| "AI让人上瘾" | "用惯了就离不开，像离不开手机" |

## 九、标题与封面

### 标题公式
```
【数字/时间锚点】+【事件/产品】+【悬念/预测】
```
- 长度：15-25字
- 优先含具体数字（30天/1亿/500元）
- 有悬念感（"就要""居然""终于"）
- 工具实操类可以不用数字，但必须有具体任务（改网站/做Excel/整理资料）

### 三平台标题改写

同一选题必须至少准备3个标题：

| 平台 | 标题重点 | 示例 |
|------|----------|------|
| 抖音 | 冲突、反常识、争议 | "我求你别再乱学AI了" |
| 小红书 | 收藏、步骤、清单 | "普通人可复刻的AI办公流程" |
| 视频号 | 职场变化、转发价值 | "打工人必须掌握的AI自动化能力" |

### 封面公式

封面文案共4项，必须按以下顺序和类型标注：

| 项 | 类型 | 说明 |
|----|------|------|
| 第一行 | 📝文案 | 大众已知观点（1-5字） |
| 第二行 | 📝文案 | 反常识/扎心结论（1-5字） |
| 第三项 | 😐人物表情 | 主播面部情绪（如：震惊/冷笑/释然） |
| 第四项 | 🤳人物动作 | 主播肢体动作（如：手指屏幕/摇头/摊手） |

- 第一行+第二行越扎心、越有争议越好
- 用户刷到，2行看完，不点进去难受

**品牌展现规则**：

> ⚠️ 稿子里提到具体品牌/产品时，封面文案**必须加第五项**：

| 项 | 类型 | 说明 |
|----|------|------|
| 第五项 | 🏷️品牌展现 | 品牌Logo的位置、大小、与人物动作的关系 |

**品牌展现写法示例**：

| 品牌场景 | 品牌展现写法 |
|---------|-------------|
| 提到单一品牌 | "画面右侧1/3处放豆包Logo，主播手指指向Logo" |
| 对比多个品牌 | "画面左上角ChatGPT Logo vs 右上角豆包Logo，主播双手各指一侧" |
| 品牌为负面对象 | "画面左下角小字DeepSeek Logo，主播摇头摆手表示否定" |
| 品牌为正面对象 | "画面右侧半透明豆包Logo，主播竖大拇指，Logo与手势呼应" |

**注意**：
- 品牌Logo不得遮挡主播面部
- Logo尺寸不超过画面1/4
- 如果稿子没有提到任何品牌，不写第五项

## 十、合规红线（一律禁止）

1. ❌ 药品/医疗类比（处方药/治病/诊断）
2. ❌ 金融投资建议（稳赚/ guarantees/荐股）
3. ❌ 夸大承诺（第一/最强/保证（学会））
4. ❌ 涉政/社会敏感内容
5. ❌ 嘲讽具体的人（只嘲讽现象/行为）

## 十二、发布前自检流程（每条必做）

```
第一步：栏目归类
   └→ 这条属于热点、深度判断、还是工具实操？
   └→ 说不清 → 选题不清，重写

第二步：平台标题检查
   └→ 抖音/小红书/视频号标题是否分别适配？
   └→ 没有 → 不要三平台用同一个标题

第三步：开头3秒测试
   └→ 第一句话是钩子吗？（反常识/信息差/数字）
   └→ 不是 → 重写

第四步：人设锚点检查
   └→ 稿子里有"晚枫"/"我是"相关的锚点吗？
   └→ 没有 → 不加进去不让发布

第五步：时长检查
   └→ ≤45秒？（单一观点）或 ≤60秒？（多观点）
   └→ 超过 → 砍内容

第六步：冗余检查
   └→ 删掉所有"你有没有发现/其实啊/话说回来"
   └→ 每句不超过20字

第七步：互动问题检查
   └→ 有"关注我/关注晚枫"的关注动机吗？
   └→ 没有 → 补上再发布

第八步：工具实操额外检查
   └→ 是否有具体工具、具体任务、具体结果？
   └→ 没有 → 不算实操，重写

第九步：合规检查
   └→ 无药品/医疗类比
   └→ 无夸大承诺
   └→ 无针对具体的人

第十步：自己打分
   └→ "如果我是刷到的用户，这条我会关注吗？"
   └→ <8分 → 重写
```
