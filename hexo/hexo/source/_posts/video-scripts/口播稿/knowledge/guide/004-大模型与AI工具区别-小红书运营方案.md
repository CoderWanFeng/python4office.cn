## ⭐️ 关于本方案的核心约定

**小红书发布产品形态统一为「图集拼接视频 + 口播音轨」**，具体含义：

- ✅ 输出物是**视频**，不是纯图文，也不是只有真人出镜的口播视频
- ✅ 视频画面 = 一张张**辅助展示用的静态/动态图片**按时间轴拼接
- ✅ 视频音轨 = **口播稿录音**（节选自配套的004口播稿）+ BGM
- ✅ **图片张数不设上限**——越多越好，理想密度是**每 1.5-2 秒一张图**
- ✅ 图片本身**只是辅助理解**——核心信息载体是口播音轨
- ❌ 不再区分"图文笔记"和"视频笔记"——统一为图集拼视频
- ❌ 不再设"6 张/9 张"等张数限制

> 设计思路：让图片成为口播稿的"实时弹幕字幕+视觉锚点"，看到图就听懂内容，不看图听口播也能懂——双轨冗余，最大化完播率与收藏率。

## 二、内容矩阵设计

### 2.1 五大内容支柱

| 支柱 | 占比 | 内容方向 | 目标 | 对标口播稿 |
|------|------|---------|------|-----------|

| 🔥 **趋势解读** | 15% | "这个AI新功能到底能不能用" | 时效+搜索流量 | 热点驱动 |
| 🫶 **踩坑自白** | 10% | "我用AI踩过的3个坑" | 共情+信任+评论 | 人设层 |

### 2.2 内容比例

```
70% 有用干货（概念秒懂 + 场景实战 + 对比避坑）
20% 趋势跟风（新功能/新工具第一时间解读）
10% 人设内容（踩坑故事/日常/互动）
```

### 2.3 内容格式（统一为图集拼视频+口播）

| 维度 | 规范 |
|------|------|
| 输出物 | 9:16 竖版视频（小红书图集拼接而成） |
| 时长 | 30 秒-3 分钟（**按口播稿决定，不被图数限制**） |
| 配图密度 | **每 1.5-2 秒一张图**（30 秒视频 ≈ 15-20 张图，60 秒 ≈ 30-40 张图，3 分钟 ≈ 90+ 张图）|
| 图片张数 | **不限——越多越好** |
| 配音 | 真人口播 / TTS 合成（节选自配套口播稿） |
| BGM | 与口播稿配乐表一致（钩子 120 BPM / 概念 90 BPM / 受挫 120 BPM / 收尾 70 BPM） |
| 字幕 | 口播全程对齐字幕（小红书自动 OCR 索引），关键词压屏 |
| 直播补充 | 每周 1-2 场，与图集视频互补（占发布量 10%） |

> **图片只是辅助**——录口播时不要被"凑张数"绑架，先把口播录好，再按节奏切图。图越密集越好，但前提是每张图都对应口播的一个具体语义点（一个名词/一个动作/一个画面）。

## 四、爆款打造方法论

### 4.1 爆款标题公式

基于小红书AI赛道的爆款规律，提炼5套标题公式：

**公式1：反常识+数字钩**
```
❌ AI工具介绍
✅ 90%的人只用了AI的1/3能力
→ 反常识（"1/3？不可能吧"）+ 数字（90%、1/3）
```

**公式2：一句人话+场景**
```
❌ 桌面版AI功能介绍
✅ 一句话让AI帮你合并10个Excel，全程没出现一行代码
→ 具体动作（一句话）+ 具体数字（10个）+ 反差（没代码）
```

**公式3：对比冲突+选谁**
```
❌ Cursor和Trae功能对比
✅ Cursor和Trae到底选哪个？非程序员只看这一条
→ 站队式选择 + 目标人群精准
```

**公式4：踩坑自白+数字**
```
❌ AI使用注意事项
✅ 我用AI踩过的3个坑，第2个坑了我一整天
→ 第一人称 + 具体数字 + 悬念（第2个是什么）
```

**公式5：时效+紧迫**
```
❌ 新AI工具发布
✅ 刚出的XXX，实测3个场景，普通人现在用正好
→ 时效词（刚出）+ 实测 + 身份锚定（普通人）
```

### 4.2 封面设计体系

**封面3要素**：
1. **大字标题**（占画面40%）：黄色/白色粗体字，深色背景
2. **场景截图**（占画面50%）：真实的AI操作界面截图
3. **视觉钩子**（占画面10%）：红圈/箭头/数字标记

**封面模板**（4套轮换使用）：

| 模板 | 适用 | 布局 |
|------|------|------|
| A. 截图+大字 | 场景实战 | 上半屏AI操作截图，下半屏黄色大字标题 |
| B. 对比左右 | 对比避坑 | 左右分屏，两种工具的截图对比，中间"VS" |
| C. 纯色+金句 | 概念秒懂 | 深蓝/黑色背景，白色大字概念名+黄色小字解释 |
| D. 俯拍桌面 | 踩坑自白 | 笔记本电脑+咖啡+便利贴的俯拍，文字叠在屏幕上 |

### 4.3 统一发布结构（图集拼视频+口播）

```
【钩子段】口播 0-4 秒，对应配图 2-3 张
  反常识观点 / 数字钩 / 痛点共鸣
  → 大字封面 + 数字放大动效

【场景设定】口播 4-9 秒，对应配图 3-5 张
  谁、在哪、要干嘛——把场景画面拍出来
  → 桌面俯拍、文件名截图、时间字幕

【主体段】口播主干，对应配图按口播节奏切
  每讲一个概念 = 体验镜头(3-5图) + 命名Logo墙(2-4图) + 类比动画(2-3图) + 压屏金句(1图)
  → 一个 4 拍子约 8-12 张图

【冲突/受挫段】口播 1-2 秒，对应配图 2-3 张
  报错红屏快切、震动特效
  → 制造张力，配音效

【收尾段】口播 5-8 秒，对应配图 3-5 张
  总结金句 + 互动问题 + 关注引导
  → "你卡在第几代？评论区告诉我"

【总配图密度】每 1.5-2 秒一张，张数不限，越多越好
```

### 4.4 配图选择原则

每张图必须满足以下至少一项才值得放进去：

1. **对应口播一个具体名词**（如"DeepSeek"出现 → 配 DeepSeek Logo 图）
2. **对应口播一个动作**（如"复制代码" → 配 Ctrl+C 操作截图）
3. **对应口播一个数据/数字**（如"10 个 Excel" → 配 10 个文件图标排列图）
4. **对应口播一个情绪**（如"报错" → 配红色错误弹窗图）
5. **对应口播一个比喻**（如"发动机" → 配发动机图标）

> **不满足以上任一条的图，不要放进去**——避免"凑张数"的灌水图。

### 4.5 评论区运营策略

**发布后2小时黄金期**：

| 时间 | 动作 | 话术 |
|------|------|------|
| 0-5分钟 | 自评1条（补充信息+互动问题） | "补充一下：文里说的桌面版AI，我自己在用的是WorkBuddy，你们试过吗？" |
| 5-30分钟 | 回复所有评论 | 每条必回，用"你"开头，带表情 |
| 30-120分钟 | 引导讨论 | "评论区好多人问XXX，我统一说一下…" |
| 2小时后 | 置顶一条总结 | "这篇火了！补充3点…" |

**评论引导钩子**（埋在口播/字幕里）：
- "你卡在第几代？" → 制造站队
- "你们公司用哪个？" → 制造比较
- "评论区打出你的职业，我告诉你该用哪个" → 个性化互动

### 5.2 字体规范

| 用途 | 字体 | 大小 |
|------|------|------|
| 封面标题 | 阿里巴巴普惠体 Bold | 48-72pt |
| 正文 | 阿里巴巴普惠体 Regular | 14-16pt |
| 代码/英文 | JetBrains Mono | 12-14pt |
| 压屏金句 | 阿里巴巴普惠体 Black | 60-80pt |

### 5.3 拍摄/截图规范

| 维度 | 规范 |
|------|------|
| 画面比例 | **9:16 竖版**（图集拼视频统一规格）|
| 截图密度 | 每 1.5-2 秒一张，**张数不设上限** |
| 截图来源 | 必须是真实操作界面，禁止纯文字描述代替截图 |
| 标注 | 红圈/箭头指向关键操作点，黄色数字标步骤 |
| 风格 | 暗色系为主（深蓝背景），文字用亮色（黄/白） |
| 一致性 | 所有封面统一深蓝背景+黄字标题，品牌识别 |
| 切图节奏 | 与口播节拍对齐（钩子段切快、概念段稍缓、压屏停留 1.5s） |

| 层级 | 关键词 | 作用 |

### 6.2 话题组合策略

每篇笔记的话题标签公式：
```
2个大词 + 3个精准词 + 1个场景词 + 1个品牌词 = 7个标签
```

示例：
```
#AI工具 #职场提效 + #大模型 #桌面AI #WorkBuddy + #Excel合并 + #晚枫说AI
```

### 6.3 搜索优化

**搜索关键词埋入位置**（按权重排序）：
1. 标题（必须含1个搜索词）
2. 笔记正文前2行（图集视频也支持文字描述）
3. 话题标签
4. 图片上的文字（OCR可识别）——**图集视频每张图都参与 OCR，张数越多 OCR 关键词覆盖越广**
5. 评论区自评

> **图越多越好的额外好处**：每张图的字幕都参与小红书 OCR 索引，关键词曝光面更大。

## 九、风险控制

### 9.1 内容红线

| 禁区 | 说明 |
|------|------|
| 不说"最好/第一/绝对" | 违反广告法 |
| 不说"AI替代XX职业" | 制造焦虑，限流风险 |
| 不做金融/医疗类比 | 004口播稿合规清单已排除 |
| 不贬低竞品 | 对比时用"更适合XX人群"而非"XX不行" |
| 不承诺"学了就能赚钱" | 效率先行，搞钱是结果不是承诺 |

### 9.2 评论区风控

| 风险类型 | 应对 |
|---------|------|
| "AI会取代我们吗？" | 回复："AI不会取代你，会用AI的人才会。所以我在做这个号呀😊" |
| 负面评论"又在割韭菜" | 不删评，真诚回复："理解你的担心，我的内容全免费，先看看再说" |
| 竞品水军 | 不对骂，用数据说话 |
