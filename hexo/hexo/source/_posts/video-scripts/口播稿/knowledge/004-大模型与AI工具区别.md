## 创作背景

**热点来源**：用户分不清大模型、AI Coding 工具（Trae/Cursor/Claude Code 等）、桌面版 AI（WorkBuddy/Codex 等）之间的关系，以为都是"AI"
**创作角度**：用"我周二下午4点，领导丢过来10个Excel"这一个完整场景，按AI进化三阶段递进——聊天→写代码→操作电脑，每个阶段因上一个阶段的局限而生，每阶段补充来龙去脉（WHY）和第4拍（现实映射）
**主线任务**：10 个 Excel 汇总成 1 个周报（贯穿全程）
**三次受挫 → 解决**：
- 第一阶段·大模型：有脑没手，只能网页写代码自己复制 → **受挫 1**：报错看不懂，复制到崩溃
- 第二阶段·AI Coding：有脑有手，但只能动代码 → **受挫 2**：要求会编程，90% 的人被卡在门外
- 第三阶段·桌面版 AI：有脑有手有脚，直接操作电脑 → **解决**：一句人话搞定
**v5 关键升级**：
- ①共情瞬间："说实话，我自己最开始也是只会打开AI聊天"
- ②来龙去脉：每代出现的WHY——"大模型没法跑到你电脑上，但它想了个招：写代码""用编程来解决问题，对普通人太复杂了"
- ③第4拍（现实映射）：每个概念补了"下次你看到XX，就知道是XX"
- ④IDE/CLI故事化：从"规格说明"改为"车间盯着干活 vs 下军令状"
- ⑤发动机类比贯穿：每阶段触达类比，不只收尾用
- ⑥收尾加因果总结："每一代都是因为上一代干不了的事"
**人设锚点**：本稿人设锚点放在结尾互动前，具体文案："关注晚枫，AI 工具的真话只在这里讲"
**系列位置**：独立篇，接续"AI 基础概念 3 讲"之后，进阶区分 AI 产品形态

### 口播正文（完整连贯版·口语化）

你说你在用 AI 干活，但 90% 的人其实只用了 AI 的 1/3 能力——你可能一直停在第一代。

说实话，我自己最开始也是。打开 DeepSeek 聊两句，觉得挺好玩，然后就没了。直到有一天……

周二下午 4 点，领导丢过来 10 个 Excel，说下班前汇总成周报。我打开 DeepSeek，敲了句"帮我把这 10 个 Excel 合并"——秒回一段 Python 代码。

这就是**大模型**，也叫 LLM。你平时打开聊天的那些——国内 DeepSeek、千问、Kimi、豆包、文心一言，国外 ChatGPT、Claude、Gemini——**全都是大模型**。

大模型是什么？就是 AI 的大脑。你问它问题，它能回答；你让它写代码，它能写。但它有个致命的问题——**它只动嘴，不动手**。

为什么？因为大模型活在一个网页里。你的 Excel 在电脑硬盘上，它够不着。中间隔着一堵墙——它出不来，你的文件也送不进去。所以它只能给你一段代码，你还得自己复制、自己粘贴、自己运行。我当时复制到终端一运行——满屏红字报错。Python 都没装，运行个锤子？

所以下次你看到 AI 只给了你文字回答，别怪它懒——它真的没手。

> **"大模型 = 只动嘴，不动手"**（压屏 1.5s）

大模型自己也没办法——它没法跑到你电脑上来。但它想了个招：**写代码，让代码来替你操作**。

这就是第二代——**AI Coding**。

同事推荐我用 Trae。我打开一看，AI 直接在编辑器里写代码，写完自动运行，报错还能自己修——不用手动复制了！

Cursor、Claude Code、GitHub Copilot、Windsurf，国内还有 Trae、通义灵码、文心快码、CodeBuddy——**都属于这一类**。

相当于给 AI 的发动机**装了方向盘**——能开了。

方向盘还分两种。IDE 版——像 Cursor、Trae——你打开它，左边是代码右边是聊天，AI 写代码你直接看效果，就像在一个车间里，AI 干活你在旁边盯着。CLI 版——像 Claude Code——你打开一个黑窗口，打一行命令，AI 执行完给你结果，就像给 AI 下军令状，说完它就去做，做完了汇报。

但不管是哪种——**你得会编程才行**。

我就不会。我只想说一句"帮我合并 Excel"，AI Coding 听不懂。它只懂代码，不懂 Excel。它打不开你的文件，读不了你的表格，更不会帮你生成新文件。

说白了，AI Coding 是给程序员用的效率工具。那些 Cursor、Trae 宣传的"AI 帮你写代码"——前提是你得先会写代码。它不是帮你不用写，是帮你写得更快。90% 不会编程的人，直接被卡在门外。

> **"AI Coding = 动代码，但前提是你会编程"**（压屏 1.5s）

问题出在哪？**用编程来解决问题，对普通人来说还是太复杂了。**我不需要学编程，我需要的是——直接告诉 AI 要干嘛，它自己去干。

于是又有了第三代——**桌面版 AI**。

我打开 WorkBuddy，说了句"帮我把这 10 个 Excel 汇总成周报"——它自己打开文件，自己读数据，自己生成结果。全程没出现一行代码。

Codex、Manus、WorkBuddy、QoderWork、扣子 Coze——**这些都叫桌面版 AI**。

相当于发动机+方向盘+**四个轮子**——坐上去就能开，不用考驾照。

下次你看到有人说"一句话搞定 XX"——那大概率用的是桌面版 AI。因为只有它能真正动你电脑里的文件。

> **"桌面版 AI = 直接动你电脑里的文件"**（压屏 1.5s）

三代产品，打个比方你就全明白了：

- 大模型——**发动机**，动力再强，没轮子跑不起来
- AI Coding——**装了方向盘的发动机**，能开了，但得会开车
- 桌面版 AI——**一辆你坐上去就能开的车**，方向盘、轮子、车身全配齐了

开头说 90% 的人只用了 AI 的 1/3 能力——因为你可能一直停留在"跟 AI 聊天"这一步。现在你知道了，AI 已经进化了三代，**每一代都是因为上一代干不了的事**：

- 大模型只能说说 → 所以有了 AI Coding，让 AI 能动手
- AI Coding 只会写代码 → 所以有了桌面版 AI，让不会编程的人也能用

你用的是第几代？评论区告诉我。关注晚枫，AI 工具的真话只在这里讲。下期讲"桌面版 AI 怎么一秒看懂你的电脑"。

### 一、概念定义表

| 维度 | 大模型（LLM） | AI Coding | 桌面版 AI |
|------|--------------|-----------|----------|
| **一句话定义** | 基于海量数据训练的语言理解与生成模型 | 将大模型集成到编程环境中的代码辅助工具 | 可直接操作用户电脑文件和应用的 AI 客户端 |
| **核心能力** | 理解语言 → 生成文字（代码、回答、翻译等） | 理解代码上下文 → 自动编写/修改/运行代码 | 理解自然语言指令 → 操作本地文件/应用/工作流 |
| **核心局限** | 只能输出文字，无法执行任何操作 | 只能操作代码，无法操作非代码文件；需要用户会编程 | 依赖本地环境和权限；复杂工作流稳定性待提升 |
| **类比** | 汽车的发动机——动力强，没轮子 | 装了方向盘的发动机——能开，但得会开车 | 一辆坐上去就能开的车——方向盘+轮子+车身 |
| **压屏金句** | 只动嘴，不动手 | 动代码，但前提是你会编程 | 直接动你电脑里的文件 |
| **为什么出现** | — | 大模型只能输出代码不能运行代码，用户需要手动复制粘贴 | AI Coding 只懂代码不懂文件，且要求用户会编程 |
| **交互方式** | 网页/APP 对话框 | IDE 编辑器 或 CLI 终端 | 桌面客户端，对话式操作 |

### 二、AI Coding 子分类：IDE 版 vs CLI 版

| 维度 | IDE 版 | CLI 版 |
|------|--------|--------|
| **形态** | 嵌入代码编辑器（VS Code 等） | 运行在终端命令行 |
| **交互** | 左边代码右边对话，所见即所得 | 打命令→AI 执行→输出结果 |
| **代表产品·国内** | Trae、通义灵码、文心快码、CodeBuddy | — |
| **代表产品·国外** | Cursor、GitHub Copilot、Windsurf | Claude Code、OpenAI Codex CLI |
| **适合谁** | 日常写代码的程序员 | 偏好终端、自动化脚本的程序员 |
| **优势** | 可视化、实时预览、上手快 | 灵活、可脚本化、适合批量操作 |
| **共同局限** | 只能操作代码，不能操作电脑上的文件（Excel、Word、PDF 等） | 同左 |

### 五、三阶段对比矩阵（创作速查）

| 对比维度 | 大模型 | AI Coding | 桌面版 AI |
|---------|--------|-----------|----------|
| 能做什么 | 生成文字/代码 | 生成+执行代码 | 生成+执行+操作文件 |
| 不能做什么 | 不能运行、不能操作文件 | 不能操作非代码文件 | 复杂工作流稳定性有限 |
| 用户需要什么能力 | 打字即可 | 会编程 | 打字即可 |
| 解决的痛点 | — | 大模型只能输出不能执行 | AI Coding 要求会编程 |
| 留下的痛点 | 只能输出文字 | 只能操作代码 | 稳定性和权限管理 |
| 类比锚点 | 发动机 | 发动机+方向盘 | 整车 |
| 压屏金句 | 只动嘴，不动手 | 动代码，但得会编程 | 直接动你的文件 |
