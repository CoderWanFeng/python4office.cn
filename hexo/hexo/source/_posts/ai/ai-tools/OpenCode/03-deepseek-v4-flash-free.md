## 进阶使用技巧

### 技巧 1：让它"思考"后再回答
对于复杂问题，让模型先"思考"再给出答案，质量会大幅提升！

**提示词：**
```
问题：[你的复杂问题]

在给出最终答案之前，请先：
1. 列出来你理解的需求要点
2. 分析可能的几种解法思路
3. 对比各解法的优缺点
4. 最后给出你认为最优的方案和代码

请用"思考：... 结论：..."的格式回答。
```

---

### 技巧 2：角色扮演

让 AI 扮演特定角色，回答会更专业！

**扮演架构师：**
```
请你扮演一位有 10 年经验的资深技术架构师，回答我以下问题：

[你的问题]

要求：
1. 考虑可扩展性、可维护性、性能、成本
2. 给出至少 2 个方案对比
3. 说明 trade-off（权衡取舍）
4. 用接地气的话解释，不要太玄乎
```

**扮演面试官：**
```
请你扮演字节跳动的后端面试官，对我进行技术面试。

面试范围：
- Python 基础
- 计算机网络
- 数据库
- 算法题（LeetCode Medium 难度）

请一道一道问，我回答后你给出点评和打分（满分 10 分）。
```

---

### 技巧 3："教学模式"学习新技术
想学新技术？让 AI 用大白话教你！

```
我想学 [技术名称，比如：Docker]，但我是小白。

请用教学模式教我，要求：
1. 先讲清楚它是什么，解决什么问题
2. 用生活化的类比解释核心概念
3. 给出"Hello World"级别的入门代码
4. 分 5 节课，从入门到实战
5. 每节课后留练习题

现在开始第 1 节课。
```

## 注意事项

### 1. 偶尔会"犯傻"
即使是最强的模型，也会有出错的时候。**重要代码一定要自己测试验证！**

### 2. 提示词越清楚，结果越好
不要吝啬你的提示词字数。花 30 秒把需求描述清楚，能节省你 5 分钟改代码的时间。

### 3. 对话太长了怎么办？
如果上下文太长，可以让它总结一下：
```
总结一下我们刚才的对话，要点有哪些？后续的 Todo List 是什么？
```

然后开一个新对话，把总结贴进去继续。
