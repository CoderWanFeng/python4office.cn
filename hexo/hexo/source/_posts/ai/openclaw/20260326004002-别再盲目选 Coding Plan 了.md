上周接到一个朋友电话，说他们公司上个月云服务账单爆了 3 万块。我一问才知道：**当初选 Coding Plan 时，只看了免费额度，没看超出后的价格！**

这就像办手机卡——月租 19 块很香，结果流量超出后 10 块钱 1MB，用一次就破产。

今天我就把选 Coding Plan 最常见的 5 个坑列出来，帮你避开这些真金白银的教训。

## 🕳️ 坑 1：只看免费额度，不看超出价格

**真实案例**：
某团队选了 A 厂商，免费额度 100 万 tokens，看起来很美。结果项目上线后用量暴涨，超出部分 0.02 元/token，一个月账单 5 万+。

**避坑指南**：
```
✅ 一定要问清楚超出免费额度的单价
✅ 计算你预期的月用量，对比总成本
✅ 优先选阶梯定价的，用量越大越便宜
```

## 🕳️ 坑 2：QPS 限制没说清楚

**真实案例**：
B 厂商免费额度给得大方，但 QPS 只有 50 次/秒。结果用户一多，API 调用全被限流，系统直接卡死。

**避坑指南**：
```
✅ 问清楚 QPS 限制（每秒请求数）
✅ 评估你的并发需求
✅ 预留 2-3 倍余量
```

## 🕳️ 坑 3：隐藏费用防不胜防

**真实案例**：
C 厂商看似便宜，结果：
- 数据存储要钱
- 流量超出要钱
- 技术支持要钱
- 发票还要额外加税点

最后算下来比原价贵 30%。

**避坑指南**：
```
✅ 要求销售列明所有可能费用
✅ 问清楚发票税点谁承担
✅ 确认是否有最低消费
```

## 🕳️ 坑 4：合同陷阱——自动续费

**真实案例**：
D 厂商首年 5 折很香，结果合同里写了"自动续费原价"。第二年想换厂商，发现已经被绑定一年。

**避坑指南**：
```
✅ 仔细阅读合同条款
✅ 关闭自动续费
✅ 设置续费提醒，提前评估
```

## 🕳️ 坑 5：技术支持响应慢

**真实案例**：
E 厂商价格便宜，但出了问题工单 3 天才回复。业务停摆 3 天，损失远超省下的钱。

**避坑指南**：
```
✅ 测试售前响应速度（这是最好的时候）
✅ 问清楚 SLA 服务等级协议
✅ 确认是否有专属技术支持
```

## 🎯 我的选购公式

```
总成本 = 免费额度内成本 (0) + 超出部分 × 单价 + 隐藏费用

优先级：
1. 免费额度 > 预期用量 → 选这个
2. 超出单价低 → 备选
3. QPS 满足需求 → 必须
4. 隐藏费用少 → 加分
5. 技术支持好 → 保险
```

## 💡 终极建议：混合使用

我现在的策略：
- **开发测试**：用 OpenClaw 社区版（免费）
- **小流量生产**：用阿里云（免费额度够）
- **大流量业务**：用华为云（超出单价低）
- **微信生态**：用腾讯云（集成好）

**鸡蛋别放一个篮子里，免费额度要薅尽。**
