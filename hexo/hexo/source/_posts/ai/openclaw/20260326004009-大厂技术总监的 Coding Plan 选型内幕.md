大家好，我是正在实战各种 AI 项目的程序员晚枫。

## 🤫 开篇：我参加了一场云厂商闭门会

上个月，受邀参加一个云厂商的技术闭门会，现场都是各公司的技术总监/CTO。

茶歇时和几个同行聊天，听到了很多**销售永远不会告诉你的真相**。

今天把这些内幕扒出来，帮你避开那些"看起来很美"的坑。

## 🎭 内幕 1：免费额度是钓鱼的

### 销售的套路

```
销售："我们免费额度 200 万/月，够你用半年！"
真相：等你用量起来了，免费额度突然"调整政策"

真实案例：
某团队用 A 厂商，前 3 个月免费额度 200 万/月
第 4 个月突然通知："政策调整，降到 50 万/月"
想迁移？API 不兼容，数据格式私有
最后只能接受涨价，单价涨了 3 倍
```

### 避坑指南

```
✅ 签合同前确认：免费额度保证期多久？
✅ 问清楚：调整后提前多久通知？
✅ 要求：书面承诺，不要口头保证
✅ 准备：Plan B，随时能迁移
```

## 🎭 内幕 2：阶梯定价的猫腻

### 表面上的优惠

```
用量越大，单价越低，看起来很划算：
- 0-100 万：0.02 元/token
- 100-500 万：0.015 元/token
- 500 万 +：0.01 元/token
```

### 实际的陷阱

```
陷阱 1：按月结算，不累计
你 1 月用了 600 万，单价 0.01 元
2 月用了 50 万，单价还是 0.02 元（没到阶梯）
平均下来，单价根本没便宜多少

陷阱 2：超额部分按高价算
免费额度 100 万，你用了 150 万
以为超出 50 万按 0.015 元算？
错！有些厂商超出部分全部按 0.02 元算

陷阱 3：阶梯门槛悄悄提高
年初说 500 万进第三阶梯
年中说"政策调整"，提到 800 万
```

### 避坑指南

```
✅ 要求：按年累计，不是按月
✅ 确认：超额部分按哪档价格
✅ 书面：阶梯门槛写进合同
✅ 监控：用量接近门槛时告警
```

## 🎭 内幕 3：SLA 保障的真相

### 合同上的数字

```
SLA 99.9% = 全年宕机不超过 8.76 小时
SLA 99.95% = 全年宕机不超过 4.38 小时
SLA 99.99% = 全年宕机不超过 52 分钟
```

### 实际的计算方式

```
真相 1：只算"计划内宕机"
计划外宕机（故障）不算在 SLA 里
维护窗口可以随便定

真相 2：赔偿上限很低
SLA 不达标，赔偿当月服务费的 10%
你损失 100 万业务，他赔你 1000 块

真相 3：举证困难
你说宕机了，他说"我们监控正常"
日志在他手里，你拿不到证据
```

### 避坑指南

```
✅ 自己监控：别信厂商的监控数据
✅ 独立日志：所有请求本地留痕
✅ 赔偿条款：争取更高赔偿比例
✅ 多云部署：别把鸡蛋放一个篮子
```

## 🎭 内幕 4：技术支持的等级

### 销售说的

```
"我们有 7×24 小时技术支持！"
```

### 实际的

```
普通客户：工单响应，24-48 小时回复
VIP 客户：专属群，工作时间响应
SVIP 客户：电话支持，7×24 小时

关键：你是什么客户，取决于你花了多少钱
年消费<10 万：普通工单
年消费 10-50 万：专属群
年消费>50 万：电话支持
```

### 避坑指南

```
✅ 问清楚：什么级别对应什么支持
✅ 写进合同：响应时间、赔偿条款
✅ 测试售前：售前响应快，售后不一定
✅ 准备自救：文档、社区、备份方案
```

## 🎭 内幕 5：数据迁移的隐性成本

### 销售说的

```
"随时可以迁移，我们支持标准格式！"
```

### 实际的

```
真相 1：API 不兼容
每家都是"标准 RESTful"，但参数、返回值全不一样
迁移 = 重写代码

真相 2：数据格式私有
训练数据、微调模型、知识库
导出？可以，但格式只有他家能用

真相 3：迁移成本被低估
销售说"一周搞定"
实际：1-3 个月，还要业务停摆
```

### 避坑指南

```
✅ 选型时：优先选兼容 OpenAI 协议的
✅ 抽象层：必须做，屏蔽底层差异
✅ 定期导出：数据定期备份，验证可用性
✅ 多云策略：别让自己被绑定
```

## 📊 大厂技术总监的选型 checklist

### 必问问题（销售回避就警惕）

- [ ] 免费额度保证期多久？书面承诺？
- [ ] 阶梯定价按年累计还是按月？
- [ ] SLA 计算方式？赔偿比例？
- [ ] 技术支持级别？响应时间？
- [ ] 数据导出格式？迁移成本？
- [ ] 合同到期后自动续费？能否取消？
- [ ] 价格调整提前多久通知？

### 必做事项

- [ ] 自己搭建监控，不依赖厂商
- [ ] API 抽象层，随时可切换
- [ ] 数据定期备份，验证可用性
- [ ] 至少准备一家备选厂商
- [ ] 关键条款写进合同，不要口头承诺

## 💬 金句总结

> 销售的嘴，骗人的鬼。合同上的字，才是真的。

**技术选型，别听销售说什么，要看合同写什么。**
