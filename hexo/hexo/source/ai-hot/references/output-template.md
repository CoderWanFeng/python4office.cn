## 标题生成规则

每个选题给 **3 个角度**的候选标题，避免 AI 味太重：

### 1. 数字型（适合"工具盘点"）
公式：`数字 + 名词 + 收益`
- 例：「我用这 3 个 Python 库，让 Excel 处理快了 10 倍」
- 例：「5 分钟搞定！Claude Code 批量处理 200 个表格」

### 2. 痛点型（适合"解决方案"）
公式：`痛点 + 转折 + 方案`
- 例：「还在手动复制粘贴？这个 AI 工具一键搞定」

### 3. 反常识型（适合"颠覆认知"）
公式：`反直觉 + 解释 + 收益`
- 例：「AI 写代码比人慢 3 倍？但用在办公场景反而更快」

---

## 相关性判断规则

每条热点打 3 个标签之一：

| 标签 | 标准 |
|---|---|
| 🟢 **必写** | 直接对应 Python 办公 / AI 工具教程，开箱即用 |
| 🟡 **可写** | 跟 AI 沾边但需要二次加工（如海外模型需要本土化讲解） |
| 🔴 **不建议** | 跟目标读者无关，或时效性太短（24h 后就没意义） |

---

## 选题密度评估

| 密度 | 标准 | 建议 |
|---|---|---|
| 高 | 🟢 ≥ 3 条 | 当天就发，抢热点 |
| 中 | 🟢 1-2 条 | 当天准备，第二天发 |
| 低 | 0 条 🟢 | 改用历史日报补选题，或休息一天 |

---

## 时段建议

| 时段 | 适合发什么 |
|---|---|
| 早 8:00 | 当日新闻型 / 工具发布型 |
| 午 12:00 | 教程型 / 实操型（午休看） |
| 晚 21:00 | 深度型 / 观点型（下班后细看） |

公众号流量峰值：早 7-9 点、午 11-13 点、晚 20-22 点。
