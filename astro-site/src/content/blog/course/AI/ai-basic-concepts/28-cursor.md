# 什么是Cursor？

**Cursor = AI帮你写代码的编辑器**

> **作者：程序员晚枫**

::: tip 一句话解释
Cursor是一个代码编辑器，但内置了AI。你写代码时AI帮你自动补全、帮你找bug、帮你解释代码、甚至帮你写整个功能。程序员用它，写代码速度翻倍，不会的直接问AI。
:::

## Cursor是什么？

### 类比理解

```
普通编辑器 = 打字机
  → 你写什么，它显示什么
  
VS Code = 智能打字机
  → 有代码高亮、自动补全

Cursor = 会写代码的助手
  → 你说需求，AI帮你写
  → 不会的代码，AI帮你解释
  → 出了bug，AI帮你修
```

### 为什么火？

```
以前写代码：
1. 打开搜索引擎
2. 搜"怎么实现XXX功能"
3. 看Stack Overflow答案
4. 复制粘贴改一改
5. 报错继续搜...
时间：30分钟起步

Cursor：
1. 描述你要做什么
2. AI给你代码
3. 直接用
时间：5分钟
```

## Cursor核心功能

### 1. AI自动补全（Tab键）
```
你写：def calculate
Cursor自动补全：
def calculate_total(items):
    return sum(item['price'] for item in items)
→ 按Tab直接接受
```

### 2. AI聊天（Ctrl+L）
```
你问：这段代码为什么报错？
AI答：因为类型不一致，
      第23行需要转成int

你问：帮我写一个用户登录功能
AI答：给出一套完整的登录代码
```

### 3. AI自动改代码（Apply）
```
选中代码 → 说"改成XXX"
→ AI自动改完
→ 不满意可以继续改
```

### 4. 代码理解
```
光标停在任何代码上
→ AI解释这段代码是干嘛的
→ 不用问同事了
```

## 使用技巧

### 新手用法
```
1. 打开Cursor → 新建文件
2. 按Ctrl+L打开AI聊天
3. 说"帮我写一个计算器"
4. 拿走用
```

### 进阶用法
```
1. 光标放在报错代码上
2. 按Ctrl+K说"修复这个bug"
3. AI自动修复 → Tab接受

批量修改：
1. 选中多行代码
2. 按Ctrl+K说"把这里的变量名改成驼峰命名"
3. 批量改完
```

### 高手用法
```
1. 打开整个项目
2. 按Ctrl+L问"这个项目的架构是怎样的"
3. AI分析整个项目
4. 继续问"帮我加一个新功能XXX"
5. AI理解上下文，生成符合项目风格的代码
```

## 🎯 实战技巧：Cursor高效使用指南

### 技巧1：Ctrl+L = 随时问AI

```
Cursor里最常用的快捷键：

Ctrl+L：打开AI聊天
  → 可以问任何问题
  → 可以让AI写代码
  → 可以让AI解释代码

Ctrl+K：编辑当前代码
  → 选中代码后按Ctrl+K
  → 说"修复这个bug"
  → AI修改 → Tab接受

Ctrl+I：composer模式
  → 生成新文件
  → 整个项目帮你写
```

### 技巧2：用好Composer模式

```
Composer = Cursor的最强功能

可以：
1. 让AI同时创建/修改多个文件
2. AI理解整个项目的上下文
3. 批量生成代码

示例：
"帮我写一个Todo App，要有：
- 前端页面
- 后端API
- 数据库连接
- 测试文件"

→ Cursor会一次性生成所有文件！
```

### 技巧3：Tab补全提高效率

```
Cursor的Tab键比Copilot更智能：

1. 代码补全：
   → 你写前半段，Cursor猜后半段
   → 按Tab接受

2. 多行补全：
   → Cursor预测接下来几行
   → 一次Tab接受多行

3. 连续补全：
   → 接受一行后，自动预测下一行
   → 连续Tab，代码自己写出来

效果：写代码速度提升3-5倍
```

### 技巧4：用好Rule配置

```
Cursor支持自定义Rule，让AI更懂你：

方法：
1. Settings → Rules for AI
2. 添加你的项目规范

示例Rule：
你是一个Vue3开发者，
- 优先使用Composition API
- 使用TypeScript
- 代码要有中文注释
- 遵循Vue3最佳实践
```

效果：AI写的代码更符合你的项目风格！
```
